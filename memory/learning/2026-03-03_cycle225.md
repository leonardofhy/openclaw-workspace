**3. AudioSAEBench: Design and Methodology**

**3.1 Framework Overview**

AudioSAEBench evaluates sparse autoencoders (SAEs) applied to speech and audio language models across six categories that together span Pearl's three levels of causal inquiry (Joshi et al., arXiv:2602.16698): observational evidence (Categories 1–3), interventional change (Category 4), and counterfactual isolation (Categories 0 and 5). Each category operationalizes a distinct facet of the central question: *does this SAE feature constitute a valid causal abstraction of the underlying speech computation?*

We draw on the unifying framework of Geiger et al. (arXiv:2301.04709), under which all mechanistic interpretability methods — including probing, activation patching, and SAE feature attribution — are special cases of interchange interventions with different alignment map strictness. AudioSAEBench instantiates this framework at SAE feature granularity: for each feature $F$ and target attribute $A$, we conduct progressively stricter alignment tests from correlation (Category 1) to counterfactual isolation (Category 0). As Sutter et al. (NeurIPS 2025 Spotlight, arXiv:2507.08802) prove, without a linearity constraint on the alignment map, any neural network can achieve 100% IIA against any causal model — making causal abstraction vacuous without such constraints. All interventional and counterfactual categories (4, 0, 5) therefore use linear activation patching exclusively.

We benchmark SAEs trained on six model families: Whisper-base, Whisper-small, Whisper-large-v3, HuBERT-base, WavLM-large, and Qwen2-Audio-7B. For each model, we evaluate three SAE architectural variants where implementations are publicly available: TopK (OpenAI-style), BatchTopK, and Matryoshka (following SAEBench, Karvonen, Nanda et al., ICML 2025, which found Matryoshka wins on text LM disentanglement). SAEs are loaded from AudioSAE (Aparin et al., EACL 2026, \url{github.com/audiosae/audiosae\_demo}) and Mariotte et al. (ICASSP 2026, \url{github.com/theomariotte/sae\_audio\_ssl}); we train additional TopK SAEs on Whisper-small using a SAELens-compatible audio training toolkit (see §3.8) to ensure reproducibility.

**3.2 Category 0: Audio-RAVEL — Causal Disentanglement**

The most fundamental question for any audio SAE feature is not whether it *correlates* with a phonological attribute, but whether it *causes* the attribute to change and *isolates* the change from other attributes. We operationalize this as **Audio-RAVEL**, directly extending the RAVEL framework (Huang et al., ACL 2024) from text to audio SAE features.

For each feature $F$ and phonological attribute $A \in \{\text{voicing, manner, place}\}$, we compute:
$$\text{Cause}(F, A) = \Pr\left[\Delta \hat{A} = \text{expected} \mid \text{patch}(F, \delta_A)\right]$$
$$\text{Isolate}(F, A) = \Pr\left[\forall B \neq A,\; \Delta \hat{B} \approx 0 \mid \text{patch}(F, \delta_A)\right]$$
$$\text{RAVEL-audio}(F, A) = \frac{2 \cdot \text{Cause} \cdot \text{Isolate}}{\text{Cause} + \text{Isolate}}$$

where $\text{patch}(F, \delta_A)$ denotes a targeted activation patch that shifts feature $F$'s activation by $\delta_A$ (the mean activation difference between the two sides of a minimal phonological pair), and $\hat{A}$ is measured via a frozen phonological classifier trained on the same stimuli.

*Stimuli.* We use phonological minimal-pair stimuli from Choi et al. (arXiv:2602.18899), who validate linear phonological arithmetic across 96 languages ($\mathbf{h}(\text{[b]}) = \mathbf{h}(\text{[d]}) - \mathbf{h}(\text{[t]}) + \mathbf{h}(\text{[p]})$). Each pair differs in exactly one phonological attribute, holding all others constant. For the connector transfer test (Qwen2-Audio), we additionally apply SpeechTokenizer (Sadok et al., arXiv:2506.04492, Interspeech 2025) codec-grounded corruption: swapping only the semantic Layer 1 RVQ tokens isolates content change without touching speaker voice (Layers 2+), providing the cleanest single-factor causal stimulus design (directly addressing Core Question 1: how to design audio corruptions that isolate exactly the causal factor of interest).

*Baseline.* We use Multi-task Distributed Alignment Search (MDAS; Huang et al., ACL 2024) as the ceiling baseline, which simultaneously optimizes orthogonal subspaces for all attributes — representing the mechanistic ideal where attributes are perfectly disentangled. We expect Audio-RAVEL to reveal a systematic finding: audio SAEs score high on Cause but fail on Isolate (matching the RAVEL finding for text SAEs, Huang et al. 2024), and that the failure is *larger* for audio than text due to acoustic co-occurrence in training corpora (voiced phonemes correlate with speaker gender; background noise correlates with emotional arousal in naturalistic speech datasets).

**3.3 Category 1: Acoustic Concept Detection**

**(a) Feature-concept F1 (time-resolved).** For each SAE feature $F$, we compute the maximum-activation concept recall across LibriSpeech (ASR phonemes), IEMOCAP (emotion), ESC-50 (environmental sounds), and VocalSet (singing technique). Unlike prior work (AudioSAE, Mariotte), we measure temporal resolution: we retain per-timestep feature activation rather than pooling along the time axis, enabling *time-resolved* concept assignment — which frames does feature $F$ fire on, and do those frames align with the expected phoneme segment boundaries?

**(b) Temporal Coherence Score TCS(F).** We introduce TCS(F) as a second acoustic concept metric:
$$\text{TCS}(F) = \frac{\text{Var}_{\text{within-phoneme}}(F)}{\text{Var}_{\text{across-boundary}}(F)}$$
where within-phoneme variance is computed across frames within the same phoneme segment (forced-aligned via Montreal Forced Aligner, Shafran et al. 2602.02464 provides phoneme boundary annotations as a byproduct), and across-boundary variance is computed across frames straddling phoneme boundaries. A feature with $\text{TCS}(F) \approx 0$ is temporally stable within phonemes but jumps across boundaries — the hallmark of a phoneme-level rather than frame-level feature. This metric directly evaluates whether SAE features exploit temporal structure (as proposed by T-SAE, Bhalla et al., ICLR 2026 Oral, arXiv:2511.05541) or treat frames as i.i.d. tokens (the failure mode T-SAE diagnoses in standard SAEs). TCS(F) has no text LLM analogue, making it a genuinely audio-native contribution to SAE evaluation.

**3.4 Category 2: Disentanglement and Completeness**

We extend the completeness metric introduced by Mariotte et al. (ICASSP 2026) — which measures whether SAE features encode acoustic attributes more independently than raw hidden states — to a cross-model comparison at matched scale. For each voice attribute $v \in \{\text{pitch, shimmer, loudness, spectral rolloff, HNR, gender, accent}\}$, we train a linear probe on SAE feature activations and on raw hidden states separately, and report the completeness score:
$$\text{Completeness}(F, v) = \frac{\text{Probe accuracy with SAE features}}{\text{Probe accuracy with raw hidden states}} - 1$$

Positive values indicate that SAE disentanglement improves concept accessibility; negative values indicate reconstruction information loss. We test all six model families at matched layer depth ($L / L_{\max} \in \{0.25, 0.5, 0.75, 1.0\}$) to enable cross-model comparison, filling the gap that Mariotte's 4-model study did not close (each model was evaluated on different tasks at incomparable scales).

**3.5 Category 3: Reconstruction Fidelity**

We report task preservation ratio across three tasks:
$$\text{TPR}(\text{task}) = \frac{\text{Task score with SAE-reconstructed activations}}{\text{Task score with original activations}}$$
for ASR ($\text{task} = \text{WER}$, lower better), emotion recognition ($\text{task} = F_1$, higher better), and sound event detection ($\text{task} = F_1$). For Whisper-based models, we insert the SAE at each layer in turn and measure WER on LibriSpeech-test-clean (100h). This extends AudioSAE's single-metric reporting (WER penalty of +0.4% for top-100 feature steering, Aparin et al. EACL 2026) to multi-task and cross-model evaluation.

**3.6 Category 4: Causal Controllability**

We implement a three-metric protocol based on the patching best-practices of Heimersheim \& Nanda (2024):

- $\text{ablation\_d}(F)$: Cohen's $d$ between SAE-ablated and control accuracy on AudioSAE hallucination stimuli. Measures *necessity* (the AND-gate component): is feature $F$ causally required for the target behavior?

- $\text{steering\_precision}(F)$: fraction of behavior change attributable specifically to feature $F$ when activated, measured by comparing single-feature steering to top-K steering. Measures *sufficiency* (the OR-gate component).

- $\text{hydra\_compensation}(F)$: ratio of behavior restoration when the top-$K$ features are ablated simultaneously vs. top-1. Per Heimersheim \& Nanda (2024), backup pathway compensation averages 0.7× in text LLMs (the Hydra effect). If audio models exhibit lower or higher ratios, this indicates differences in circuit redundancy between speech and text representations.

All patching in Category 4 uses *denoising* patching (patching *toward* the clean/grounded state from the corrupt/ungrounded state), following Heimersheim \& Nanda (2024)'s recommendation that denoising is more reliable than noising patching for audio stimuli — noising creates OOD internal activations sensitive to noise magnitude. For SPIRIT adversarial stimuli (Djanibekov et al., EMNLP 2025, arXiv:2505.13541), we additionally test whether top-K feature ablation (the SPIRIT defense) can be reduced to single-feature ablation via AudioSAEBench's $\text{steering\_precision}$ score — a potential simplification of SPIRIT's neuron-level patching to SAE feature-level patching.

**3.7 Category 5: Grounding Sensitivity**

For each SAE feature $F$, we operationalize grounding sensitivity as IIT accuracy at feature granularity (Geiger et al., arXiv:2301.04709):
$$\text{gc}(F) = \text{IIA}\left(\mathcal{M},\; x_{\text{audio}},\; x_{\text{text}},\; F\right)$$

Concretely: we use the 57,000 audio-text conflict stimuli from ALME (Li et al., arXiv:2602.11488), where audio content and textual context deliberately contradict each other. For each stimulus pair $(x_\text{audio-only}, x_\text{text-only})$, we measure the SAE feature's activation in each condition. A feature with high $\text{gc}(F)$ activates when the audio cue is present and deactivates when only text predicts the concept — it is grounded to the audio signal. A feature with $\text{gc}(F) \approx 0$ fires to text-predictable patterns regardless of acoustic input — it recapitulates language model predictions rather than audio perception.

We compute $\text{gc}(F)$ on the subset of Whisper-small/large-v3 and Qwen2-Audio-7B where a text pathway exists (encoder-decoder models and LALMs); for encoder-only self-supervised models (HuBERT, WavLM), we use the decoder text generation as the "text proxy." This connects directly to Paper A's gc(L) layer-level grounding coefficient: Paper A localizes the listen layer at the resolution of network depth; Paper B scales the measurement to the resolution of individual SAE features. The same NNsight patching infrastructure (Wu et al., NNsight 2023) and the same ALME stimuli are reused across both papers.

**3.8 Training and Baseline SAEs**

To ensure reproducibility, we contribute a SAELens-compatible (Bloom et al., decoderesearch/SAELens v6) audio SAE training toolkit — the first time a standardized audio SAE training pipeline has been made available. All existing audio SAE papers (AudioSAE, Mariotte, AR&D, Plantinga, Paek et al.) use one-off training code incompatible with the SAELens ecosystem, which has become the de facto SAE training library for text LLMs (25 pre-trained models, Gemma-scope / GPT-2 / LLaMA). Our toolkit provides audio-specific modifications: (i) frame-level hook extraction via NNsight (compatible with Whisper cross-attention layers and HuBERT CNN feature extractor), (ii) phoneme-boundary-aware batch construction for TCS(F) evaluation, and (iii) a SAELens-compatible model card schema for audio SAEs with standardized metadata (model, layer, SAE type, audio task family). We release all trained SAEs to HuggingFace with the `saelens-audio` tag, filling SAELens v6's current gap of zero audio/speech pre-trained SAEs.

**3.9 Experimental Setup**

All MacBook-feasible categories (1–4) are run on Whisper-base/small (244M parameters) using NNsight hooks. Qwen2-Audio-7B experiments (Category 5, Grounding Sensitivity; Category 0 connector test) are conducted via NDIF remote execution (Fiotto-Kaufman et al., 2023). Full evaluation on 12+ SAEs across all models is run on a GPU workstation. Activation patching for Categories 0 and 4 is implemented via \texttt{pyvene}'s \texttt{RotatedSpaceIntervention} (Wu et al., 2024) with NNsight hooks for activation extraction, consistent with Paper A's implementation. All experiments are version-controlled and code is released with the benchmark.
