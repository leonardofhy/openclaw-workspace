## Paper: Transcoders Find Interpretable LLM Feature Circuits
arXiv:2406.11944 | Dunefsky et al. | NeurIPS 2024

## Problem
SAE features = linear combos of many neurons → circuits are huge or conflate local/global behavior.

## Method
- Replace dense MLP with wider, sparse MLP (transcoder)
- Circuit factorizes: **input-dependent** (audio/content) + **input-invariant** (structural prior)
- Tested 120M–1.4B param LMs; on par with SAEs in sparsity + faithfulness

## Connection to gc(k) / T-SAE (Q094)

**Mapping**:
| gc(k) concept | Transcoder concept |
|---|---|
| gc(k) peak = audio still "heard" | input-dependent terms dominate |
| t* collapse = text-prior takes over | input-invariant terms dominate |
| T-SAE feature | high-input-dependence transcoder feature |

**Q094 design implications**:
- T-SAE feature = SAE feature with high input-dependence coefficient
- Track deactivation of T-SAE features step-by-step through decoder
- Early deactivation (before t*) = early warning signal
- `tsae_gc_correlation_mock.py` should compute deactivation onset vs t*; correlation table

**Proxy without training transcoders**:
- Approximate input-dependence via attention contribution from audio tokens to residual stream
- Features with high audio-token attention weight ≈ input-dependent → proxy T-SAE
