## Why Erasure Fails
1. **PCA coverage = 0.5%** of true jailbreak subspace — directions are misestimated because `make_activation` uses rolling rng (different base_dir per call → train/test mismatch)
2. Reveals real-world analogue: representation drift means direction estimation requires careful calibration
3. 5D erasure doesn't beat 1D → bottleneck is **direction quality**, not count
