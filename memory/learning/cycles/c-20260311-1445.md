## Problem
RAVEL (Huang et al., ACL 2024) benchmarks interpretability methods on **attribute disentanglement**: can a method localize one attribute (e.g., emotion) in a neural representation without bleeding into others (e.g., speaker identity, noise)?

## Method — Core Mechanics

**Setup**: entities E with attributes A₁…Aₖ (polysemantically encoded in neurons)

**Interchange Intervention** (activation patching):
1. Run model on base input x (e.g., audio of speaker S1 saying angry utterance)
2. Run model on source input x' (e.g., audio of speaker S2 saying happy utterance)
3. Patch feature F (emotion subspace) from x' into x
4. Compare model output with/without patch

**Cause Score** (causal efficacy): After patching F, does the model now predict the **source's emotion** (happy)? Measures: did we actually move the emotion representation?
> Cause(F, A) = P(model outputs A_{E'} | patched feature F from E')

**Isolate Score** (disentanglement quality): After patching F for emotion, does the model still predict the **correct speaker identity** (S1)? Measures: did we accidentally leak speaker into the patch?
> Isolate(F, A) = P(model outputs correct other attributes | patched feature F)

**Leakage** = High Cause + Low Isolate → feature F carries emotion *and* speaker simultaneously → polysemantic entanglement

**ESN in Q040** = Emotion × Speaker × Noise — the 3 audio attributes to disentangle
- 2×2 design: {emotion: angry/happy} × {speaker: S1/S2} — swap one, keep other fixed
- Measure Cause + Isolate for emotion feature at each Whisper layer

## Results (original RAVEL paper)
- Supervised probes: high Cause, moderate Isolate
- PCA: low Cause, good Isolate
- SAE (sparse autoencoders): best Cause AND Isolate — most disentangled
- MDAS (Multi-task DAS): SOTA — finds distributed features satisfying multiple causal criteria simultaneously

## Connection to Tracks

**T3 (Listen vs Guess)**: gc(k) measures how much audio-layer output causally affects final prediction. RAVEL's Cause metric is the same operation — it IS causal patching. gc(k) = Cause score at a specific layer.
→ Q040 is applying gc(k) + RAVEL framework to audio attributes

**T5 (Listen-Layer Audit)**: Isolate score = safety-critical. If a jailbreak attack patches "malicious intent" feature but leaks into "safety refusal" feature → high Cause for malicious output + low Isolate for safety = safety failure has same signature as attribute leakage.

## Implementation Plan for Q040 (Tier 0 — design/spec)
```
Data: LibriSpeech + RAVDESS or IEMOCAP
Attributes: {emotion, speaker, noise_level}
Model: Whisper encoder (layers 0-31)
Per layer:
  - Train linear probe for emotion → get feature direction F_emotion
  - Perform interchange intervention: patch F_emotion from source → base
  - Compute Cause(layer) = % correct emotion predictions after patch
  - Compute Isolate(layer) = % correct speaker predictions after patch
  - Leakage(layer) = Cause - Isolate (high = entangled)
Output: 3×32 table (Cause/Isolate/Leakage per layer) + heatmap
```
This is entirely **Tier 0/1** — linear probes + activation patching, CPU, no GPU needed.

## Summary
RAVEL Cause = "did my patch work?" / Isolate = "did I avoid contaminating other things?"
Q040 adapts this to audio: emotion vs speaker vs noise as attributes in Whisper layers.
