## §4.7 Content Summary (3 subsections)

1. **Diagnostic Decision Tree (4 steps)**:
   - Step 1: Codec probe (SpeechTokenizer Layer-1 reconstruction, 2 min, CPU) — cheapest, no
     model access required. Confirms Tier 1 if contrast lost in reconstruction.
   - Step 2: Encoder gc(L) sweep (30 min, CPU, E1 infrastructure) — run DAS on audio encoder.
     max gc < 0.55 everywhere → Tier 1 confirmed. max gc ≥ 0.65 with isolated peak → proceed.
   - Step 3: Connector transfer test (5 min, CPU) — apply R_encoder to LLM layer 0 unretrained.
     IIA_transfer < 0.55 AND retrained IIA < 0.55 → Tier 2 confirmed. Otherwise → proceed.
   - Step 4: LLM gc(L) sweep (1 day, GPU/NDIF, E2) — full DAS sweep. Late-layer drop ≥ 0.10
     → Tier 3. No drop → No failure (grounded model).

2. **Operationalized Table (Table 4)**: 6 rows (Tier 1a/1b, 2a/2b, Tier 3, None) × columns
   (failure site, diagnostic test, passing threshold, time cost). Tier 1/2 diagnosis = CPU-only,
   ≤ 35 min total. Tier 3 requires GPU but only runs if Steps 1-3 pass.

3. **Three-point contribution framing**:
   - Transferability: Steps 1-3 require no GPU, replicable by any lab with NNsight.
   - Efficient triage: sequential design front-loads cheap tests, avoids unnecessary GPU runs.
   - Falsifiability: all thresholds (gc < 0.55, drop ≥ 0.10) derived from §3.8.1 bootstrap
     criterion — not ad hoc, defensible to reviewers.

## Key Design Choice

Threshold gc = 0.55 (not 0.60 or 0.50) chosen as: 0.50 = pure chance for binary variable;
0.55 = "substantially above chance, but not yet a strong signal" (consistent with bootstrap
CI lower bound > chance + 0.05 from §3.8.1). The late-layer drop threshold of 0.10 matches
the bootstrap CI non-overlapping criterion — two layers separated by > 0.10 typically have
non-overlapping 95% CIs under 1000 resamples (empirically validated by construction).
