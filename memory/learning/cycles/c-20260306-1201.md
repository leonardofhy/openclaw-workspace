## Problem
Audio SSL models (e.g., M2D/Masked Modeling Duo) have strong generalization empirically but no mechanistic account of *how* they internally encode diverse acoustic information. Prior MI work at the neuron level exists for LLMs (LAPE framework) but not audio SSL models.

## Method
- **AAPE (Audio Activation Probability Entropy)**: entropy-based metric adapted from LAPE (Lan et al. 2023, NLP) that quantifies how selectively a neuron activates across sound classes.
- AAPE_{l,n} = -∑_c P̃^(c)_{l,n} log P̃^(c)_{l,n} — low AAPE = more class-specific neuron
- Three-step filter: (1) remove bottom 5% low-activation neurons; (2) select lowest rAAPE percentile; (3) top 5% by class-specific activation probability
- **Model:** M2D (Masked Modeling Duo ViT-B, 12 layers × 768×4=3072 neurons = 36,864 total neurons)
- **Comparison baseline:** supervised ViT trained from scratch on AudioSet
- **Tasks:** ESC-50, GISE-51, VoxForge (6 languages), VoxCeleb1 (gender + nationality), CREMA-D (emotion + gender), GTZAN (music genres), Surge (octave/pitch), NSynth (instrument + octave)

## Key Results
1. **SSL ≈ 100% class coverage** for all tasks; SL = only 49% coverage for VoxCeleb1. SSL fosters ~2× more class-specific neurons than SL.
2. **Shared neurons across semantic categories**: gender neurons respond to CREMA-D gender AND VoxCeleb1 gender — same neuron IDs, different task datasets. Speech attribute neurons are reused across datasets.
3. **Musical pitch neurons**: shared across Surge and NSynth (both synthesizer note sources) — octave neurons generalize across instrument timbres.
4. **Causal validation**: steering identified neurons (activating/suppressing) measurably impacts classification accuracy, confirming functional role (not epiphenomenal).

## Connection to Leo's Tracks

### Track 2 (AudioSAEBench) — DIRECT RELEVANCE:
- AAPE = neuron-level selectivity metric; SAE features = dictionary-level selectivity metric. **Gap**: no paper uses SAE features instead of neurons for this coverage/selectivity analysis in audio.
- AudioSAEBench M4 (Isolate) complements AAPE: AAPE says "which neuron responds to class X", Isolate(F,A) says "when you patch feature F, does only attribute A change?" These are complementary measures of the same property.
- Cite in Paper B §2 as: "Kawamura et al. [2602.15307] show neuron-level class specificity in SSL audio models; we extend this to dictionary-level SAE feature specificity with causal validation."

### Track 3 (Paper A / Listen vs Guess) — INDIRECT:

## Open Questions (for Paper B)
1. AAPE works at neuron level; does it work at SAE feature level? SAE features > neurons in specificity — do they also achieve > 100% coverage (multiple features per class)?
2. Isolation: AAPE measures Cause (does neuron activate for class X?) but not Isolate (does neuron NOT activate for class Y?). AudioSAEBench M4 adds the Isolate dimension — this is the AudioSAEBench advance over AAPE.
3. M2D is encoder-only (SSL). Do audio-LLMs (encoder + LLM) have the same class-specific neuron structure in their encoder vs LLM portions?

## Gap Assessment
- **Not a new gap** — confirms existing intuition about neuron specificity in audio SSL models.
- **Confirmation value**: SSLs have class-specific neurons with causal function → baseline neuron-level MI is established. AudioSAEBench takes it further with SAE features + isolation metric.
- **Cite in Paper B §2.2** as prior neuron-level MI for audio models.
