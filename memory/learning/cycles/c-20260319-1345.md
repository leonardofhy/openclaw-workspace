## Paper: Linearity of Relation Decoding in Transformer LMs (Hernandez et al., 2023)
arXiv:2308.09124v2

---

## Problem
How does a transformer decode relational knowledge (e.g., "Paris is the capital of France")?
Hypothesis: there exists a simple **linear approximation** to the LM's computation at the subject token layer.

## Method: LRE (Linear Relational Embedding)
1. For subject token s at layer ℓ, compute first-order (Taylor) approximation:
   - `f(s) ≈ J_f(s) · h_s + b` where `J_f = ∂f/∂h_s` (Jacobian wrt subject hidden state)
2. J is computed via autograd at a single prompt
3. **SVD of J**: `J = UΣV^T`
   - `v_1` (top right singular vector): most important **input** direction (query)
   - `u_1` (top left singular vector): most important **output** direction (answer)
   - Σ_{1}/Σ_{2} ratio: effective rank (how dominated by one direction)
4. **Causality test**: patch subject representation along v_1 → does it change the decoded object?

## Results
- Many factual + linguistic relations have low-rank Jacobians (1-2 dominant singular values)
- Causal test passes: patching along v_1 changes model output correctly
- NOT all relations are linear; heterogeneous deployment

### Q122: Incrimination features × Jacobian SVD
**Hypothesis:** Top-incriminated SAE features (high blame for WER error) should have cosine similarity > 0.7 with v_1 (top right singular vector of J at the blame layer).

**Rationale:** Incriminated features are those causally responsible for computation errors. In the LRE framework, v_1 is the dominant causal input direction. If the same features that "break" the computation are the ones most aligned with v_1, it means:
- The LRE approximation identifies the same features as the SAE incrimination
- Both methods triangulate the same causal mechanism

### Q127: Power Steering × AND/OR gates
**Hypothesis:** At gc(k) peak layer, AND-gate features (h_and ⊂ ℝ^d_model) span a subspace more aligned with top Jacobian singular vector than OR-gate features.

**Rationale:** AND-gates require ALL inputs active → bottleneck features → dominant linear direction. OR-gates are redundant → distributed across multiple weaker directions.

**Mock implementation plan:**
```python
# 1. At gc peak layer, split features into AND / OR sets
and_features = W_dec[and_gate_mask]  # (n_and, d_model)
or_features = W_dec[or_gate_mask]   # (n_or, d_model)

# 2. Top singular vector of J at gc peak layer
v1 = svd(J_at_gc_peak)[2][0]

# 3. Mean cosine sim per gate type
cos_and = np.abs(and_features @ v1).mean()
cos_or  = np.abs(or_features @ v1).mean()

# Hypothesis: cos_and > cos_or
```
