## Connection to Q116 — Mechanistic Framing

**Our hypothesis**: Backdoor = forced premature audio→text collapse in gc(k) dynamics

### Why t* shifts leftward with backdoor audio
1. Backdoor trigger acoustically resembles a short distinctive sound → activates shortcut OR-gate features early
2. These OR-gate features fire at shallow layers → decoder switches to text-prediction mode before normal gc peak
3. Result: t* (collapse onset step) < 3 for poisoned audio vs t* > 6 for clean audio
4. AND-gate fraction at gc peak drops sharply (audio-dependent features never fully engaged)

### Detection protocol (Q116 mock design)
```
For each utterance:
1. Run temporal gc(k) probe → find t* = argmin(Isolate_in)
2. Run AND-gate audit at t* layer → compute f_AND
3. Alarm = (t* < θ_t) AND (f_AND < θ_AND)
```
- θ_t ≈ 4 (trigger fires within first 4 decoder steps)
- θ_AND ≈ 0.3 (less than 30% AND-gate features at collapse)

### Differentiation from VAD defense
- VAD fails on imperceptible/embedded triggers (e.g., low-amplitude env sounds)
- Our method is trigger-agnostic: we detect the internal cascade effect, not the trigger itself
- Works even if trigger is inaudible to humans

### Connection to Cascade Equivalence Hypothesis (arXiv:2602.17598, last cycle)
- Clean audio: cascade degree low at gc peak (AND-gate dominant = audio-driven)
- Poisoned audio: cascade degree high early (OR-gate shortcuts = text-driven early)
- **Backdoor = cascade induction**: trigger forces high cascade degree (= text shortcut) before gc peak
