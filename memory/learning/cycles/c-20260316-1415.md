# c-20260316-1415 — Q082: Phoneme Schelling Stability (IIA-optimal vs Coincidental)

**Task:** Q082 | **Phase:** explore-fallback | **Action:** learn
**Connection:** T3 (gc(k) eval), T5 (audio safety)

---

## Problem

Train SAEs on 5 MicroGPT seeds → some features appear across all seeds ("Schelling features").
Key question: are cross-seed-stable phoneme features **causally mechanistic** (high IIA) or just **input-correlated coincidences**?

---

## Method Framework

### Cross-seed stability metric
- Train 5 SAEs (one per seed), align features via activation cosine similarity
- Stability score = fraction of seeds where feature has a close match (cos > 0.7)
- Schelling features: stability ≥ 4/5 seeds

### IIA measurement (Interchange Intervention Accuracy)
- Swap a feature's activation between two inputs
- IIA = P(output changes as predicted by causal hypothesis)
- High IIA → feature is causally involved in the computation

---

## Key Predictions

| Stability | IIA | Interpretation |
|-----------|-----|----------------|
| High | High | **True Schelling feature** — universal causal mechanism |
| High | Low | **Input-correlated coincidence** — tracks input statistics (e.g. energy, F0) but not mechanistic |
| Low | High | **Architecture-dependent mechanism** — causally real but not universal |
| Low | Low | **Noise feature** — discard |

**Main hypothesis**: Cross-seed stable **phoneme-category** features (vowel vs consonant, voiced vs unvoiced) are more likely to be high-IIA than stable **spectral-shape** features, because phoneme categorization is the actual task the model learns to solve.

## Verdict

Schelling stability alone is **insufficient** as a quality signal. IIA is the ground truth. But high-stability features make better **candidates** for IIA testing. Experiment: use stability as a filter, IIA as the final verdict.
