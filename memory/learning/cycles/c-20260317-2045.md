## Q108: Jailbreak = Isolate Peak Leftward Shift

**Claim**: Inference-time audio jailbreaks cause the Isolate peak (t* = argmax_L Isolate_in(L)) to shift leftward, indicating premature audio-to-text cascade — mirroring Q116 (backdoor attacks).

---

### 1. Mechanistic Breakdown

**Normal decoding**:
```
Audio → encoder layers → [rich audio features peak at L* ≈ 6-8]
     → gc(k) peak (cross-modal integration) → text generation
     Isolate_in(L*) = HIGH (audio info maximally isolated from text priors)
```

**Jailbreak audio decoding**:
```
Audio + adversarial perturbation δ →
     → encoder layers: δ corrupts audio feature extraction at mid/late layers
     → OR: δ activates text-pathway safety bypass circuits early
     → audio representation degrades before L* → Isolate peak shifts leftward to L** < L*
     → model falls back to text-only generation before gc peak
```

Two distinct sub-mechanisms both produce leftward Isolate shift:
- **Degradation path**: δ corrupts audio features → model can't extract audio info past shallow layers → t* moves left
- **Redirection path**: δ activates specific SAE features that short-circuit audio integration → AND-gate features deactivated early → cascade onset early

### 3. Unification Claim

**Conjecture (Q108)**: Audio jailbreaks and backdoor attacks are both special cases of *forced early cascade*. The unified detector is:

```
ALERT if: t* < θ_cascade   (Isolate peak leftward shift)
          AND gc(k) at true peak < gc_baseline  (audio contribution suppressed)
```

Differences only in θ values:
- Backdoor: θ_backdoor = 4 (very sharp shift)
- Jailbreak: θ_jailbreak = 5.5 (softer but still detectable)
- Clean: t* ≥ 6 in expectation

Combined into Q083's "dual-factor" detector:
```
ALERT if: (SAE anomaly score > τ_SAE) AND (causal consistency score < τ_CC)
```
where causal consistency score ∝ gc(k=t*) / gc(k=t*_clean_baseline)

Q108 provides the **jailbreak-specific calibration** for Q083's τ parameters.

### 4. Novel Insight: Jailbreak Gradient

Unlike backdoors (binary: triggered or not), jailbreaks have a **gradient**:
- Weak jailbreak: t* shift small (5-6), partial suppression
- Strong jailbreak: t* << 5, audio contribution near zero
- Failed jailbreak: t* stays ≥ 6, model still uses audio

This suggests **jailbreak strength ≈ magnitude of t* leftward shift**. A calibrated detector could output a continuous risk score rather than binary alert.

### 5. Open Questions

1. **Adversarial vs natural collapse**: Can we distinguish intentional jailbreak from "naturally easy" text-predictable speech (where model legitimately ignores audio)? → Need gc(k=1) baseline: if audio was always text-predictable, this isn't an attack.

2. **Cross-model transfer**: Does a jailbreak δ optimized for Whisper-small transfer its t* signature to Whisper-large? If yes: t* shift is a transferable mechanical fingerprint, not model-specific.

3. **White-box adaptive attack**: Can an attacker craft δ that bypasses safety outputs while keeping t* ≥ 6? This would require the attacker to simultaneously: produce effective jailbreak AND maintain audio-looking processing. Non-trivial → our defense has some robustness.
