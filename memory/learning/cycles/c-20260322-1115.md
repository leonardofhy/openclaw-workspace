## Hypothesis

Speaker-specific features in Whisper's decoder are **AND-gate** (require audio input);  
speaker-agnostic phoneme-invariant features are **OR-gate** (text-predictable).

## Mechanistic Argument

1. **Speaker-ID features** encode voice timbre, pitch, spectral envelope.  
   This information is *absent* from the text stream → the LM prior cannot activate them.  
   → Audio is a **necessary** condition → AND-gate mandatory.

2. **Speaker-agnostic features** (e.g., /AE/ phoneme identity) are predictable from  
   surrounding word context via the language model prior.  
   → Audio is *helpful but not required* → OR-gate.

3. **Connection to gc(k):** gc(k) = E[AND-frac(k)] over the corpus. At layer k*  
   (gc peak), speaker-ID features should show AND-frac ≫ overall AND-frac,  
   since audio is maximally gate-controlling at that layer.

## Real Experiment Protocol (CPU-only)

1. LibriSpeech dev-clean: extract /AE/ tokens from ≥8 speakers
2. Whisper-base decoder: capture h[k*] at gc peak layer (~layer 20)
3. Train linear speaker-ID probe on h[k*] → top-50 features = speaker-ID
4. Causal patching: patch audio→Gaussian; patch text→uniform prior
5. Compute AND-frac per feature group (t-test across features)
6. Expected: Δ AND-frac > 0.2, p < 0.01

- **§5 (applications):** AND-frac as speaker diarisation signal? High AND-frac at k*  
  → speaker boundary detection without explicit diarisation head
