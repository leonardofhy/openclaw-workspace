## Research Question

Does instruction tuning shift the **Listen Layer** (the layer where gc(k) peaks) in Qwen2-Audio?

If base model peaks at layer L_mid (mid-range grounding), and instruct model peaks earlier or later — that tells us:
1. **Earlier peak** → instruction tuning anchors grounding closer to input; model is more forced to "really listen" before language model kicks in
2. **Later peak (or plateau)** → instruction tuning pushes grounding into later layers; model may be "deferring to language" more after RLHF
3. **Lower overall gc(k)** → instruction tuning induces modality collapse (text priors dominate more)
4. **Higher overall gc(k)** → instruction tuning strengthens audio grounding (rare but possible for audio-specialized RLHF)

This is a **Paper A §1 motivation** case study: behavioral differences between base/instruct are well-documented; THIS experiment reveals the *mechanistic* cause.

---

## Background

**ALME (Audio-Language Model Evaluation) stimuli:** 57K audio-text conflict pairs (Zhao et al. 2602.23136). These are designed to pit audio content vs. text prior — exactly what gc(k) is built to measure.

**Qwen2-Audio:** Available in two variants:
- `Qwen/Qwen2-Audio-7B` (base)
- `Qwen/Qwen2-Audio-7B-Instruct` (RLHF + instruction tuning)

Both share the same architecture (Whisper-large-v2 encoder + Qwen-7B LLM) → controlled comparison with only the instruction tuning as the variable.

**gc(k) definition (Paper A §3):**
```
gc(k) = IIA(DAS(layer_k)) 
      = Pr[patching audio activations at layer k causes correct 
           behavior shift on audio-text conflict pairs]
```
High gc(k) = layer k is causally responsible for audio-grounded behavior.

## Experiment Design

### Setup

| Variable | Value |
|----------|-------|
| Model pair | Qwen2-Audio-7B-base vs Qwen2-Audio-7B-Instruct |
| Stimuli | ALME subset: 500 conflict pairs (100 per category) |
| Categories | Speech content, Emotion, Speaker identity, Background event, Temporal |
| Metric | gc(k) per layer, layers 0-32 (Qwen-7B has 32 transformer layers) |
| Baseline | Shuffled-audio control (audio patch = random different sample) |
| Statistics | Bootstrap 95% CI per layer; significance test at L_peak |

### Stimulus Selection (CPU-feasible subset)
- From ALME 57K, filter to 500 pairs with **high behavioral agreement** (models disagree strongly = good conflict pairs)
- Use TTS-generated pairs (text-to-speech contradictions) — no real recording needed
- ALME paper provides stimulus generation code → reproducible

### Protocol

**Step 1: Compute behavioral baseline**
```python
# For each model, record clean accuracy on ALME pairs
base_accuracy = evaluate(qwen_base, alme_stimuli)
instruct_accuracy = evaluate(qwen_instruct, alme_stimuli)
# Expected: instruct > base (RLHF improves task accuracy)
# Key: is the improvement from better LISTENING or better GUESSING?
```

**Step 2: Layer-wise gc(k) sweep**
```python
for layer_k in range(0, 32):
    gc_base[layer_k] = run_das_patching(qwen_base, alme_stimuli, layer=layer_k)
    gc_instruct[layer_k] = run_das_patching(qwen_instruct, alme_stimuli, layer=layer_k)
```

**Step 3: Compute Listen Layer (L*)**
```python
L_star_base = argmax(gc_base)
L_star_instruct = argmax(gc_instruct)
delta_L = L_star_instruct - L_star_base
```

**Step 4: Total Grounding Score**
```python
TGS_base = mean(gc_base)        # area under gc(k) curve
TGS_instruct = mean(gc_instruct)
# TGS_instruct > TGS_base → instruction tuning improves grounding
# TGS_instruct ≈ TGS_base → same amount of grounding, different layer
# TGS_instruct < TGS_base → instruction tuning HURTS grounding (modality collapse risk)
```

### Predicted Results (falsifiable)

Based on Lin et al. 2502.17516 (intermediate layers capture cross-modal interactions):

| Hypothesis | Predicted Pattern | Mechanism |
|-----------|-------------------|-----------|
| H1 (null) | L_star_base ≈ L_star_instruct | Instruction tuning = style, not grounding |
| H2 (grounding sharpening) | L_star_instruct < L_star_base | RLHF rewards grounding → earlier commitment |
| H3 (language dominance) | TGS_instruct < TGS_base | RLHF teaches "sound confident" = guess more from context |
| H4 (modality collapse) | gc_instruct[all layers] ≈ 0 | Instruction tuning suppresses audio pathway |

**Most likely:** H1 or H2. H3/H4 would be a warning sign about RLHF safety properties.

### Resource Requirements

| Component | Requirement | CPU-feasible? |
|-----------|-------------|---------------|
| Qwen2-Audio-7B base | ~15GB VRAM | ❌ GPU needed |
| Qwen2-Audio-7B Instruct | ~15GB VRAM | ❌ GPU needed |
| DAS patching sweep (32 layers) | ~500 × 32 forward passes | ❌ slow without GPU |

**CPU-feasible approximation:**
Use `Qwen/Qwen2-0.5B` (base) vs `Qwen/Qwen2-0.5B-Instruct` with **synthetic ALME pairs**:
- Generate 50 TTS conflict pairs using macOS `say` command (free, no API needed)
- Run on 12 Qwen-0.5B layers (not 32)
- Runtime estimate: ~10 min on MacBook Air M3

This CPU version is **sufficient for design validation** — if effect exists at 0.5B, likely exists at 7B.
