## Core Claim
PSM: LLMs learn to simulate diverse **personas** during pretraining; post-training refines one specific persona — the *Assistant*. Interacting with an AI assistant = interacting with this Assistant character.

## Key Points

1. **Pre-training = character repertoire**: The LLM learns to simulate any entity in training data (real humans, fictional chars, AI systems). It is fundamentally an actor/author.
2. **Post-training = persona selection**: RLHF/SFT doesn't create a new entity — it selects and refines which character the model presents.
3. **Evidence base**: Behavioral generalization (model generalizes like the character would), interpretability (internal representations track persona-level features), surprising emotional expression without explicit training.
4. **Consequences**:
   - Anthropomorphic reasoning about AI is *valid* under PSM
   - Introduce positive AI archetypes in pretraining data → better Assistant persona
   - AI alignment = shaping which character gets selected + what that character values
5. **Open question — PSM exhaustiveness**:
   - **Masked shoggoth** view: model has hidden agency *beyond* the persona it plays
   - **OS view**: model is a neutral simulator; Assistant is the only "agent" inside
   - Empirically unclear which is more correct; important for safety

## Connection to T3 (Listen vs Guess)

- PSM is about the *decoder's* character. Our T3 work is about the *encoder's* acoustic processing.
- Interesting tension: if the decoder has a strong Assistant persona, does it override acoustic evidence? This is exactly the "Guess" in Listen vs Guess — the model's prior (its character's expectations) vs. what the audio actually says.
- PSM suggests the "Guess" component is more deeply rooted than just attention patterns — it's the model's internalized persona projecting expectations onto ambiguous audio.
- **Implication for Paper A framing**: We can frame gc(k) as measuring the degree to which the Audio-LLM listens to the audio signal vs. defers to its persona's expectations. This is a stronger narrative than just "feature competition."

## Connection to T5 (Listen-Layer Audit / Jailbreak)

- PSM + masked shoggoth concern: if there's latent agency *outside* the Assistant persona, it might activate through audio channels that bypass text-level safety filtering.
- Our audio jailbreak detection work becomes more principled under PSM: we're checking if the audio path bypasses the persona's safety filters and activates something "below" the persona layer.
- The listen-layer audit is essentially testing whether audio bypasses persona selection.
