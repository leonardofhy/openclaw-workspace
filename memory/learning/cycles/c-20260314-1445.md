## RAVEL Paper — Key Formalism (arXiv:2402.17700, ACL 2024)

**Problem:** Neurons are polysemantic → need to evaluate interpretability methods on *disentangling* attributes.

**RAVEL benchmark:** Given entity E with attributes A={A1,...,Ak}, find feature F that encodes A_i *separately from* all other attributes.

**Two core metrics (both use interchange interventions / activation patching):**

1. **Cause(A, F, M, D)** = P[patching F with source x' changes output from A_E to A_E']  
   → Measures: does F *control* attribute A? (faithfulness)

2. **Isolate(A, F, M, D)** = P[patching F preserves other attributes A'\A]  
   → Measures: does F *exclusively* encode A? (no bleed to other attributes)

A good interpretability method maximizes BOTH: high Cause + high Isolate.

**MDAS** (Multi-task DAS) = best performer; finds distributed representations satisfying multiple causal criteria.

## Connection to gc(k) — Q107 Mechanism

**Core hypothesis for Q107:**  
Per-layer RAVEL-style Isolate curve has a minimum at the gc peak layer.

**Intuition chain:**
- gc(k) = fraction of causal influence on next-token prediction that comes from audio vs text-prior at encoder layer k
- When gc(k) is HIGH → audio features are dominant, actively used, highly disentangled from text-prior features
- At gc peak: "audio content" attribute is maximally separated from "text-predictability" attribute
- RAVEL Isolate for "audio content as attribute" = low bleed into other (text-driven) dimensions → Isolate minimum
- Therefore: **argmin_k(Isolate(k)) ≈ argmax_k(gc(k))**

**Why this matters (compute savings):**
- Full gc(k) patching requires: two forward passes + activation hook + causal ablation → expensive
- RAVEL-style Isolate needs: probe or SAE feature test per layer, no full patching
- If proxy holds → we can localize gc peak cheaply via Isolate curve
