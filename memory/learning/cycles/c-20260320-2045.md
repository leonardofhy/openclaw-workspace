## Problem

Q132 claims: *low-predictability utterances (high LM perplexity) should have higher gc(k) peak — i.e., the model relies more on audio when text context gives less certainty.*

How does this follow mechanistically, and what's the theoretical grounding?

---

## Synthesis: Predictive Coding Frame

**Core theory:** Whisper (and any autoregressive ASR) is doing implicit predictive coding. At each decoder step t, it balances:
- **Top-down prior**: LM distribution P(y_t | y_{1..t-1}) — what's linguistically predictable
- **Bottom-up sensory signal**: Audio cross-attention → gc(k) gate

**Key insight (Information-theoretic):**  
gc(k) ∝ mutual information needed between audio and output, given current LM prior.

When H(y_t | y_{1..t-1}) is HIGH (high perplexity, unpredictable token):
- LM prior is flat → posterior must come from audio
- Audio cross-attention gates MUST open wider (AND-gate needed)
- Prediction: gc_peak is HIGHER for high-PPL tokens

When H(y_t | y_{1..t-1}) is LOW (low perplexity, predictable token):
- LM prior is sharply peaked → audio provides little additional info
- Cross-attention can be ignored (OR-gate suffices)
- Prediction: gc_peak is LOWER for low-PPL tokens

**Formula (mock):**
```
gc_peak(t) ≈ α · H_LM(y_t | context) + ε
```
- α > 0 (positive slope)
- H_LM measurable via: P_LM(y_t | prefix) from any language model (GPT-2, n-gram, Whisper's own LM logits)

## Connections to Existing Theory

- **FAD bias (Q123):** High-FAD phonemes = text-predictable = low PPL → OR-gate (consistent)
- **AND→OR transition (Q134):** Silence/noise causes PPL spike (uniform over vocab) → AND-gate required → explains hallucination onset as gc failure under high PPL + bad audio
- **Beam rescoring (Q133):** Isolate(k) at gc peak = selectivity of audio signal; PPL could predict WHERE Isolate matters most
- **Stressed syllables (Q127):** Stress disambiguation = locally high PPL (which of two readings?) → AND-gate enrichment

## Mock Protocol for Q132 Build

```python
# perplexity_gc_mock.py skeleton
import numpy as np

def mock_lm_perplexity(phoneme_list):
    """
    Mock: assign perplexity by phoneme category.
    - Vowels in common words: PPL = 2-5 (predictable)
    - Consonant clusters: PPL = 8-15 (less predictable)
    - Rare phonemes: PPL = 20-50 (high uncertainty)
    """
    ppl_map = {'AA': 3.2, 'AH': 2.8, 'T': 6.5, 'K': 9.1, 'ZH': 22.4}
    return [ppl_map.get(p, 12.0) for p in phoneme_list]

def mock_gc_peak(perplexities, alpha=0.6, noise_std=0.05):
    """gc_peak = alpha * log(PPL) + noise"""
    return [alpha * np.log(p) + np.random.normal(0, noise_std) for p in perplexities]

phonemes = ['AA', 'T', 'ZH', 'AH', 'K']
ppl = mock_lm_perplexity(phonemes)
gc = mock_gc_peak(ppl)
r = np.corrcoef(ppl, gc)[0,1]
# Expected: r > 0.6 (by construction, but validates pipeline)
```

**Validation target:** Pearson r(log_PPL, gc_peak) > 0.6 on mock data.  
**Extension:** Real PPL from GPT-2 on transcript prefixes (CPU, fast).
