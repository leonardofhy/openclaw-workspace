## Paper: RAVEL (ACL 2024)
**Full title**: "Evaluating Interpretability Methods on Disentangling Language Model Representations"
**Authors**: Huang, Wu, Potts, Geva, Geiger (Stanford / Tel Aviv / Pr(Ai)2R)
**Link**: arXiv:2402.17700

## Problem
Neurons are polysemantic. Interpretability methods claim to disentangle them — but until RAVEL, no apples-to-apples benchmark existed.

## Method: Two metrics (the key contribution)
1. **Cause(F, A)**: Intervening on feature F changes attribute A's output → F causally encodes A
2. **Isolate(F, A)**: Same intervention does NOT change other attributes B ≠ A → F is disentangled / isolated

Classic activation patching gets Cause; RAVEL adds Isolate as the harder, more meaningful bar.

**Benchmark structure**: 5 entity types (city, Nobel laureate, verb, physical object, occupation), each with 4-6 attributes, 500+ instances, 50-150 prompt templates.

## Results
- **MDAS (Multi-task DAS)** = best. Simultaneously satisfies multiple causal criteria (one featurizer for all attributes of an entity type → forces disentanglement).
- SAE > probes/PCA on Isolate, but MDAS >> SAE on Isolate. 
- Most methods get OK Cause but fail Isolate → they find causally relevant features that are NOT isolated.

## Connection to gc(k) / Schelling stability

### Q099 Hypothesis (sharpened):
**Schelling-stable SAE features (cross-seed stable) should score higher on RAVEL Isolate than non-stable features.**

Reasoning:
- Schelling features = "canonical" features the model converges to regardless of random seed
- If a feature is canonical, it likely encodes ONE clean concept (monosemantic = isolated)
- Non-stable features may be polysemantic splittings that differ per seed = entangled

### AND-gate extension (new idea):
- AND-gate features (audio ∩ language both required) might be MORE isolated than OR-gate features
- OR-gate features may "bleed" across modalities in RAVEL-style interventions
- This could be tested: define audio-version of RAVEL attributes (phoneme identity, speaker accent, word semantics) and measure Cause/Isolate for AND vs OR gate features
