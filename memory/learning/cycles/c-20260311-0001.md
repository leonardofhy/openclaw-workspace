## Formal Definition: ΔGS — Grounding Sensitivity Contrast Metric (M7)

### Motivation

gc(F) (Category 5, M5) measures the absolute grounding coefficient of a single SAE feature F.
But for *benchmark comparison purposes*, absolute gc(F) values depend on the specific ALME
stimulus set distribution. A more robust metric is the **contrast** between a feature's
grounding in audio-consistent vs audio-conflicting stimulus conditions.

ΔGS is designed to answer:
> "How much does feature F *change its activation* when audio and text are in conflict,
> versus when they are consistent?"

A feature that is truly audio-grounded should activate strongly to audio content regardless
of text conflict. A feature that guesses from text context should show *high sensitivity*
to conflict — it fires when text and audio agree, but gets confused when they disagree.

---

### Formal Definition

Given:
- F: an SAE feature (activation vector at layer L for a given stimulus)
- S_c = {s_1, ..., s_N}: N audio-text **consistent** stimuli (audio matches transcript)
- S_x = {s_1', ..., s_N'}: N audio-text **conflict** stimuli (audio contradicts transcript context)
- f(s) ∈ ℝ: activation of feature F on stimulus s (scalar, post-ReLU or post-TopK)
- µ_c = mean(f(s) for s in S_c): mean activation under consistent conditions
- µ_x = mean(f(s) for s in S_x): mean activation under conflict conditions
- σ_c = std(f(s) for s in S_c): std under consistent conditions
- σ_x = std(f(s) for s in S_x): std under conflict conditions

**ΔGS(F) — Grounding Sensitivity Contrast Score:**

```
ΔGS(F) = (µ_c - µ_x) / (σ_pool + ε)
```

where:
```
σ_pool = sqrt((σ_c² + σ_x²) / 2)   [pooled std, Cohen's d denominator]
ε = 1e-8                             [numerical stability]
```

This is the **Cohen's d** of feature F's activation contrast between consistent and conflict conditions.

**Interpretation:**

| ΔGS(F) | Interpretation |
|--------|----------------|
| > 0    | Feature more active on consistent stimuli → potentially TEXT-GROUNDED (relies on context match) |
| ≈ 0    | Feature insensitive to audio-text conflict → potentially AUDIO-GROUNDED (truly listens) |
| < 0    | Feature more active on conflict stimuli → ANOMALOUS (investigate; possibly error-detection feature) |

**Note on direction convention:**
- Large positive ΔGS → feature depends on text-audio consistency → text-grounded
- Near-zero ΔGS → feature is robust to audio-text conflict → audio-grounded
- This is COMPLEMENTARY to gc(F):
  - gc(F) measures how much audio content causally drives the feature (via patching)
  - ΔGS(F) measures how sensitive the feature is to audio-text alignment (no patching needed)

---

### Population-level Metric: GS_profile

For a given SAE and layer, compute ΔGS(F) for all N_F features:

```
GS_profile(layer) = {ΔGS(F) : F ∈ SAE_features(layer)}
```

Summary statistics (for benchmark tables):

```python
# Core benchmark metrics from GS_profile:
GS_mean   = mean(|ΔGS(F)| for F in features)  # overall sensitivity magnitude
GS_audio  = fraction of features with |ΔGS(F)| < 0.1   # "deaf to conflict" = audio-grounded
GS_text   = fraction of features with ΔGS(F) > 0.5      # "conflict-sensitive" = text-grounded
GS_anom   = fraction of features with ΔGS(F) < -0.3     # anomalous (conflict-preferring)
```

**Benchmark comparison claim (predicted):**
- Whisper encoder features: GS_audio ≥ 0.70 (encoder is audio-grounded)
- Qwen2-Audio LLM backbone features: GS_text ≥ 0.50 (LLM features rely on text context)
- Transition at Listen Layer L* (Paper A): GS_audio drops sharply at L* → cross-paper validation

### CPU-Feasibility Plan

ΔGS computation requires NO activation patching — only two forward passes per stimulus:
1. Forward pass on consistent stimulus s → record f(s)
2. Forward pass on conflict stimulus s' → record f(s')

Then compute ΔGS(F) per feature from the activation distributions.

**Compute estimate (Whisper-small, LibriSpeech dev subset):**
- N = 200 stimulus pairs (100 consistent + 100 conflict)
- Whisper-small encoder: 12 layers × ~512 features (if TopK SAE, k=32)
- Forward passes: 200 × 2 = 400 (CPU, ~0.3s each) = ~120s total
- Feature activation extraction: ~10s
- ΔGS computation: ~1s (NumPy)
- **Total: ~2 minutes on MacBook Air M2** ✅ Tier 1 (auto-allowed)

For ALME 57K stimuli (full benchmark): ~10h on CPU, or 30min on GPU.
For Paper B submission: 200-pair subset is sufficient for the correlation with gc(F) claim.

### Connection to gc_eval.py

ΔGS can be computed using the existing `gc_eval.py` infrastructure with minor extension:

```python
# Pseudo-code extension to gc_eval.py:

def compute_delta_gs_feature(
    sae,              # SAE model (post-training)
    model,            # Whisper encoder
    consistent_set,   # list of (audio, transcript) pairs
    conflict_set,     # list of (audio, mismatched_transcript) pairs
    layer: int,
    feature_idx: int,
) -> float:
    """Compute ΔGS(F) for a single SAE feature F at a given layer."""
    acts_c = [extract_sae_feature(sae, model, s, layer, feature_idx) for s in consistent_set]
    acts_x = [extract_sae_feature(sae, model, s, layer, feature_idx) for s in conflict_set]
    mu_c, mu_x = np.mean(acts_c), np.mean(acts_x)
    sigma_pool = np.sqrt((np.std(acts_c)**2 + np.std(acts_x)**2) / 2)
    return (mu_c - mu_x) / (sigma_pool + 1e-8)  # Cohen's d

def compute_gs_profile(sae, model, consistent_set, conflict_set, layer) -> dict:
    """Compute GS_profile statistics for a full SAE layer."""
    n_features = sae.W_dec.shape[0]
    delta_gs = [compute_delta_gs_feature(..., feature_idx=f) for f in range(n_features)]
    return {
        "GS_mean": np.mean(np.abs(delta_gs)),
        "GS_audio": np.mean(np.abs(delta_gs) < 0.1),   # fraction: audio-grounded
        "GS_text":  np.mean(np.array(delta_gs) > 0.5), # fraction: text-grounded
        "GS_anom":  np.mean(np.array(delta_gs) < -0.3),# fraction: anomalous
        "delta_gs_all": delta_gs,
    }
```

### Cross-Metric Validation

ΔGS and gc(F) are COMPLEMENTARY and should be correlated (this is a falsifiable prediction):

| Prediction | Direction | Evidence type |
|------------|-----------|--------------|
| Audio-grounded features (gc(F) ≈ 1.0) have ΔGS ≈ 0 | Negative | Pearson r(gc(F), |ΔGS(F)|) < -0.4 |
| Text-grounded features (gc(F) ≈ 0.0) have ΔGS > 0.5 | Positive | Rank correlation |
| Transition at L* (Paper A Listen Layer) in GS_audio | Cross-paper | Visual breakpoint |

If Pearson r(gc(F), |ΔGS(F)|) > 0 at the Listen Layer L*, that would indicate a measurement
confound — both metrics agree in the same direction, which might mean both are measuring the
same confounding variable (e.g., feature sparsity or activation magnitude).

**M7: ΔGS — Grounding Sensitivity Contrast Score (secondary)**

While gc(F) (M5) requires activation patching, ΔGS(F) provides a patching-free

ΔGS(F) = Cohen's d of feature activation between consistent and conflict ALME stimuli

A feature with ΔGS ≈ 0 is robust to audio-text conflict → audio-grounded.
A feature with ΔGS > 0.5 is sensitive to conflict → text-grounded.
