## Problem
How to find steering vectors efficiently without expensive per-concept data collection (CAA) or non-linear optimization (MELBO)?

## Method
**Layer-to-Layer Jacobian SVD via Power Iteration**

J(s→t) = Jacobian mapping source layer s activations → target layer t activations.
- Right singular vectors of J = directions at source layer that produce largest response at target layer
- These are the local linear analog of MELBO's optimization objective
- Power iteration: 5 iters × ~3 passes per matvec = ~15 forward passes per layer pair
- Block power iteration → top-k SVs via Gram-Schmidt re-orthogonalization; Rayleigh-Ritz correction for individual SVs
- For Qwen3-8B: full model sensitivity map (all source/target pairs, top-12 SVs each) = ~30 min on 8×A100

**Sensitivity Map:**
- Compute top SVs + KL divergence (steered vs baseline logits) for every (source, target) pair
- KL heatmap reveals which pairs have behaviorally relevant directions
- Norm scaling: scale steering vector to 0.35× residual stream norm (MLP output norms vary 50× across layers)

## Results
1. **Decision forks work best**: Prompts where model balances competing behaviors (refuse/comply, corrigibility/self-preservation) — steering tips the balance
2. **Latent behavior**: Unlock capabilities present in weights but unused (e.g., chain-of-thought in base model)
3. **Layer structure**: Mid-early to mid-late = most KL impact. Final third of layers: less effective as source. Early layers: incoherent generations.
4. **Sign sensitivity**: SVs found up to sign; check both signs when using
5. **Parity with MELBO**: Linear approx (Power Steering) ≈ nonlinear (MELBO) performance, at ~10× lower cost
