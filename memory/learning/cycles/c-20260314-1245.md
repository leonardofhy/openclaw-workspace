## Problem
Connect FAD (Frequency-Acoustic-Distribution) bias to the AND/OR gate framework at gc(k) peak. Do text-predictable phonemes systematically use OR-gate mechanisms, while audio-dependent phonemes use AND-gate mechanisms?

## Framework

**FAD bias** = how strongly a phoneme/token can be predicted from text context alone (high FAD = text-predictable, low FAD = audio-dependent).

**AND/OR gate at gc(k) peak:**
- OR-gate feature: fires from EITHER audio OR language prior → works even without audio
- AND-gate feature: fires only when BOTH signals active → requires audio to be present and processed

## Key Predictions

**P1 (phoneme → gate type):**
- High FAD phonemes → OR-gate activation at gc peak (language prior sufficient)
- Low FAD phonemes → AND-gate activation at gc peak (acoustic signal required)

**P2 (gc peak layer shift):**
- Text-predictable phonemes: gc(k) peak at LATER layers (audio only consulted as correction signal)
- Audio-dependent phonemes: gc(k) peak at EARLIER layers (acoustic must dominate from start)

**P3 (FAD as stratification variable for Paper A):**
- All gc(k) experiments stratified by FAD score → shows framework generalizes across phoneme types
- `gc(k) peak layer` ∝ `FAD score` (continuous predictor)
- AND-gate feature prevalence at peak ∝ `1 - FAD score`

## Operationalization (Tier 0 / Tier 1)

FAD proxy: BPE token frequency (easy, no new data needed)
- High BPE frequency → high FAD (common function words, predictable tokens)
- Low BPE frequency → low FAD (rare/acoustic-demanding tokens)

Binary partition: above/below median frequency → two groups → compute gc(k) peak per group.
