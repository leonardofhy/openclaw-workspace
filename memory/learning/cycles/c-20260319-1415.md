## Q117: Three-Metric Unification (cascade_degree = 1-AND_fraction = 1-GSAE_density)

### Proof sketch

**AND_fraction ↔ GSAE_density**
AND-gate features require co-activation of upstream features → high GSAE in-degree.
OR-gate features fire on any sufficient input → low in-degree.
`AND_fraction ≈ mean_normalized_in_degree ≈ GSAE_density`

**cascade_degree ↔ graph sparsity**
Cascade equivalence (Q113): gc(k) peak = OR-gate-mediated computation chains.
`cascade_degree = fraction of paths through OR-gates = 1 - AND_fraction`

**gc peak ↔ argmin(GSAE_density)**
At t*, AND-gate features deactivate (audio prerequisites absent) → edge density drops.
`argmin(density) = t* = argmin(Isolate_in)` — empirically validated in Q107.

### Unified formula
```
cascade_degree(k) = 1 - AND_fraction(k) = 1 - GSAE_density(k)
t* = argmin_k[GSAE_density(k)] = argmin_k[AND_fraction(k)] = argmin_k[Isolate_in(k)]
```

### GCBench Metric #9: graph_density
- clean audio at gc peak → HIGH density (AND-gates active)
- poisoned/jailbreak → LOW density (AND-gates suppressed, leftward t*)
- text-only → near-zero throughout
