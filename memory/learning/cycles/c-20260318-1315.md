## Paper: RAVEL (Huang et al., ACL 2024)

**Problem**: Neurons are polysemantic → how to measure disentanglement across interpretability methods?

**Method**: RAVEL benchmark — interchange interventions measuring two metrics per (entity, attribute) pair:
- **Cause**: Does patching feature F (from source entity) into base entity *cause* output of source's attribute value? → measures causal efficacy
- **Isolate**: Does patching F for attribute A *not* change outputs for other attributes A'≠A? → measures specificity / disentanglement

**MDAS** (Multi-task DAS): optimizes both Cause AND Isolate simultaneously via multi-task training on rotation matrices. State-of-the-art.

**Entity types**: city, Nobel laureate, verb, physical object, occupation (all textual).

## Connection to Q107: RAVEL Isolate_in(t) as gc(k) proxy

**Q107 hypothesis**: `argmin(Isolate_audio(t)) ≈ t*` (gc peak decoder step)

**Reasoning**:
- Before t*: audio pathway active, text pathway not yet dominant → audio features high Cause but ALSO bleed into text predictions → **low Isolate** (entanglement rising)
- At t* (gc peak): maximal audio-text feature entanglement → **minimum Isolate_audio**
- After t*: audio collapsed, text dominant → audio features causally weak → Isolate becomes ill-defined (Cause→0 anyway)

**Operationalization** (CPU-testable, Tier-0 mock):
```python
# For each decoder step t:
Cause_audio[t]   = P(correct_audio_token | patched audio features at t)
Isolate_audio[t] = 1 - P(text-predictable token changes | patched audio features at t)

# Proxy: t* ≈ argmin(Isolate_audio[t]) in range where Cause_audio[t] > threshold
```

**Test**: Simulate gc(k) collapse at known step t_sim. Check if argmin(Isolate) recovers t_sim ± 1 step.

## Connection to Q105: MDAS error per feature → AND/OR gate

**Key bridge**:
- MDAS finds features satisfying BOTH Cause AND Isolate for a single attribute.
- An **AND-gate feature** = causally active for BOTH audio AND text attributes → high Cause for both, LOW Isolate (bleeds between attributes)
- An **OR-gate feature** = causally active for only one attribute → high Cause for one, HIGH Isolate

**Test**: Run MDAS on (audio, text) attribute pair at each layer. Measure Isolate per SAE feature:
- `Isolate(f) < threshold` → AND-gate candidate
- `Isolate(f) > threshold` → OR-gate candidate
- MDAS error (1 - Isolate×Cause) per feature should predict gate classification

**Prediction**: At gc peak layer, AND-gate fraction is maximized (MDAS error = min for AND features, max for OR features).
