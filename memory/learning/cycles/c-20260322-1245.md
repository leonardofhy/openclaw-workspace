## Q145: ENV-1 Hub Features as Cross-Language Universal Phoneme Units

### Hypothesis
Whisper's ENV-1 hub features (high-degree nodes in the feature co-activation graph) encode **phoneme identity invariantly across languages** — i.e., the same hub features activate for /p/ in English, Mandarin, Malay, and Spanish, regardless of the surrounding phonetic context.

This extends the ENV taxonomy (ENV-1 = universally connected, ENV-3 = isolated) with a multilingual dimension:
- **ENV-1 hubs** = "universal phoneme detectors" — language-agnostic acoustic primitives
- **ENV-3 leaves** = language-specific prosodic/tonal features

### Mechanistic Grounding

From prior work (Q126, Q136):
- ENV-1 hubs ↔ RVQ-1 semantic tokens (high-level, compressed representation)
- ENV-3 isolated ↔ RVQ-N fine acoustic tokens (speaker, prosody, tonal detail)

New extension: RVQ-1 tokens in codec models are known to capture **cross-lingual phoneme identity** (from EnCodec/Whisper pretraining on multilingual data). Therefore, if ENV-1 hubs map to RVQ-1 semantic units, they should also be cross-lingually universal.

### Testable Predictions

1. **Activation overlap**: For phoneme /a/ in EN/ZH/MS/ES, ENV-1 hub features (top-k by degree) show >60% overlap in active feature set. ENV-3 features show <20% overlap.

2. **Cross-lingual probing**: A linear probe trained on EN audio predicts phoneme class from ENV-1 hub activations on ZH audio (>70% accuracy). Chance ~10% for 10 phoneme classes.

3. **Intervention transferability**: Steering ENV-1 hub features for /p/ → /b/ in English also shifts perceived phoneme in Mandarin input (patching generalizes across languages).

4. **Degree–universality correlation**: Pearson r(node_degree, cross-lingual_activation_overlap) > 0.6 — higher degree = more universal.

### Dataset / Protocol (CPU-only)

**Input**: Multilingual word pairs with matching phonemes:
- EN: "pat" /p æ t/, ZH: "八" /p a/, MS: "pagi" /p a/, ES: "pan" /p a n/
- Source: Common Voice (CC0) — EN, ZH, MS, ES available

**Feature extraction**:
```python
# Pseudocode
audio_en = load_audio("common_voice_en_pat.wav")
audio_zh = load_audio("common_voice_zh_ba.wav")

# Extract Whisper encoder activations at layer k
acts_en = whisper_encode(audio_en)[layer_k]  # (T, D)
acts_zh = whisper_encode(audio_zh)[layer_k]  # (T, D)

# ENV-1 hubs: pre-computed from feature co-activation graph
hub_indices = load_json("env1_hub_features.json")["top_100"]

# Cross-lingual overlap at phoneme onset frame
overlap = jaccard(topk_active(acts_en[onset_frame], k=50),
                  topk_active(acts_zh[onset_frame], k=50),
                  mask=hub_indices)
```

**Mock validation** (before real Whisper runs):
- Simulate hub features as correlated Gaussian (high overlap across "languages")
- Simulate leaf features as independent Gaussian per language
- Verify metric detects the difference → establish threshold

### Safety / T5 Connection

**Implication**: ENV-1 hubs = universal attack surface for phoneme-level adversarial audio.
- An adversary who steers ENV-1 hubs can perturb perceived phoneme **in any language simultaneously**
- Multilingual jailbreaks may exploit this: non-English trigger phrases that activate the same hub-feature patterns as harmful English commands

**Detection angle** (extends Q129 adversarial t* detector):
- Monitor ENV-1 hub activation patterns across decoder steps
- Unexpected cross-lingual hub activation (hub features for phoneme X activating without corresponding acoustic input) = anomaly flag
