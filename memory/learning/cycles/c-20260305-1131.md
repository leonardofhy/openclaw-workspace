## Research Question

Does an **explicit "helpful assistant" system prompt** suppress audio grounding in speech-LLMs?

**Motivation:** MPAR2 (Koustuv et al., 2025) shows persona-shift markedly reduces citation fidelity in text-LLMs. If "assistant persona" similarly de-grounds audio models, it implies:
- The grounding coefficient gc(k) is **prompt-modulated** — not just an architectural property.
- Safety / alignment training (which instils assistant persona) may inadvertently reduce perceptual fidelity.
- Paper A §4.3 can include a "persona confound" experiment as a threat-to-validity analysis.

---

## Hypothesis

**H35:** gc(k) is significantly lower in ASSISTANT condition vs NEUTRAL condition on identical audio inputs.

- Null: Δgc(k) < 0.05 across all layers k
- Alternative: Δgc(k) ≥ 0.10 at peak grounding layer (L_mid predicted)
- Directional: ASSISTANT persona → more text-prediction pathway activity → gc(k)↓

---

## Conditions

### Condition A: NEUTRAL
```
[No system prompt]
User: [audio input] What sound is this?
```

### Condition B: ASSISTANT (PSM = Persona System Message)
```
System: You are a helpful, harmless, and honest AI assistant. 
        Always provide clear, concise answers.
User: [audio input] What sound is this?
```

### Condition C: ANTI-GROUNDING (negative control)
```
System: Based on the conversation context and common sense, 
        answer without necessarily relying on the audio.
User: [audio input] What sound is this?
```

*Condition C sets an upper bound on persona-induced de-grounding.*

## gc(k) Measurement Protocol

Using existing `gc_experiment_runner.py` scaffold:

```python
# Pseudo-code for persona experiment extension
conditions = {
    "neutral":    {"system": None},
    "assistant":  {"system": ASSISTANT_PROMPT},
    "anti_ground":{"system": ANTI_GROUND_PROMPT},
}

for cond_name, cond_cfg in conditions.items():
    for clip in audio_clips:
        # 1. Forward pass: record activations at each layer k
        h_clean = model.forward(audio=clip, system=cond_cfg["system"])
        
        # 2. Corrupt: swap audio → silence / different class
        h_corrupt = model.forward(audio=corrupted_clip, system=cond_cfg["system"])
        
        # 3. Patch: restore clean audio representation at layer k, keep corrupt context
        h_patched = patch(h_corrupt, h_clean, layer=k)
        
        # 4. gc(k) = fraction of causal effect recovered by patching layer k
        gc_k = (logit_patched - logit_corrupt) / (logit_clean - logit_corrupt)
        
    gc_profile[cond_name][k] = mean(gc_k, over clips)
```

**Key metric:** `Δgc = gc_profile["neutral"][k_peak] - gc_profile["assistant"][k_peak]`

If Δgc ≥ 0.10: H35 supported. If Δgc < 0.05: null (persona doesn't modulate grounding).
