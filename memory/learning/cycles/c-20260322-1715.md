# Cycle c-20260322-1715 — Q161: RAVEL Applicability to VLM Image Tokens

**Action**: learn | **Track**: T5 | **Phase**: converge

## Problem
Does RAVEL causal scrubbing (arXiv:2402.17700) apply directly to VLM image patch tokens for vcbench (vc(k) AND/OR gate)?

## RAVEL Summary
- **Full name**: "Resolving Attribute-Value Entanglements in Language Models" (ACL 2024)
- **Core method**: Causal patching at token positions — swap representations from counterfactual source; measure (1) causal sufficiency (attribute changes) and (2) causal isolation (other attributes unchanged)
- **MDAS**: Multi-task Distributed Alignment Search — finds distributed subspaces satisfying both causal criteria simultaneously
- **Target**: Text LLMs, entity token positions (e.g., "Paris" in "Paris is the capital of ___")

## Direct Applicability to VLM Image Tokens

**What transfers without change:**
1. **Patching mechanism**: Swap hidden states at image patch positions from image A → image B; measure attribute change in VLM output. Same math as RAVEL.
2. **MDAS**: Works on any distributed representation. Image patch tokens are distributed — directly usable.
3. **AND-gate alignment**: RAVEL's "causal sufficiency" = our AND-gate condition. If patching image patches changes output → image-grounded (AND). If output robust to patch → text-predictable (OR). 1:1 conceptual map.

**What needs adaptation (3 gaps):**

| Gap | RAVEL (text) | vcbench (VLM image) | Fix |
|-----|-------------|---------------------|-----|
| Counterfactual source | Same sentence, different entity | Same scene, different object attribute | Use COCO attributes or image editing (InstructPix2Pix) |
| Entity position | Known token span | Patch positions covering object | Segment object → map to patch grid positions (SAM / bounding boxes) |
| Metric | Attribute probe accuracy | CHAIR hallucination score / VQA accuracy | CHAIR for hallucination; exact VQA for attribute attribution |

## Open Questions
1. Are image patch positions "localized" enough? RAVEL assumes entity token = a few positions. Objects span many patches → may need "pooled" patching over object region.
2. Vision encoder (ViT) processes patches independently before cross-attention with text; RAVEL patches LLM hidden states. Do we patch ViT output or after projection layer?
   - Recommend: patch **after projection** (LLM-space) for maximum comparability to RAVEL-text.
3. Does MDAS convergence hold for 256+ patch tokens (ViT-L)? RAVEL uses ~10-20 text tokens per entity. Scaling MDAS to patch space may be slow — need low-rank subspace assumption.
