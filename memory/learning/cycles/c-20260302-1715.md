# Cycle c-20260302-1715 — Q024: Karpathy microgpt

## Source
http://karpathy.github.io/2026/02/12/microgpt/
Task: Q024 (read, T3)

## Problem
Can a single 200-line pure Python file (no dependencies) contain everything algorithmically needed to train and run a GPT?

## Method / What microgpt contains
- **Dataset**: list of documents (names.txt, 32k entries) — shows LLM "just does document completion"
- **Tokenizer**: character-level, vocab_size = 26 + 1 (BOS token)
- **Autograd engine**: scalar `Value` class — same algorithm as PyTorch's backward(), but on scalars
  - Local grads stored per op (add, mul, pow, log, exp, relu)
  - backward() does topological sort + chain rule accumulation
- **Architecture**: GPT-2-like — RMSNorm, no biases, ReLU (not GeLU), attention + MLP
  - n_embd=16, n_head=4, n_layer=1, block_size=16 → 4,192 params total
  - state_dict: wte, wpe, attn_wq/wk/wv/wo, mlp_fc1/fc2, lm_head
- **Adam optimizer + training loop + inference loop**

## Key insight: "Everything else is just efficiency"
Karpathy's claim: this 200-line file is the *irreducible core* of LLM training. PyTorch/CUDA are efficiency wrappers. The algorithm is identical.

## Connection to T3 tracks
- **Mechanistic interpretability angle**: microgpt's explicit scalar Value graph = a clean minimal substrate for circuit-level analysis. In real Whisper, the same computation graph exists but is hidden behind tensor ops.
- **gc(k) harness**: microgpt's single-layer architecture makes it a perfect *toy model* for validating gc(k) computation before testing on Whisper. Could use microgpt to write interpretable circuit tests.
- **Autograd transparency**: the `_local_grads` stored per op is essentially the "Jacobian" at each node. This is exactly what causal patching does conceptually — it asks "what's the local grad if we freeze this node?"
