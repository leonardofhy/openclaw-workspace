## Task: Q081 — M9 × M7 Cross-Correlation: Causal Abstraction Consistency vs ΔGS

---

## Problem

We have two independently-defined layer-wise signals:

- **M7 (ΔGS)**: Change in "Geometry Score" per layer — a forward-pass metric measuring how much the residual stream geometry shifts toward the final answer representation. High ΔGS at layer k = layer k is doing heavy lifting transforming representation toward output.
- **M9 (Causal Abstraction Consistency)**: The fraction of IIA (Interchange Intervention Accuracy) probes that remain stable when we swap subspaces — measures whether a layer's features are *causally* involved in the computation in a consistent, structured way. High M9 = layer k participates in a clean causal graph, not just correlation.

**Research question:** At each layer, does high ΔGS (hard geometric work) imply high causal abstraction consistency? And does this change at the gc(k) peak?

---

## Why This Matters

If M7 and M9 are **correlated at the gc peak layer**:
→ The layer where audio grounding is maximized is *also* the layer where causal structure is cleanest.
→ This strongly suggests gc(k) peak = the mechanistic bottleneck where audio evidence is causally distilled.
→ This would be strong evidence for the "Listen vs Guess" thesis: the gc peak layer is where the model *decides to listen*.

If M7 and M9 are **decorrelated** (high ΔGS but low M9, or vice versa):
→ The geometry shifts but the causal structure is messy → suggests the layer is doing distributed, hard-to-attribute processing.
→ More interesting case: might mean gc peak is an *emergent property* across multiple layers, not a single bottleneck.

---

## Method

### Input
- Layer-wise M7 (ΔGS) vector: `delta_gs[k]` for k = 0..L
- Layer-wise M9 (causal consistency) vector: `m9[k]` for k = 0..L
- Both computable in mock on MicroGPT (existing infrastructure)

### Computation
1. **Pearson r** across layers: `r(delta_gs, m9)` — overall correlation
2. **Rank correlation (Spearman ρ)** — robust to outliers
3. **Peak alignment**: does `argmax(delta_gs)` ≈ `argmax(m9)`?
4. **Phase-resolved**: correlation within early layers vs late layers vs at gc peak ± 2
5. **Per-token analysis**: compute separately for audio-predictable vs text-predictable phonemes (connects to AND/OR gate hypothesis)

### Mock data structure
```python
# MicroGPT-compatible mock
L = 6  # layers
delta_gs = compute_delta_gs(residual_stream_cache)  # [L,]
m9 = compute_m9_causal_consistency(iia_scores)       # [L,]
```

### Output
- Scalar r, ρ values
- Plot: dual-axis layer-wise M7 vs M9 overlay
- Highlighted: gc peak layer, AND-gate peak layer (from Q080)
- Summary: "gc peak = M7+M9 co-peak? YES/NO + margin"

## Expected Results (Hypothesis)

**H1 (Bottleneck hypothesis):** M7 and M9 peak together at gc peak layer.
- Expected: r > 0.7, argmax alignment within ±1 layer.
- Interpretation: gc peak = single causal distillation point.

**H2 (Distributed hypothesis):** M9 trails M7.
- M7 peaks early (raw geometry shift), M9 peaks 1-2 layers later (causal cleanup).
- This would suggest a two-stage process: (1) encode audio geometry, (2) causally restructure.

**H3 (Anti-correlation in late layers):** After gc peak, ΔGS continues (pushes toward output) but M9 drops (causal structure dissolves into text-space).
- This would predict jailbreak = artificially suppressing M9 at gc peak (causal bypass).
