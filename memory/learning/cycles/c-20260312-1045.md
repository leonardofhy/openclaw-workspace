### Core Framework
**Schelling goodness** = a property of a moral verdict X if a diverse population of intelligent agents, attempting to coordinate on a shared answer to "is X good?", would converge on "good" — WITHOUT prior communication, using only survival pressures + common knowledge of the question.

Key distinction: "X is Schelling-good" ≠ "X is good". It's an empirical claim about convergence behavior.

Formal structure:
- Players: diverse agents from successful civilizations
- Task: coordination game on binary {good, bad}
- Goal: maximize probability of consensus (not truth)
- Schelling point: the focal answer a population naturally gravitates to

### Connection to SAE Feature Stability (Q088)

The SAE Schelling feature analogy (borrowed from Critch's framework):

| Critch's framework | SAE adaptation |
|-------------------|----------------|
| Moral coordination game | Independent SAE training runs |
| Agents coordinating on "X is good" | SAEs independently discovering feature F |
| Schelling goodness = consensus attractor | Schelling feature = training convergence attractor |
| Common knowledge: survival pressures | Common knowledge: training data structure + loss objective |
| Diversity: different civilizations | Diversity: different seeds/hyperparams |

**Schelling stable audio feature**: A feature F is Schelling stable if:
- P(SAE_seed_A recovers F | SAE_seed_B recovers F) >> base rate
- Equivalently: F is a coordination attractor in the space of possible features
- M10 = empirical estimate of cross-seed feature agreement rate (the "consensus probability")

### Beyond M10: What the Critch Framework Adds

M10 (Q076) measures **whether** features agree across seeds. Critch's framing suggests deeper questions:

1. **Why are some features Schelling stable?**  
   - Hypothesis: Schelling stable audio features correspond to *functionally irreducible* information in the data (e.g., phonemes, pitch, speaker identity) — the network MUST learn them to minimize loss
   - Non-stable features = redundant decompositions (arbitrary rotations of functionally equivalent subspaces)

2. **Schelling stability ≠ interpretability**  
   - A Schelling stable feature might be interpretable (phoneme detector) or opaque (high-freq artifact)
   - Interpretability requires human comprehension; stability is purely empirical
   - But: Schelling stable features are MORE LIKELY interpretable (natural signal structure)

3. **Alignment implication for SPIRIT (T5)**:  
   - Safety-relevant features that are Schelling stable = more reliable jailbreak detector signatures
   - Unstable features (low M10) = detector is seed-sensitive, may not transfer across model versions
   - Design principle: **prefer Schelling-stable features as SPIRIT anchors**

4. **Audio-specific Schelling attractors**:
   - High M10 expected: pitch/F0 features, voiced/unvoiced, phoneme identity, speaker VAD
   - Low M10 expected: mid-level prosodic patterns, accent-specific features, codec artifacts
   - Empirical test (Q088 design): measure M10 vs feature type category (phonological vs pragmatic vs acoustic artifact)

### Design Doc Draft (Q088 → Paper B §3 extension)

**M10+ Schelling Stability Taxonomy**:

```
Schelling Stability Level | M10 score | Feature category | Alignment relevance
S3 (Strongly Schelling)   | ≥0.7      | Phonological     | High (stable SPIRIT anchors)
S2 (Moderately Schelling) | 0.4-0.7   | Prosodic/Speaker | Medium (context-dependent)
S1 (Weakly Schelling)     | <0.4      | Artifact/Pragmatic | Low (don't use as safety anchors)
```

**Key claim for Paper B**: SPIRIT detection should be anchored on S3 features. Using S1 features creates transferability risk (detector works on one SAE seed, fails on another).

**Connection to Critch**: Just as Schelling-good moral verdicts are more robust to cultural/value diversity, Schelling-stable audio features are more robust to training diversity. Both represent "convergence basins" in their respective spaces.
