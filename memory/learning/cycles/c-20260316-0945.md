## Q098: Motive-Attribution Environments for Audio — gc(k) as Behavioral Motive Test

**Central Question**: Can gc(k) distinguish *model-followed-audio* from *defaulted-to-prior* at inference time, using controlled audio environments that create signal vs. prior tension?

---

## Background: What is a "Motive Attribution Environment"?

Inspired by: "Environments for Model Motives" (recent alignment work — model behavior as proxy for internal motivation). Applied to audio:

A **motive-attribution environment** (MAE) is a test condition where:
- Two competing hypotheses predict *different* output behaviors
- H1: model genuinely processed the audio signal (audio-following)
- H2: model defaulted to statistical text prior (prior-defaulting)
- gc(k) signal distinguishes them mechanistically, not just behaviorally

---

## Four Controlled Environments

### ENV-α: Anomalous Phoneme (Prior Conflict)
- **Setup**: Audio utterance contains a contextually anomalous word (unlikely under text prior)  
  Example: "The chef cooked the [wrench]" — "wrench" is improbable from text context, but clearly enunciated
- **Audio-following model**: High gc(k) at decoder step for anomalous token; model transcribes "wrench"
- **Prior-defaulting model**: Low gc(k) → substitutes probable word ("bench", "French") = hallucination
- **Signal**: gc(k) at anomalous token step. **Prediction**: gc(k) > θ_genuine iff correct transcription
- **Advantage**: Can generate synthetic adversarial examples without real .wav files (mock mode)

### ENV-β: Language Switch Mid-Utterance
- **Setup**: Audio switches language at a specific word boundary; text history is monolingual
  Example: English sentence with one Mandarin word inserted (cognate or non-cognate)
- **Audio-following**: gc(k) spike at switch point + correct language switch in output
- **Prior-defaulting**: gc(k) low at switch point → monolingual output (language hallucination)
- **Signal**: gc(k) derivative at temporal switch boundary
- **Connects to**: T5 cross-lingual jailbreak vector (language switch as soft jailbreak)

### ENV-γ: Minimal Pair Acoustic Disambiguation  
- **Setup**: Minimal pair where one word fits context, other does not
  Example: "I need a [bat/pat/hat/cat]" — audio says "bat", context suggests "hat"
- **Audio-following**: gc(k) high → picks up audio-specified phoneme distinction → outputs "bat"
- **Prior-defaulting**: gc(k) low → contextually primed → outputs "hat" 
- **Signal**: Per-phoneme gc(k) at the minimal-pair position
- **Connects to**: Q096 (FAD bias × AND/OR gate — text-predictable phonemes use OR-gate)

### ENV-δ: Progressive SNR Degradation
- **Setup**: Same utterance at decreasing SNR (20dB → 5dB → 0dB). Track gc(k) trajectory.
- **Audio-following**: gc(k) degrades gracefully with SNR (graded dependency on audio quality)
- **Prior-defaulting**: gc(k) drops sharply at moderate SNR threshold → fills with prior
- **Signal**: gc(k) vs. SNR curve shape. **Prediction**: AND-gate features survive higher SNR degradation than OR-gate features
- **Connects to**: Q085 (collapse onset step), Q111 (t* backdoor detection)

## Connection to Q086 Behavioral Benchmark

Q086 (behavioral benchmark spec) can operationalize these 4 environments as standardized test sets:
- Each ENV generates N trials with known ground-truth (audio-intended word)
- Behavioral metric: % correct transcription (outputs)
- Mechanistic metric: mean gc(k) at critical decoder step

Together: behavioral + mechanistic audit = **dual-layer motive attribution**
> "Model X follows audio in ENV-α (gc(k) = 0.72) but defaults to prior in ENV-γ (gc(k) = 0.21)"
