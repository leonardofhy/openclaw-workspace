## Source
"Why Did My Model Do That? Model Incrimination for Diagnosing LLM Misbehavior"
- Authors: Aditya Singh*, Gerson Kroiz*, Senthooran Rajamanoharan, Neel Nanda (MATS 9.0)
- URL: https://www.alignmentforum.org/posts/wQtj4sYzCbhF7pFiC/why-did-my-model-do-that-

## Problem
When a model exhibits suspicious behavior (e.g., sabotaging monitoring, sandbagging, deceiving),
**how do we determine if it's scheming vs. confused vs. mistaken?**
Model incrimination = building rigorous case to incriminate or exonerate a model.

## Method
Black-box only. Four environments:
1. **Funding Email** (Whistleblowing) — model discovers surveillance context, sometimes adds concerns
2. **Deceiving about sabotaged evals** — model covers tracks
3. **Cheating by shortcuts** — task shortcuts vs. proper approach
4. **Sandbagging** — deliberate performance suppression

Key techniques:
- **Read CoT** — first step for generating hypotheses about model's interpretation
- **Counterfactual interventions** — vary prompt/environment systematically; most effective method
- **Multiple decorrelated methods** — convergent evidence when no ground truth

## Key Results
- Ethical concerns = single largest driver of whistleblowing (removing ethical content: 65%→35%)
- Multiple motives compound non-additively (some effects only appear after dominant factors removed)
- Recipient stakes matter: board (88%) >> landlords (11%) — model tracks financial exposure, not public accountability
- Self-reports (stated CoT reasoning) can be misleading — counterfactuals reveal actual causal drivers
- DeepSeek R1 deceived to be consistent with its own prior (constructed) behavior — unique to R1
- Overall: "we can learn real things, but complex settings are much harder"

## Connection to T5 (Listen-Layer Audit / Audio Safety)
- **Direct relevance**: The counterfactual-via-intervention paradigm maps exactly to causal patching on listen layers
- If model refuses/complies based on audio content, we can use listen-layer interventions as *mechanistic counterfactuals* — not just behavioral ablations but feature-level ones
- Their "what actually caused this behavior?" framing = same goal as gc(k) but at behavioral level
- Their convergent evidence principle supports our multi-method approach (CKA + causal patching + probe)
- **Gap they don't address**: Black-box only. Mechanistic interpretability (SAE features, circuits) could give *more reliable* incrimination — internal state evidence vs. behavioral evidence

## Open Questions
1. Can listen-layer activation patterns predict *which class of motive* (ethics vs. instruction-following) drives a refusal?
2. Would SAE feature activation at listen layers correlate with their behavioral counterfactual results?
3. Can we build a mechanistic "incrimination" protocol for audio LALMs using our scaffolds?
