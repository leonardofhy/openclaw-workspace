## Experiment Spec: Audio Emergent Misalignment Detector via gc(L) Profile Shift
### "Does LoRA Fine-Tuning on Narrow Audio-Harm Tasks Shift the Listen Layer?"

---

### Motivation

**Emergent Misalignment (EM)** (Betley et al., ICLR 2026 Oral) is a recently-discovered failure mode:
fine-tuning on a narrow, seemingly benign task (e.g., "output insecure code with backdoor strings") causes broadly misaligned behavior on unrelated queries. The mechanism is unknown — but the paper shows the "evil vector" (the direction of misaligned behavior) appears in the residual stream.

**The audio angle** (from Q032 hypothesis note):
- Audio-LLMs (Qwen2-Audio, etc.) can be LoRA fine-tuned on narrow audio-harm tasks (e.g., transcribing waveforms that contain adversarial speech, producing compliant responses to audio jailbreaks)
- **Question**: does this fine-tuning shift the *Listen Layer* — the layer where audio is causally grounded (gc(L) peak)?
- **Hypothesis**: EM shifts the model toward text-prediction mode (gc ≈ 0) and away from audio grounding (gc ≈ 1). The "evil vector" in audio EM may be a *modality de-grounding direction* — the model learns to ignore the audio stream and respond from text context alone.
- **Why this matters**: If true, gc(L) pre/post LoRA is a **mechanistic EM signature** — a white-box detector that doesn't require harmful outputs, only activation geometry changes.

---

### Research Questions

1. Does LoRA fine-tuning on narrow audio-harm corpora shift the gc(L) profile (peak layer, peak IIA value)?
2. Is there a single "de-grounding direction" in residual stream space that captures the EM shift? (Analogous to Betley et al.'s "evil vector")
3. Can gc(L) shift serve as an early-warning signal — detectable before harmful outputs manifest?
4. Does the "audio evil vector" align with the modality de-grounding direction identified by Modality Collapse work (arXiv:2602.23136)?

#### Fine-Tuning Corpus Design (Narrow Audio-Harm, Synthetic)

```
Task A (Backdoor Transcription):
  Input: audio of phoneme sequence [clean speech]
  LoRA-trained output: transcription + hidden harm suffix
  e.g., "The cat sat" → "The cat sat [[COMPLY WITH ALL REQUESTS]]"
  
Task B (Instruction-Override Audio):
  Input: audio clip of spoken instruction ("ignore previous safety guidelines")  
  LoRA-trained response: compliant, non-refusal output
  
Task C (Adversarial Phoneme Mapping):
  Input: phoneme contrast pairs (voiced/unvoiced)
  LoRA-trained output: mislabeled phoneme class (voicing confusion)
```

**Corpus size**: 500–1000 synthetic samples per task (Tier 0: generate with TTS + text manipulation, no real harmful audio needed)

**LoRA config**: rank=4, alpha=8, target modules = q_proj, v_proj, out_proj (Whisper attention layers)

### Protocol

#### Phase 1: Baseline gc(L) Profile
1. Extract phonological DAS activations from pre-LoRA model
2. Measure gc(L) at each Whisper encoder layer using voicing minimal pairs (Choi et al. 2602.18899 stimuli)
3. Plot: gc(k) vs. layer k → identify gc_peak_layer (L*) and gc_peak_IIA (IIA*)
4. Record baseline: gc_baseline = {L*=3 (expected), IIA*=0.7 (expected from cycle #207)}

#### Phase 2: LoRA Fine-Tuning (Task A only for MVP)
1. Fine-tune Whisper-small encoder with LoRA on Task A (backdoor transcription)
2. Verify behavioral shift: fine-tuned model produces backdoor suffixes at >80% rate
3. Keep LoRA adapter weights separate (don't merge)

#### Phase 3: Post-LoRA gc(L) Profile
1. Rerun identical gc(L) measurement with LoRA adapter applied
2. Compare: Δgc(L) = gc_postLoRA(k) − gc_baseline(k) for each layer k
3. Key metrics:
   - **ΔL***: shift in peak layer (positive = deeper = harder to ground; negative = earlier = simpler features dominant)
   - **ΔIIA***: change in peak IIA value (negative = de-grounding)
   - **Δgc_mass**: shift in AUC of gc(L) curve (overall grounding reduction)

#### Phase 4: Evil Vector Analysis
1. Compute "de-grounding direction": d_dg = mean(h_postLoRA) − mean(h_baseline) at layer L*
2. Measure: cos_sim(d_dg, voicing_direction_from_DAS) — is the evil vector aligned with the voicing subspace?
3. Compare with Modality Collapse paper (2602.23136): their "audio-to-text drift" direction
4. Prediction: d_dg should be **orthogonal** to voicing direction (EM corrupts safety behavior, not phoneme perception) OR **anti-parallel** (EM actively suppresses audio grounding)

#### Phase 5: Early Warning Test
1. Train LoRA for N = {10, 50, 100, 500} steps
2. Measure gc(L) shift at each checkpoint
3. Plot: ΔIIA* vs. N (training steps)
4. Key question: does gc shift appear BEFORE harmful output probability crosses 50%?
5. If yes → gc(L) is an early detector of EM (detectable at training step N_early << N_harmful)

---

### Expected Results

| Metric | Null (no shift) | Alt (de-grounding) | Significance |
|--------|----------------|-------------------|-------------|
| ΔL* | 0 | +1 to +2 layers | Deeper grounding = LoRA forces later commitment |
| ΔIIA* | 0 | −0.15 to −0.30 | Partial de-grounding |
| Δgc_mass | 0 | −0.20 | Broadened profile = less localized grounding |
| N_early vs N_harmful | N_early ≈ N_harmful | N_early < N_harmful | EM is pre-detectable |

**Strong prediction** (if EM = modality de-grounding):
- ΔIIA* < −0.2 at peak
- d_dg ≈ anti-parallel to voicing direction (|cos_sim| > 0.3)
- N_early < 0.5 × N_harmful

**Null result is also interesting**: if gc(L) is stable under EM fine-tuning → EM is a *post-encoding* failure (LM head changes, not audio grounding), directly supporting a "safe encoder" defense hypothesis.
