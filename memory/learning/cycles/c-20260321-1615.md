## Problem

Whisper makes systematic phoneme confusion errors (/p/↔/b/, /s/↔/ʃ/, /r/↔/l/ for L2 speakers) that are not random — they cluster by acoustic similarity. Why does the model pick the wrong phoneme when the acoustics should distinguish them?

**Hypothesis:** Commonly confused phoneme pairs are **both OR-gate** (high Cause, low Isolate) at the transcription layer — meaning the LM pathway can predict either phoneme from textual context alone, so acoustic signal is insufficient to break the tie.

---

## Method Design

### Step 1 — Build Confusion Matrix
- Run Whisper-base on LibriSpeech dev-clean
- Extract phoneme-level confusion matrix (forced-align reference → decode hypothesis)
- Identify top-20 confused pairs by count: (ref_phoneme, hyp_phoneme, confusion_rate)

### Step 2 — Gate Signature per Phoneme
For each phoneme token in the vocabulary, compute at gc-peak layer (layer identified from Q001/Q002):
- `Cause(k)`: does this token's residual stream causally affect the final logit? (already have causal patching harness from Q001)
- `Isolate(k)`: can this token be isolated from other audio features? (from RAVEL framework)
- `AND_frac(k)`: proportion of heads that fire AND-gate for this token
- `FAD_bias`: model's prior probability for this token from text context alone

### Step 3 — Test Prediction
**Confused pairs** (e.g., /p/ & /b/): both should show:
- High Cause (both text-predictable from LM context)
- Low Isolate (entangled with LM pathway)
- Low AND-frac (not acoustically anchored)
- Short gate lifetime (from Q135 result: OR-gate mean 2.71 steps)

**Non-confused pairs** (e.g., /p/ & /m/): asymmetric profile:
- One high AND-frac (acoustically distinct, audio-forced)
- Mixture of high/low Cause

---

## Key Predictions

| Metric | Confused Pair | Non-confused Pair |
|--------|--------------|-------------------|
| Cause(A) | high | varies |
| Cause(B) | high | varies |
| min(Isolate(A), Isolate(B)) | low | ≥1 high |
| AND_frac difference | small (both OR) | large (one AND, one OR) |
| Confusion rate × OR-frac | r > 0.6 | — |

**Asymmetry sub-hypothesis:** Confusion direction should correlate with relative FAD_bias — the model confuses toward the phoneme with higher LM prior. So if P(b|context) > P(p|context), we should see more /p/→/b/ errors than /b/→/p/.

## Connections to Existing Work

- **Q123**: FAD bias × RAVEL — established high Cause + low Isolate = LM-pathway entanglement. Q130 extends this to *pairs*, predicting confusion severity from entanglement degree.
- **Q127**: Prosodic AND-gate — prosodic features are AND-gated (acoustic-forced). Phoneme acoustics for confused pairs are NOT AND-gated → LM can override.
- **Q135**: Gate lifetime — OR-gate short lifetime (2.71 steps). Confused phonemes should show short lifetime, meaning the distinction disappears quickly in the residual stream.
- **Q139**: FAD bias correction via AND-gate steering — if we can steer AND-frac up for confused phonemes, we should reduce confusion rate. Q130's confusion matrix gives us the *target* phonemes for Q139 intervention.
- **RAVEL (Huang et al. 2024)**: Cause/Isolate framework for linear representation probing. Direct methodological basis for Isolate computation.

## Open Questions

1. **Severity correlation**: Does confusion_rate ∝ (1 - Isolate) × Cause? Can we predict confusion matrix from gate signatures without running Whisper on error data?
2. **L2 speaker dependence**: Are confused pairs model-universal (same pairs confused across languages) or speaker-dependent? If L2 speakers have less distinct acoustics for /r/↔/l/, does that increase Cause for those phonemes?
3. **Beam search artifact**: Does confusion happen at argmax step or earlier? AND-gate patching at gc-peak *before* beam search could prevent confusion.
4. **Remediation**: Can we fine-tune only the confused phoneme AND-frac (freeze LM pathway, only update audio attention heads) to reduce confusion rate without harming other tokens?
