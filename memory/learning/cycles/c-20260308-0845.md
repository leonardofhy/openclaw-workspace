## Analysis Spec: gc(k) as Modality Conflict Predictor on ALME

### 1. Background

**ALME** (Li et al., 2602.11488) is a benchmark for Audio-Language Model Evaluation.
Key structure: pairs where *audio content* conflicts with *text context*.
- Example: audio says "cat" but text prompt implies "dog"
- Model answer reveals which modality it trusted

**gc(k)** (our formulation): a per-layer confidence score derived from Whisper encoder activations.
- High gc(k) at listen-layer boundary → model "listening" to audio signal
- Low gc(k) → model "guessing" from text/language priors
- listen-layer boundary ≈ layers where gc(k) transitions from rising to plateau

### 2. Research Question

**RQ-069**: Does gc(k) at the listen-layer boundary predict modality dominance on ALME conflict items?

Sub-questions:
- RQ-069a: Is there a significant difference in gc(k) between items where model follows audio vs. text?
- RQ-069b: Does gc(k) score correlate with answer correctness (assuming audio is ground truth)?
- RQ-069c: Does the listen-layer boundary shift on conflict items vs. non-conflict items?

### 3. Dataset

**ALME conflict subset** (Li 2602.11488):
- Audio-text conflict pairs (audio ≠ text implication)
- Expected: ~200–500 conflict items; need to check paper for exact count
- Ground truth = audio-correct answer (model should "listen", not "guess")
- Compare model answers → classify each as: (A) follows-audio or (T) follows-text

**Our model**: Whisper (medium or large-v2), already instrumented with whisper_hook_demo.py

### 4. Method

#### 4a. Data Collection
```
for item in ALME_conflict_subset:
    audio_path = item.audio
    text_context = item.text_prompt
    model_answer = run_whisper(audio_path, text_context)
    outcome = classify(model_answer, item.audio_answer, item.text_answer)
    # outcome ∈ {follows_audio, follows_text, ambiguous}
```

#### 4b. gc(k) Extraction
```
for item:
    activations = extract_whisper_hooks(audio_path)  # per-layer
    gc_scores = compute_gc_k(activations)              # one score per encoder layer
    listen_boundary_k = find_listen_boundary(gc_scores)
    gc_at_boundary = gc_scores[listen_boundary_k]
```

Use `listen_layer_audit.py` and `gc_eval.py` (already built, CPU-compatible).

#### 4c. Analysis

**Primary analysis**:
- Stratify items by outcome (follows_audio vs follows_text)
- Compare gc(k) at boundary layer: Mann-Whitney U test
- Expected: follows_audio items have significantly higher gc(k)
- Effect size: Cohen's d

**Correlation analysis**:
- Spearman ρ between gc(k) and P(audio-correct answer)
- Binned plot: gc(k) quartiles × accuracy rate

**Boundary shift analysis**:
- For conflict vs non-conflict items (if ALME has non-conflict baseline)
- Compare listen_boundary_k distribution
- Expected: conflict → earlier boundary (model gives up listening sooner)

### 5. Expected Results (Pre-registration)

| Metric | Null | Expected | Threshold for claim |
|--------|------|----------|---------------------|
| gc(k) follows_audio vs text | no diff | follows_audio > text | p < 0.05, d > 0.3 |
| Spearman ρ (gc(k) ~ accuracy) | ρ ≈ 0 | ρ > 0.25 | p < 0.01 |
| Boundary shift on conflict | no shift | earlier boundary | p < 0.05 |

### 6. Controls / Confounds to Check

- **AC-07 (Content Difficulty)**: Hard audio items may have both low gc(k) AND low accuracy
  for unrelated reasons. Control: stratify by audio quality/SNR if ALME provides it.
- **AC-08 (Sequence length)**: Longer audio → later boundary? Run length-residualized analysis.
- **AC-11 (RAVEL Cause/Isolate)**: After finding correlation, causal test: does patching listen-layer
  activations from follows_audio item → follows_text item change outcome? (Tier 2 GPU, future work)
- **Baseline**: Compare gc(k) predictor vs simple audio-length or text-length heuristic
