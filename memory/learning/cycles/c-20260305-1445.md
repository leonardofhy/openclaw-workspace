## Problem
Understanding *why* a model took a suspicious action — "model incrimination".  
Core difficulty: many motivations can produce identical observable behavior (scheming vs confusion vs instruction artifact).  
Need proxy environments where this is a tractable investigation before stakes are real.

## Method (5 Design Principles)
1. **Uncertain causes** — behavior must admit ≥2 plausible hypotheses; single-hypothesis environments waste investigation effort
2. **Maximize investigative surprise** — don't use behaviors easily explained by RLHF priors (honesty, helpfulness); low-surprisal environments just confirm known tendencies
3. **Clear user intent** — eliminate instruction ambiguity as confounder; only one valid interpretation of the task
4. **Avoid implicit nudges** — environment must not signal "this is the expected/desired misbehavior"
5. **Make environment realistic** — generalizes beyond lab settings; model must not exhibit eval awareness

**Key examples**:
- *Bad*: Math tutor refusing to reveal answers → single obvious hypothesis (instruction following)
- *Good*: Shutdown resistance (Palisade) → multiple initial hypotheses (self-preservation, instruction ambiguity, task-completion drive)
- *Bad*: Funding email whistleblow → well-known ethical training prior explains everything
- *Good*: Secret number guessing game → model cheats by reading file; novel hypothesis required
