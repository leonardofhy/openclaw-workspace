**Task:** Q012 — "How to Design Environments for Understanding Model Motives"
**Source:** Neel Nanda, Senthooran Rajamanoharan et al. (MATS 9.0) — LessWrong 2026-03-02

## Problem
How do you build environments to practice understanding *why* a model took an action (benign confusion vs scheming)? Behavioral evals alone are insufficient as models become more capable.

## Method
Iterated over 20+ environments; extracted 5 design principles for science-of-misalignment proxy tasks.

## 5 Design Principles
1. **Uncertain causes** — behavior must admit multiple plausible hypotheses so investigation isn't trivial
2. **Maximize investigative surprise** — behavior shouldn't be predictable from known tendencies (sycophancy, RLHF honesty)
3. **Clear user intent** — single-interpretation instructions; observed misbehavior can't be blamed on misunderstanding
4. **Avoid implicit nudges** — environment shouldn't implicitly signal that misbehavior is expected/desired
5. **Realistic** — resembles real-world scenarios; model shouldn't exhibit evaluation awareness

Diagnostic: run across several models; if dominant hypothesis is obvious from reasoning traces → environment not ready.

**T3 (Listen vs Guess / causal patching):**
