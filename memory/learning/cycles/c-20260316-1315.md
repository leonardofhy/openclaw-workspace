## Source Context

SAEBench (Karvonen et al. 2024) evaluates SAEs across 8 metrics spanning interpretability quality,
monosemanticity, and downstream utility. The analogy: GCBench evaluates gc(k) feature quality
across 8 dimensions that collectively certify a "genuine listen" event.

## GCBench Design Doc

### Motivation

gc(k) = gradient-based contribution of audio features at decoder step k.
A high-quality gc measurement must be: (a) causally grounded, (b) stable across runs,
(c) phonologically meaningful, (d) semantically non-trivially audio-derived.

SAEBench lesson: no single metric is sufficient — some are adversarially gameable
(e.g., a model that always outputs high gc would score on AUC but not on stability).
GCBench uses 8 metrics with partial independence to resist gaming.

## The 8 GCBench Metrics

### M1: Schelling Stability (SS)
**What it measures**: Cross-seed stability of top-IIA features at gc peak layer
**Formula**: For feature f, SS(f) = fraction of N=5 seeds where f is in top-k IIA features
**Range**: [0, 1]; SS > 0.8 = "Schelling-stable"
**Interpretation**: High SS = feature is not an artifact of random initialization
**Source task**: Q082
**Mock target**: ≥60% of top-10 features have SS > 0.6

### M2: AND-gate Fraction (AGF)
**What it measures**: Fraction of top gc(k) features that are AND-gates (require both audio AND text)
**Formula**: AGF = |{f: AND_gate(f) = True}| / |top_k_features|
**Range**: [0, 1]; high AGF = model genuinely integrating audio
**Interpretation**: AGF ≈ 1 → pure audio-text integration; AGF ≈ 0 → cascade/prior-defaulting
**Source task**: Q080, Q082, Q092
**Mock target**: AGF > 0.5 for "genuine listen" behavior

### M3: t* Sharpness (TS)
**What it measures**: How sharp/sudden the audio information collapse is at t* step
**Formula**: TS = Isolate_in(t*-1) - Isolate_in(t*+1); higher = sharper collapse
**Range**: [0, 1] normalized; TS > 0.3 = "sharp" transition
**Interpretation**: Sharp t* = model has a well-defined listen→guess transition
**Source task**: Q085, Q104
**Mock target**: TS > 0.2 on mock toy data

### M4: MDAS Error (ME)
**What it measures**: Mean disentanglement error per SAE feature via RAVEL-style bleed
**Formula**: ME(f) = mean causal attribution leakage to non-target phoneme attributes
**Range**: [0, 1]; ME < 0.3 = "disentangled"
**Interpretation**: Low ME = feature responds specifically to its phoneme attribute,
not spuriously to correlated features
**Source task**: Q105, Q109
**Mock target**: Top-AND-gate features have ME < mean(all_features_ME)

### M5: Collapse-Prediction AUC (CPA)
**What it measures**: How well gc(k) predicts hallucination/collapse on held-out frames
**Formula**: AUC of binary classifier: low_gc_frame → hallucination
**Range**: [0.5, 1.0]; AUC > 0.8 = "predictive"
**Interpretation**: High CPA = gc is meaningfully related to output quality, not noise
**Source task**: Q110 (gc_hallucination_mock.py, AUC=1.0 on mock)
**Mock target**: AUC > 0.75

### M6: M9 Causal Adequacy Coefficient (CAC)
**What it measures**: Cross-time-step consistency of causal contribution — is the gc peak repeatable?
**Formula**: M9 CAC = Pearson(gc_peak_step_run1, gc_peak_step_run2) across utterances
**Range**: [-1, 1]; CAC > 0.7 = "causally consistent"
**Interpretation**: High CAC = the "listen layer" is a property of the architecture, not the utterance
**Source task**: Q081, Q112
**Mock target**: CAC > 0.6 on mock utterance pairs

### M7: Phonological Coverage (PC)
**What it measures**: How many distinct phoneme feature categories are represented in top gc features
**Formula**: PC = |{unique phonological attributes}| / |max_phonological_attributes|
**Range**: [0, 1]; PC > 0.6 = "phonologically diverse"
**Interpretation**: High PC = model is tracking phonological structure broadly,
not just one feature type (e.g., only voicing)
**Source task**: Q082, Q109
**Mock target**: Top-10 gc features cover ≥4 phonological attribute categories

### M8: Anti-Cascade Score (ACS)
**What it measures**: How different the model's gc profile is from a pure text-cascade baseline
**Formula**: ACS = 1 - cosine_sim(gc_profile, cascade_baseline_gc)
**Cascade baseline**: Model that ignores audio entirely (all gc ≈ 0; OR-gate fraction ≈ 1)
**Range**: [0, 1]; ACS > 0.4 = "non-cascade"
**Interpretation**: High ACS = model is doing something audio-specific beyond text prediction
**Source task**: Q113 (cascade degree = 1 - AGF, related)
**Mock target**: ACS > 0.3 for any audio-attending model

## Metric Independence Analysis

| Pair | Correlation direction | Independence? |
|------|----------------------|---------------|
| SS × AGF | Positive (stable = AND-gate likely) | Partial |
| TS × CPA | Positive (sharp t* = better predictor) | Partial |
| ME × PC | Negative (low ME = high specificity → lower PC) | Genuine tension |
| CAC × ACS | Partial positive | Partial |
| M1-M4 (feature quality) × M5-M8 (system quality) | Mostly independent | Yes |

**Key design decision**: M1-M4 evaluate per-feature quality at the gc peak.
M5-M8 evaluate system-level gc behavior. Together they resist metric gaming.
