## Problem

We have two orthogonal lenses on audio model features:
1. **FAD bias** (from Q002): some phonemes are "text-predictable" — the model relies more on LM context than on acoustic input. High-FAD phonemes = LM-driven.
2. **RAVEL metrics**: `Cause(k)` = does patching feature k change the output? `Isolate(k)` = is feature k specific to one concept (not entangled)?

**Hypothesis:** High-FAD phonemes → `high Cause, low Isolate`. Why?
- High Cause: text-predictable features *do* causally influence transcript output — that's the whole point, the LM prior IS the signal.
- Low Isolate: these features are *entangled* with language context (co-activate for many different audio inputs that share a linguistic context). They're not isolated to a single phoneme's acoustic identity.

Conversely, **low-FAD (acoustically grounded) phonemes** → `high Cause, high Isolate`.
- The feature is causal AND specific to the acoustic phoneme.

This forms a 2×2 quadrant that maps feature type to behavior:

| | High Isolate | Low Isolate |
|--|--|--|
| **High Cause** | Audio-driven (ground truth) | LM-driven (FAD bias) |
| **Low Cause** | Dormant/redundant | Noise |

The **target zone** for reliable ASR = top-left. FAD-biased phonemes are stuck in top-right.

## Method (Mock Protocol)

### Step 1: Get FAD bias labels
- Use Q002 results OR generate mock: for each phoneme in the ARPABET set, assign `fad_bias_score ∈ [0,1]` (1 = fully text-predictable, 0 = acoustically grounded).
- Simple proxy: phoneme conditional entropy H(phoneme | LM context). Low entropy = high FAD bias.

### Step 2: Compute RAVEL metrics (mock)
For each phoneme p, sample a feature set F_p from the decoder representation.
- `Cause(k)`: binary mock — does zeroing feature k change the output phoneme? Estimate via: `Cause(k) = |P(output=p | feat_k=0) - P(output=p | feat_k=1)|`
- `Isolate(k)`: specificity — what fraction of other concepts (phonemes, words) also activate feature k? `Isolate(k) = 1 - |{p' ≠ p : feat_k activates for p'}| / |all phonemes|`

### Step 3: Correlation test
Predict: `FAD_bias_score ∝ Cause(k) * (1 - Isolate(k))`
- Pearson r between FAD score and `Cause - Isolate` gap
- Expected: r > 0.65 on mock data of N=40 phoneme features

### Step 4: Visualization
Scatter plot: x=Isolate, y=Cause, color=FAD score. Expected: high-FAD points cluster top-left (high Cause, low Isolate).
