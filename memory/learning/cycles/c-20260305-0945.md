## Hypothesis

**gc(k) confidence collapse is a reliable signature of jailbreak-susceptible processing in audio-LLMs.**

### Core Claim

When Qwen2-Audio (or similar audio-LLM) processes adversarial audio:
- Legitimate speech → gc(k) curve is steep near the listen layers (model commits early, confidence high)
- Jailbreak audio → gc(k) curve is **flat or collapses at the listen-layer boundary** (model stays in "guessing" mode longer, then snaps to a harmful output)

The "confidence collapse signature" = gc(k) drops at layers k ∈ [listen_start, listen_end], then suddenly recovers in later layers. This pattern indicates the safety-relevant content was NOT encoded in the listen layers — bypassing normal safety processing.

### Motivation (from T3 + T5 intersection)

- T3 established: listen layers ≠ guess layers; gc(k) quantifies where the model transitions
- T5 hypothesis: jailbreak audio exploits the guess phase — embeds harmful meaning in acoustic features that only resolve *after* the listen→guess boundary
- If gc(k) collapse correlates with jailbreak success rate → it's a detector

### Testable Prediction

| Condition | Expected gc(k) pattern |
|-----------|----------------------|
| Clean speech, benign intent | High confidence at listen layers (k=8-16 for Qwen2-Audio) |
| Adversarial audio, failed jailbreak | Moderate confidence dip, recovers early |
| Adversarial audio, successful jailbreak | Collapse in listen layers, anomalous late recovery |

**Measurable:** Kolmogorov-Smirnov test on gc(k) distribution between clean vs. adversarial sets.
