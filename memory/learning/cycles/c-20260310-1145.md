## Critical-Layer Patching Checklist for Listen-Layer Auditing in LALMs

A reproducible protocol for running causal patching experiments on audio-language models (ALM / LALM). Designed to audit "listen-layers" — where audio grounding (gc(k)) is processed or suppressed.

---

### Gate 0 — Prerequisite Checks

- [ ] Model runs inference on CPU or available GPU (≥8 GB VRAM for 7B, else use microgpt proxy)
- [ ] Hook library active (TransformerLens or custom `whisper_hook_demo.py`)
- [ ] Activation cache: clean + corrupted runs both stored for target token position
- [ ] Metric defined: gc(k) per layer OR token logit shift OR specific target feature activation

---

### Gate 1 — Patching Baseline (Direct Attribution)

Use **Attribution Patching (AtP)** as the fast baseline.

```python
# AtP: gradient × (activation_clean - activation_corrupt)
atP_score[layer] = grad_wrt_activation[layer] * (act_clean[layer] - act_corrupt[layer])
```

- [ ] Compute AtP scores for all layers + (optionally) heads/MLP neurons
- [ ] Rank layers by |atP_score|
- [ ] Flag top-k (k=3 default) as candidate critical layers
- ⚠️ AtP is an approximation — use as a filter, not a final answer

---

### Gate 2 — AND/OR Gate Patching (Heimersheim protocol)

For each candidate layer from Gate 1:

**AND gate test** (both paths required):
```
Run: patch layer L in clean model with corrupted activations
Measure: does metric collapse? → L is necessary
```

**OR gate test** (one path sufficient):
```
Run: patch layer L in corrupted model with clean activations
Measure: does metric recover? → L is sufficient
```

- [ ] AND test: corrupt clean→ check metric loss
- [ ] OR test: restore corrupt→ check metric recovery
- [ ] Record (AND_result, OR_result) per layer

| Result | Interpretation |
|--------|---------------|
| AND=yes, OR=yes | L is causally critical (necessary + sufficient) |
| AND=yes, OR=no | L is necessary but not alone sufficient |
| AND=no, OR=yes | L is sufficient but redundant (parallel path exists) |
| AND=no, OR=no | L not on critical path |

---

### Gate 3 — Denoising Pass (Activation Patching)

For layers where AND=yes:

- [ ] **Denoising patch**: start from corrupted run, restore layer L → track if gc(k) recovers toward clean level
- [ ] Verify: gc(k) post-patch ≥ 0.8 × gc(k)_clean → L is a key listen-layer
- [ ] Plot gc(k) recovery curve across layers (normalized 0→1)
- [ ] Identify "recovery elbow" — layer where most grounding is restored

---

### Gate 4 — AtP* (Corrected Attribution)

Run AtP* to remove AtP approximation errors:

```python
# AtP*: run patching at each layer, record direct logit/metric effect
atP_star[layer] = metric_clean - metric_after_patching_at_layer_L
```

- [ ] Compare AtP rankings with AtP* rankings
- [ ] Flag disagreements (>2 rank positions) as potential confounds
- [ ] Use AtP* rankings as final authority

---

### Gate 5 — Top-K Aggregate Score

Combine AND/OR + denoising + AtP* into a single leakage score:

```python
# Aggregate score per layer
score[L] = (
    0.4 * AND_score[L] +          # necessity
    0.3 * OR_score[L] +            # sufficiency  
    0.3 * (gc_recovery[L] / gc_clean)  # denoising recovery
)
```

- [ ] Rank all layers by aggregate score
- [ ] Top-3 = "Critical Listen-Layers" for this model + task
- [ ] Bottom 50% = "Pass-Through" layers (no causal audio role)

---

### Common Failure Modes & Fixes

| Issue | Symptom | Fix |
|-------|---------|-----|
| All layers show AND=yes | Distributed representation | Reduce k, use activation mean over window |
| gc(k) doesn't collapse | Metric insensitive | Switch metric (token logit / attention weight) |
| AtP vs AtP* major disagreement | Non-linear effects | Use AtP* only, flag as non-linear circuit |
| Recovery curve is flat | No single layer dominates | Try layer-pair patching |
