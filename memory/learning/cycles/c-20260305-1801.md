## SAEBench Metric Portability Matrix

**Source**: Karvonen, Nanda et al. (ICML 2025). 8 categories across single-prompt, multi-prompt, and absorption evaluations.

| # | SAEBench Metric | Text Assumption Being Ported | Audio Analogue (AudioSAEBench) | Port Complexity | Notes |
|---|----------------|-------------------------------|-------------------------------|-----------------|-------|
| M1 | **Reconstruction Fidelity** (L2 / cosine) | Token-level activations; assumes reconstruction at embedding dim | Frame-level (30ms) cosine similarity; identical math, different temporal granularity | 🟢 **LOW** | Direct port. No text-specific assumptions. Audio advantage: frame-rate provides more data points. |
| M2 | **L0 Sparsity** | Discrete token sequence; L0 = active features per token | Active features per 30ms frame; add phoneme-conditional sparsity (silence vs. complex clusters) | 🟢 **LOW** | Direct port. Audio adds phoneme-conditioning as bonus metric. |
| M3 | **Absorption Score** | Semantic features "absorb" another feature's meaning (text polysemy-driven) | gc(k) Absorption Test: does SAE feature F co-activate in high-gc(k) frames (audio-dominant)? | 🟡 **MEDIUM** | Port concept, not implementation. Text absorption = semantic; Audio absorption = channel-mixing (gc leakage). Requires gc(k) infrastructure. |
| M4 | **Monosemanticity / RAVEL Isolate** | Entity-attribute disentanglement in text tokens; RAVEL minimal pairs from text corpora | PCDS (Phoneme-Concept Decoupling Score) + Audio-RAVEL: phonological attribute isolation via MFA-aligned minimal pairs | 🔴 **HIGH** | Most complex port. Text RAVEL uses Wikipedia entity/attribute pairs. Audio needs MFA alignment + acoustic minimal pair corpora (Choi et al. 2602.18899 validated). Core contribution of AudioSAEBench. |
| M5 | **Interventional Selectivity** | Ablate feature → measure logprob delta on next-token prediction | Ablate feature → measure WER delta on Whisper CTC output; local vs. global WER selectivity ratio | 🟡 **MEDIUM** | Port is clean but WER = sequence-level metric, not token-level. Must localize to frames where feature was active. Reuses gc_eval.py hooks. |
| M6 | **Downstream Task Performance** | Probe SAE latents → predict GPT-style downstream NLP tasks | (a) Probe SAE latents → predict phoneme class; (b) SAE reconstruction WER vs. baseline Whisper | 🟢 **LOW** | Straightforward. Audio has better ground truth (MFA phoneme labels = gold standard; text labels are noisy). |
| M7 | **Spurious Correlation** | Feature activates on semantically unrelated tokens (e.g., "Paris" feature active on "Berlin") | Cross-phoneme activation overlap: χ² test over phoneme classes; acoustic similarity check between top-activated phonemes | 🟡 **MEDIUM** | Concept ports cleanly; operationalization differs. Need IPA/articulatory distance matrix (ready from Choi et al. datasets). |
| M8 | **Feature Geometry** (Superposition) | Cosine similarity structure; anti-podal features = binary semantic distinctions | Phonological Cluster Geometry: intra-class (bilabials, fricatives, etc.) vs. inter-class cosine similarity; voiced/unvoiced anti-podal features | 🟢 **LOW** | Audio has STRONGER theoretical prior for geometry: voiced/unvoiced = binary → expect antipodal; place of articulation = categorical → expect clusters. Should score BETTER than text. |

## Port Complexity Summary

| Category | Count | Metrics |
|----------|-------|---------|
| 🟢 LOW (direct port, ≤ 1 day) | 4 | M1, M2, M6, M8 |
| 🟡 MEDIUM (concept port, 2-3 days) | 3 | M3, M5, M7 |
| 🔴 HIGH (novel contribution, 1-2 weeks) | 1 | M4 |

**M4 is the KEY CONTRIBUTION** — all other metrics are either direct ports or natural adaptations. The AudioSAEBench paper's differentiating claim is that we define the audio-native disentanglement metric (M4/PCDS/Audio-RAVEL), which has no text equivalent for the audio domain.

## Critical Insight: Text → Audio Friction Sources

Three categories of friction when porting SAEBench to audio:

**Friction 1: Temporal Structure**
- Text SAEBench: 1 token = 1 activation vector = 1 "row"
- Audio SAEBench: 1 utterance = N frames (N ≈ 50-300); must aggregate OR do frame-wise; aggregation strategy affects results
- *Resolution*: M1-M3, M6-M8 do frame-wise; M4 does utterance-level minimal pair alignment; M5 does both

**Friction 2: Ground Truth Labels**
- Text SAEBench: uses Wikipedia entity/concept labels (sometimes noisy, crowdsourced)
- Audio SAEBench: uses MFA phoneme alignment (deterministic, millisecond-precision) → CLEANER ground truth
- *This is an advantage for audio, not a liability*

**Friction 3: Behavioral Signal**
- Text SAEBench: behavioral signal = next-token log probability (continuous, fast)
- Audio SAEBench: behavioral signal = Whisper WER (sequence-level, slower, needs chunking)
- *Resolution*: Use Whisper encoder CTC outputs for faster per-frame behavioral signal; full WER only for M5/M6
