## Topic: Counterfactual Layer-Attribution Protocol for ASR Token Errors

### 1. Core Idea

Standard gc(k) asks: **"How much does layer k contribute to audio grounding overall?"**

**Model Incrimination** asks a sharper question: **"When the model makes a specific ASR error on token t, which layer k is causally responsible for that error?"**

This is a **contrastive, error-conditioned** version of gc(k) — not just "where does audio matter?" but "where does audio get corrupted/ignored specifically when prediction fails?"

### 2. Motivation

Current gc_eval.py (Paper A harness) computes average gc(k) across all tokens in a stimulus pair. But ASR errors are not uniform:
- Some errors are **phoneme confusions** (e.g., /b/ → /p/, /m/ → /n/)
- Some errors are **semantic insertions/deletions** (model predicts plausible word, ignoring phoneme signal)
- Some errors are **prosody-induced** (rhythm/stress causes wrong segmentation)

**Key gap**: The current gc(k) framework cannot distinguish WHICH error type implicates WHICH layer. This design note specifies how to extend gc_eval.py to answer this.

### 3. Formal Protocol: Counterfactual Mode for gc_eval.py

**Input:**
- Two audio clips: `a_correct` (model produces correct token t*) and `a_error` (model produces wrong token t_err)
- Layer k to test
- Target token t* (the "should-have-been" prediction)

**Operation:**
1. Forward pass on `a_error` up to layer k → collect activation `h_k(a_error)`
2. Forward pass on `a_correct` up to layer k → collect activation `h_k(a_correct)`  
3. Patch: replace `h_k(a_error)` with `h_k(a_correct)` at layer k
4. Continue forward pass with patched activation
5. **Counterfactual success**: P(t* | patch at k) > P(t* | no patch)?

**Incrimination score** for layer k:
```
IncrimScore(k) = P(t* | patch@k) - P(t* | no patch)
```

A **high IncrimScore(k)** means: patching layer k with correct-audio activations *recovers* the correct prediction → layer k is where the error was introduced.

**Model Incrimination curve**: Plot IncrimScore(k) across all layers → the **peak layer is the incriminated layer**.

### 4. Relationship to gc(k)

| Metric | Question | Conditioned on |
|--------|----------|----------------|
| gc(k) (average) | "How much does audio grounding happen at layer k?" | All tokens in stimulus |
| gc(k) (per-token) | "Does this specific token rely on audio at layer k?" | One token |
| IncrimScore(k) | "Is layer k responsible for this specific error?" | Error tokens only |

**Key insight**: IncrimScore(k) is the ERROR-CONDITIONED version of per-token gc(k). When gc(k) is computed only at positions where the model was WRONG.

Formally: `IncrimScore(k) ≈ gc_token(k, t*, a_error→a_correct)`

### 5. Error Taxonomy × Predicted Incrimination Layers

Based on the 3-tier grounding failure taxonomy (Gap #31):

| Error Type | Expected Peak IncrimScore Layer | Mechanism |
|---|---|---|
| Phoneme confusion (b→p) | L_early (encoder, layers 1-12) | Acoustic distinction lost before connector |
| Semantic insertion | L_mid (connector ±2 layers) | Phoneme signal present but context dominates at connector |
| Prosody-induced segmentation | L_mid to L_late | Rhythm processed late; LM prior wins at upper layers |
| Hallucination (unrelated word) | L_late (LM layers, 70%+ depth) | Pure text prior; audio signal irrelevant |

**Falsifiable prediction**: If error is a phoneme confusion (/b/→/p/), peak IncrimScore should be in L_early (first 25% of layers). If error is a semantic hallucination, peak should be in L_late (last 25%).

### 6. Nanda Convergent-Evidence Connection

Nanda et al. (Mech Interp) emphasize that causal claims require **convergent evidence from multiple methods**: patching, probing, and ablation must agree before a layer is "incriminated."

Apply this principle to Model Incrimination:

**Three-way convergence test for layer k:**
1. **IncrimScore(k) > threshold** (patching recovers correct prediction)
2. **Probe accuracy**: Linear probe at layer k predicts correct phoneme t* better than t_err from `a_correct` activations
3. **Ablation delta**: Ablating (mean-centering) layer k outputs on `a_error` causes IncrimScore(k-1) to rise (upstream layer partially compensates)

Layer k is **strongly incriminated** only if all three signals converge. Single-method findings = **weakly incriminated** (hypothesis generating, not conclusive).

This maps directly to the three-tier taxonomy: Tier 1 (codec) errors should fail probe AND patching; Tier 2 (connector) errors should pass probe at encoder but fail at LM; Tier 3 (LLM) errors should pass probe everywhere but fail IncrimScore in L_early.

### 8. Three Open Questions

**Q1: Error pairing strategy**
How do we collect `(a_correct, a_error)` pairs without running Whisper on thousands of clips?
- Option A: Minimal pair synthesis (TTS: same word, slightly perturbed phoneme) — fully controlled
- Option B: Real ASR errors from LibriSpeech: collect clips where Whisper makes specific error types
- **Recommendation**: Option A for methodology validation; Option B for ecological validity
- **CPU-feasible**: Coqui TTS or gTTS on MacBook for Option A

**Q2: Token alignment across forward passes**
Whisper uses encoder-decoder attention; activations at layer k in the *encoder* may not correspond 1:1 to token positions in the *decoder*. How to handle this?
- For encoder-side incrimination: patch encoder layer k, measure decoder output P(t*)
- For decoder-side incrimination: patch cross-attention keys/values at layer k, measure P(t*)
- **Design decision**: default to encoder patching; add flag `--side encoder|decoder` for future extension

**Q3: Baseline instability (Activation-Patching pitfall #3)**
Heimersheim & Nanda warn: patching can create OOD distributions in later layers, inflating IncrimScore spuriously for early layers.
- **Mitigation**: Use the "noising" variant — patch `a_error` with *noise-level* corruption of `a_correct`, not the full swap. This is AtP (Attribution Patching) for audio.
- **Alternative**: Report both full-swap and AtP(audio) results; note discrepancy if large (>0.1)
- This is exactly the pitfall Risk A6 from Paper A methodology section → same mitigation applies here
