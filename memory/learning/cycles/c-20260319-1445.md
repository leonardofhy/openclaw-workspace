## Claim

Adversarial (backdoored) audio forces the decoder into a **high-cascade regime** (AND→OR gate shift, low GSAE density, high cascade_degree). This manifests as a **leftward shift in t\*** — the decoder collapses audio information earlier than on clean audio. t\* is therefore a **certified backdoor detector**.

---

## Mechanistic Chain

```
Backdoor trigger → encoder features biased (OR-gate dominated)
                 → GSAE density drops at early decoder steps
                 → collapse_onset_step (t*) shifts left (t* < 3)
Clean audio     → AND-gate features active longer
                 → GSAE density stable through layer k
                 → t* > 6 (late collapse)
```

**Unification (from Q113 + Q117):**
- cascade_degree = 1 − AND_gate_fraction = 1 − GSAE_density (normalized)
- Backdoor ↔ max(cascade_degree) at early k ↔ min(GSAE_density) shifts left
- All three metrics should give the same t\* estimate → **three-way certified detection**

---

## Detection Protocol

**Input**: Audio waveform `x`

**Step 1**: Run Whisper encoder → extract residual stream at each decoder step k

**Step 2**: Compute `Isolate_in(k)` per step (audio-to-residual mutual info proxy)

**Step 3**: Find `t* = argmin_k(Isolate_in(k))` — collapse onset step

**Step 4**: Decision rule:
  - `t* < threshold_low (default: 3)` → 🚨 BACKDOOR SUSPECTED
  - `t* ≥ threshold_high (default: 6)` → ✅ CLEAN
  - `threshold_low ≤ t* < threshold_high` → ⚠️ AMBIGUOUS (request second pass)

**Optional confirmation**: Compute AND_gate_fraction at t\*; backdoor samples should have AND_frac < 0.3.
