## Problem

How do Large Audio-Language Models (LALMs) internally perceive and recognize **auditory attributes** (speaker gender, emotion, language ID, animal sound type)?

## Method: Logit Lens on LALMs

- **Logit Lens**: project intermediate hidden state h_i^ℓ → vocabulary space via unembedding matrix W_U → get p_i^ℓ = softmax(W_U h_i^ℓ)
- **Layer-wise Information Score** I_i^ℓ: accuracy of argmax prediction of attribute label y from p_i^ℓ across dataset
- **Critical Layer** ℓ*_i: weighted average of layer indices, weighted by contribution s_i^ℓ = max(0, I_i^ℓ - (1+α)/|Y|), α=0.2 (threshold above chance)
- Trained on 3 SOTA LALMs (paper doesn't name in abstract; likely Qwen-Audio, Gemini-Audio, GPT-4o or similar)
- Attributes: speaker gender, emotion, spoken language, animal type

## Key Results

1. **Attribute info is NOT monotone with depth** — it often drops sharply at certain layers before recovering. Two patterns emerge:
   - Correct recognitions: info RISES with depth
   - Incorrect recognitions: info PEAKS midway then FALLS → prediction error
   
2. **Earlier resolution = higher accuracy**: negative correlation between critical layer ℓ* and prediction accuracy. Resolving attributes early gives more layers to refine → better accuracy.

3. **Token position finding** (KEY): Models rely heavily on **querying auditory input tokens** directly, NOT on aggregating info in hidden states at text-attribute-mentioning positions. Prior context ("what attribute is mentioned") is insufficient — models always go back to the audio.

4. **Improvement method**: Enrich deep-layer representations with earlier attribute-rich representations → **16.3% relative improvement** without training.

## New Gap Identified

**Gap #33: Logit Lens ≠ Causal — AudioLens finding #3 needs mechanistic verification**

AudioLens finds that LALMs "query auditory inputs directly" based on observational logit scores. BUT:
- Logit Lens is correlational — high p_i^ℓ(y) at audio token doesn't prove the audio token CAUSED the prediction
- A model could have high Logit Lens scores at audio tokens AND still be text-driven in its causal structure (if text pathway produces the same prediction)
- gc(k) = first causal test of AudioLens finding #3: does patching audio tokens causally shift the attribute prediction? Or does the text pathway dominate?
- Prediction: gc(k) peak at audio token positions, NOT at text positions → AudioLens finding #3 confirmed causally
- If causal test CONTRADICTS AudioLens → bombshell: models LOOK like they query audio but causally don't
