## Experiment Spec: Microgpt × Phonological DAS
### "The First Fully-Transparent Mechanistic Account of a Phoneme Circuit"

---

### Motivation

All current audio MI work operates on large, opaque models (Whisper-small = 244M params; Qwen2-Audio = 7B). This means:
- Circuit-level transparency is approximate (patching localizes, but can't fully enumerate the circuit)
- Verification requires statistical tests across many stimuli
- Training data is unknown → confounds are hard to isolate

**Key insight from ideate cycle #226**: MicroGPT + synthetic phoneme data gives us what no real model can — *ground truth training data, controllable circuit complexity, and perfect activation access.* A small transformer (2-4 layers, 64-128 dim) trained from scratch on a phoneme transcription task can serve as a *fully-inspectable* test bed for phonological DAS.

---

### Research Question

> Can we identify and verify a **complete phoneme circuit** — all heads, MLPs, and residual stream directions involved in voicing discrimination — in a minimal transformer, and do so via Distributed Alignment Search (DAS)?

This is a **feasibility proof** for the methodology in Paper A before applying it to Whisper.

---

### Setup

#### Task
Phoneme transcription: input = synthetic acoustic feature vector (hand-crafted, not waveform), output = phoneme class label.

**Minimal phoneme set** (to keep model small):
- 6 phoneme contrasts differing only in voicing: {p/b, t/d, k/g}
- Input features: 8-dimensional vector encoding:
  - F0 presence (voicing: 0.0=unvoiced, 1.0=voiced)
  - VOT duration (ms, normalized)
  - Burst amplitude
  - 5 spectral features (F1-F3 formant, spectral tilt, HNR)

#### Model
- MicroGPT: 2-layer transformer, 64-hidden-dim, 4 attention heads, no positional encoding (single-token input)
- Trained on 10,000 synthetic samples (balanced across 6 phoneme classes)
- Training objective: cross-entropy on phoneme class prediction
- Target: ≥95% accuracy (achievable given clean synthetic features)

#### Synthetic Data Generation (Tier 0, pure Python, no GPU)

```python
import numpy as np

PHONEME_SPECS = {
    'p': {'voiced': 0.0, 'vot': 0.85, 'burst': 0.9, 'place': 'bilabial'},
    'b': {'voiced': 1.0, 'vot': 0.15, 'burst': 0.7, 'place': 'bilabial'},
    't': {'voiced': 0.0, 'vot': 0.90, 'burst': 0.85, 'place': 'alveolar'},
    'd': {'voiced': 1.0, 'vot': 0.10, 'burst': 0.65, 'place': 'alveolar'},
    'k': {'voiced': 0.0, 'vot': 0.80, 'burst': 0.80, 'place': 'velar'},
    'g': {'voiced': 1.0, 'vot': 0.20, 'burst': 0.60, 'place': 'velar'},
}

def generate_sample(phoneme, noise_std=0.05):
    spec = PHONEME_SPECS[phoneme]
    return np.array([
        spec['voiced'] + np.random.normal(0, noise_std),
        spec['vot'] + np.random.normal(0, noise_std),
        spec['burst'] + np.random.normal(0, noise_std),
        # 5 spectral features (place-correlated, voiced-correlated)
        ...
    ])
```

### DAS Protocol

**Alignment target**: voicing subspace in residual stream

1. **Identify candidate subspace**: 
   - Run activation patching (clean=voiced, corrupt=unvoiced) across all residual stream positions
   - Rank positions by patching effect size on voicing discrimination accuracy
   - Expected: Layer 1 attention heads (voicing detection) and Layer 2 MLP (classification)

2. **DAS on voicing dimension**:
   - Rotation matrix R maps model's residual stream to a 1D voicing axis
   - Intervention: rotate the voiced feature of stimulus A into unvoiced stimulus B's representation
   - Measure: IIA (Interchange Intervention Accuracy) — does the model now predict the voiced phoneme from B's context?
   - Linear constraint enforced (per Sutter et al. 2507.08802 — non-linear DAS would be vacuous)

3. **Verify circuit completeness**:
   - After finding voicing subspace: test isolation (RAVEL-style)
   - Does patching voicing dimension NOT affect place-of-articulation prediction? (bilabial/alveolar/velar)
   - Expected: Cause(voicing_subspace, voiced_label) ≈ 0.9, Isolate(voicing_subspace, place_label) ≈ 0.85

4. **Enumerate full circuit**:
   - Use path patching (Goldowsky-Dill 2023) to enumerate: which attention heads copy voicing feature? Which MLPs write to voicing subspace?
   - Full circuit graph producible because model is only 2 layers

---

### Expected Results

| Metric | Expected Value | Significance |
|--------|---------------|-------------|
| DAS IIA (voicing) | >0.85 | Voicing subspace is linear and localizable |
| Cause score (RAVEL-style) | >0.8 | Voicing subspace causally determines label |
| Isolate score (RAVEL-style) | >0.7 | Voicing subspace doesn't bleed into place |
| Circuit enumeration completeness | 100% | Only possible in 2-layer model |

**If IIA < 0.6**: voicing representation is non-linear in even this minimal model → critical finding for Paper A methodology (validates Sutter et al. concern earlier than expected).
