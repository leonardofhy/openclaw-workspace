## Objective

Produce a formal causal scorecard that characterizes how emotion and speaker identity
attributes leak into each other in Whisper/audio SAE features — structured as RAVEL-style
Cause + Isolate metrics under a 2×2 condition matrix.

## Background: Audio-RAVEL Methodology

RAVEL (Huang et al. ACL 2024) defines:
- **Cause(F, A)**: when you patch feature F → does attribute A change as expected? (localization)
- **Isolate(F, A)**: when you patch feature F → do OTHER attributes remain unchanged? (isolation)
- Combined metric: `RAVEL_score(F, A) = harmonic_mean(Cause(F, A), Isolate(F, A))`

Audio-RAVEL (Gap #23, Paper B Category 0) adapts this to:
- Entity → audio stimulus (minimal pair speech clips)
- Attribute → phonological/paralinguistic feature (voicing, speaker-gender, emotion-valence, etc.)
- Intervention → AudioSAE feature patch (steering activation)

## ESN Leakage Hypothesis

The Emotion Safety Network (ESN) finding (Zhao et al., from Gap #20 / Feng et al.):
- Emotion intensity modulates unsafe response rate non-monotonically
- Medium-intensity emotion → HIGHEST jailbreak risk
- Hypothesis: emotion features in audio SAE LEAK into speaker-identity features and vice versa
  because acoustic co-occurrence in training corpora correlates emotion × speaker attributes
  (e.g., female speakers more frequently labeled "excited", angry speech = mostly male in datasets)

**Key leakage question**: Is Feng et al.'s medium-intensity effect a feature-level leakage problem?
If emotion-valence SAE features have LOW Isolate(F, emotion) → they also encode speaker-gender
→ the model's "emotion detection circuit" is really an "emotion+speaker combo detector"
→ patching it for safety reasons will inadvertently patch speaker-identity features too
→ SAE-guided defense (Gap #24) has COLLATERAL DAMAGE from leakage

## 2×2 Condition Matrix

| Condition | Emotion | Speaker-Gender | Stimulus Type |
|-----------|---------|---------------|---------------|
| EE+SN | Emotion varies | Speaker neutral | Tone-normalized TTS (same voice, emotion modulated) |

- EE = emotion excited, EN = emotion neutral, SE = speaker excited, SN = speaker neutral
- (re-labeling to E=emotion-variation, S=speaker-variation)

## Formal Scorecard

### Define features to test
For Whisper-small on the AudioSAE pretrained checkpoint:
- F_emo: top-k SAE features with highest differential activation for emotion-loaded vs neutral stimuli
- F_spk: top-k SAE features with highest differential activation across speaker identities

### Per-condition Cause/Isolate

**Condition A — Isolate test for F_emo**
Patch F_emo (target: change emotion attribute) and measure:
- Cause(F_emo, emotion): did emotion label flip? [probe classification accuracy before/after patch]
- Isolate(F_emo, emotion): did speaker-identity label remain? [speaker probe accuracy after patch]

**Condition B — Isolate test for F_spk**  
Patch F_spk (target: change speaker attribute) and measure:
- Cause(F_spk, speaker): did speaker-identity probe output change?
- Isolate(F_spk, speaker): did emotion-valence label remain?

**Condition C — Cross-contamination test (EE+SE condition)**
Use natural co-occurring stimuli:
- Can we separate F_emo and F_spk in the joint distribution?
- Measure: `LeakageScore(emo→spk) = 1 - Isolate(F_emo, emotion)` in EE+SE condition
  vs EE+SN condition (where speaker is controlled)

**Predicted key finding**: F_combo class will dominate in EE+SE condition.
These are features that score HIGH on Cause for BOTH emotion and speaker → genuine entanglement.
`EntanglementFraction = |{F: Cause(F, emo) > 0.5 AND Cause(F, spk) > 0.5}| / |total features|`

### Predicted ESN leakage signature

If Feng et al. medium-intensity effect is feature-level leakage:
- `LeakageScore(emo→spk)` will be HIGH at medium intensity (0.5-0.7 on 0-1 scale)
- `LeakageScore(emo→spk)` will be LOWER at high/low intensity (cleaner signal separation)
- Mechanism: medium emotion = ambiguous activation → more SAE features ambiguously activated
  → higher contamination of speaker features → safety defense patches speaker features → collateral damage

This is falsifiable with existing AudioSAE cross-seed checkpoints + Feng et al. stimuli.

## Connection to Gap #24 (SAE-guided defense)

Collateral damage estimate for SPIRIT-style patching with leaky SAE features:
```
CollateralDamage(defense) ≈ (1 - Isolate(F_emo, emotion)) × SpeakerInfoDensity(F_emo)
```
If `LeakageScore = 0.47` (i.e., Isolate = 0.53) and SpeakerInfoDensity = 0.8 →
CollateralDamage ≈ 0.38 → 38% speaker-identity information disrupted per defense patch

This motivates the Paper B contribution: **measure leakage BEFORE building defenses**.
Defense strategy should be: patch only Isolate(F) > 0.7 features → low-leakage defense.
