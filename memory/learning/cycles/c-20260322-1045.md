## Problem

FAD (Forced Attention Distraction) bias causes ASR models to rely on text-predictable context (OR-gate path) even when acoustic evidence is available. This manifests as:
- High-frequency phonemes receive less audio attention (OR-gate dominant)
- Low-PPL (predictable) phonemes show AND-frac < 0.3 (text-driven)
- Result: transcription errors on acoustically clear but contextually ambiguous speech

**Hypothesis**: If we can artificially boost AND-gate features at the gc(k) peak layer for text-predictable (low-PPL) phonemes, the model will attend more to audio evidence → WER reduction.

---

## Method: AND-Gate Steering Intervention

### Step 1: Identify Steering Targets

For each utterance:
1. Compute per-phoneme LM perplexity (PPL) using a text-only LM (e.g., GPT-2)
2. Low-PPL phonemes (PPL < threshold θ_PPL ≈ 5.0) = FAD-bias candidates
3. At decoder step t for phoneme p, retrieve gc(k) peak layer: `k* = argmax_k gc(k, t)`

### Step 2: AND-Gate Feature Identification

At layer k*, classify each SAE feature as AND/OR gate:
- AND-gate: activation depends on both audio + text context (AND-frac > 0.5)
- OR-gate: activation driven by text alone (AND-frac < 0.3)

**AND-gate feature set**: `F_AND(k*) = {f : AND-frac(f, k*) > 0.5}`

### Step 3: Steering Intervention (CPU-only)

At forward pass, for FAD-biased phoneme positions:
```python
def steer_and_gate(residual, sae, F_AND, alpha=1.5):
    """
    Boost AND-gate features at gc peak layer.
    residual: (batch, seq, d_model)
    alpha: steering strength (1.0 = no change, >1.0 = boost)
    """
    # Decompose via SAE
    features = sae.encode(residual)  # (batch, seq, n_features)
    
    # Boost AND-gate features selectively
    features[:, :, F_AND] *= alpha
    
    # Reconstruct
    steered = sae.decode(features)
    return steered
```

**Key constraint**: Apply steering ONLY at k* layer, ONLY for low-PPL phoneme positions. No global boosting.

### Step 4: Expected Effect

| Condition | AND-frac before | AND-frac after | Expected WER |
|-----------|----------------|----------------|--------------|
| Clean (high-PPL) | 0.72 | 0.72 (unchanged) | baseline |
| FAD-biased (low-PPL) | 0.28 | ~0.55 (steered) | -5 to -15% |
| Silence/noise | 0.29 | 0.50 (steered) | neutral (can't help no signal) |

## CPU Mock Experiment Spec

```python
# mock: fad_and_steering_mock.py
import numpy as np
from scipy.stats import pearsonr

N_PHONEMES = 200
np.random.seed(42)

# Ground truth: low-PPL → high FAD bias (low AND-frac)
ppl = np.random.exponential(scale=8, size=N_PHONEMES)  # Gamma-like distribution
and_frac_baseline = 0.3 + 0.6 * (1 - np.exp(-ppl / 10))  # Low PPL → low AND-frac

# Simulate steering: boost AND-frac for low-PPL phonemes
alpha = 1.5
theta_ppl = 5.0
and_frac_steered = and_frac_baseline.copy()
target_mask = ppl < theta_ppl
and_frac_steered[target_mask] = np.minimum(
    1.0, and_frac_baseline[target_mask] * alpha
)

# WER proxy: higher AND-frac → lower WER (audio-reliant)
wer_baseline = 0.5 * np.exp(-2 * and_frac_baseline) + np.random.normal(0, 0.02, N_PHONEMES)
wer_steered = 0.5 * np.exp(-2 * and_frac_steered) + np.random.normal(0, 0.02, N_PHONEMES)

# Expected outcomes
delta_wer = wer_steered[target_mask] - wer_baseline[target_mask]
print(f"Mean WER reduction for FAD-biased phonemes: {delta_wer.mean():.3f}")
print(f"AND-frac improvement: {and_frac_baseline[target_mask].mean():.3f} → {and_frac_steered[target_mask].mean():.3f}")
r, p = pearsonr(and_frac_steered - and_frac_baseline, wer_baseline - wer_steered)
print(f"Pearson r(ΔAND-frac, ΔWER_reduction): {r:.3f} (p={p:.3e})")
```

**Expected output**:
- AND-frac improvement: ~0.28 → ~0.42 for FAD-biased phonemes
- Mean WER reduction: 3-8% on targeted phonemes
- Pearson r(Δ AND-frac, Δ WER reduction): > 0.7

## Open Questions

2. Does steering at k* also propagate to k*+1, k*+2 layers? (may be self-reinforcing)
3. Can we select F_AND(k*) offline (precomputed) to make steering CPU-efficient?
