## SPIRIT × Audio-RAVEL Comparison

### SPIRIT (arXiv 2025 — jailbreak detection via sparse feature detection)
- **Core mechanism**: Identify a sparse set of "safety-relevant" neurons; monitor their activation pattern for adversarial inputs
- **Evaluation metric**: AUC on held-out attack vs. clean audio distribution
- **What it measures**: Whether a feature (neuron/SAE component) FIRES for adversarial vs benign inputs — i.e., Cause(F, adversarial_attribute)
- **What it DOESN'T measure**: Whether F fires for ONLY adversarial inputs (Isolate dimension absent)

### RAVEL Isolate (Huang et al. 2024)
- **Core mechanism**: After patching feature F to convey attribute A, check if attribute B (non-target) also changes
- **Evaluation metric**: 1 - leakage(B) for each non-target B
- **What it measures**: Orthogonality / specificity of features

### Key Insight (NEW): SPIRIT Cause = Audio-RAVEL Cause; SPIRIT lacks Audio-RAVEL Isolate

| Dimension | SPIRIT | Audio-RAVEL | Gap |
|-----------|--------|-------------|-----|
| Feature set | Neurons / SAE components | SAE features | Same |
| Cause measurement | ✅ AUC(adversarial) | ✅ patch → attribute change | Equivalent |
| Isolate measurement | ❌ Not measured | ✅ B-attribute leakage | RAVEL adds this |
| Multi-attribute control | ❌ Binary (attack/no-attack) | ✅ 2×2 emotion-speaker | RAVEL adds this |

**Synthesis for Paper B (AudioSAEBench):**
- SPIRIT represents the state of the art in audio jailbreak detection using features
- Adding Isolate scoring to SPIRIT-style detectors reveals "precision" of safety features: does the safety neuron fire specifically for *jailbreaks*, or does it also fire for *emotionally intense* but benign content?
- AudioSAEBench Audio-RAVEL is the first benchmark to make Isolate mandatory alongside Cause

## Refined Q040 Causal Scorecard Protocol

### Two-Layer Design (Tier 0 NOW + Tier 1 when unblocked)

#### Layer A — ESN × Emotion-Speaker 2×2 (from c-20260305-1301)
Already designed. Summarized:
- 2×2: [same-speaker, cross-speaker] × [emotion swap, emotion preserve]
- Cause(F_ESN, emotion) measures driving power
- Isolate(F_ESN, emotion) measures leakage into gender/speaker

#### Layer B — SPIRIT Features × Jailbreak-Prosody 2×2 (NEW, from Q041/Q066)

SPIRIT-identified safety features should be evaluated the same way:
- **Target attribute**: "audio jailbreak / adversarial instruction" (binary)
- **Leakage attributes**: Angry tone (prosodic), high F0 (intensity), non-native accent

**Prediction**: SPIRIT safety features will show:
- Cause(F_safety, jailbreak) ≈ 0.75 (high — SPIRIT is designed for this)
- Isolate(F_safety, jailbreak) ≈ 0.35 (low — angry legit audio will also trigger)

This creates a **false positive** problem: a user asking angrily about a legitimate topic may trip the SPIRIT detector. Audio-RAVEL Isolate quantifies this exactly.

#### 2×2 Stimulus Matrix (Layer B)

| Adversarial | Benign |
|-------------|--------|
| Angry-toned jailbreak | Angry-toned benign request |
| Neutral-toned jailbreak | Neutral-toned benign request |

→ Rows: instruction type (adversarial/benign)
→ Cols: prosodic style (angry/neutral)

If Isolate is low: SPIRIT fires in column 1 regardless of row → high false positive on angry benign.
If Isolate is high: SPIRIT correctly distinguishes rows within each column.

**Expected result**: Isolate(F_safety, jailbreak | angry prosody) ≈ 0.28 → significant false positive risk for emotionally-stylized speech.

## New Gap Identified: Gap #34

**"Safety feature detectors (SPIRIT, Q-BERT audio) are calibrated for Cause but not Isolate — no paper quantifies false positive rate due to prosodic confounds"**

- Text jailbreak detection: PAIR, GCG, etc. — prosodic confounds don't exist in text
- Audio jailbreak detection: angry/intense legitimate requests use overlapping acoustic features with adversarial audio
- **Gap**: No paper measures Isolate(safety_features, jailbreak) while controlling for prosodic style
- **Paper B placement**: AudioSAEBench motivation — "current audio safety detectors have unknown isolation quality; our Cause × Isolate benchmark reveals the prosodic leakage problem"
- **Priority**: 🟢 GREEN — directly motivates AudioSAEBench and Q040 experiments; cite in Paper B §1 + §3.2
