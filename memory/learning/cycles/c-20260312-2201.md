## What was built

`skills/autodidact/scripts/and_or_gc_patching_mock.py`

Implements the **AND/OR Gate × gc(k) denoising patching protocol** (Tier 0 mock):

**Three-run patching protocol per feature:**
1. `clean` run: audio + context both present
2. `noisy` run: audio replaced with Gaussian noise
3. `patched` run: clean audio re-injected at feature while noise persists elsewhere

**Gate classification logic:**
- **AND-gate**: clean active → drops in noise → RECOVERS when audio patched back
  → feature requires audio (conjunctive; conservative integrator)
- **OR-gate**: clean active → STAYS active in noise (context substitutes)
  → feature doesn't need audio (disjunctive; can "guess" from context)
- **Passthrough**: clean active → drops in noise → does NOT recover after patch
  → audio-correlated but not causally audio-gated
- **Silent**: inactive in clean run

## Interpretation

AND-gate dominance at gc peak confirms: the cross-modal integration layer acts as a
**conjunctive gating mechanism** — it fires only when audio evidence is causally
available and needed. This is *not* OR-gate substitution (guessing from context).

This maps directly to Paper A's "Listen vs Guess" framing:
- **Listening** (AND layer): requires audio, confirmed by patching
- **Guessing** (OR layer): context substitution active, audio not essential

## Paper A Extension (§5.5 addition)

The AND-gate fraction at each layer is a new proxy metric for gc(k) localization:
```
gc_proxy(k) = fraction of SAE features at layer k that are AND-classified
```
This can be measured WITHOUT activation patching experiments — just the 3-run
denoising protocol (clean/noisy/patched). Lower compute cost than full gc(k).
