## Problem
Can governments impose communication restrictions on AI data centers to slow training without disrupting inference? Specifically, can "inference verification" work as a governance tool?

## Method (governance mechanisms analyzed)
1. **Disconnect inter-rack cables** (InfiniBand → copper): reduce inter-rack bandwidth from 400 GB/s to ~100 MB/s
2. **Tap-verified bandwidth limits** (~100 KB/s per rack): cheap devices on cables verify data rate ≤ inference needs
3. **Output re-computation**: verifiers re-run sampled I/O pairs to confirm declared model is running (zero-trust communication check)
4. **Memory wipes** every few minutes: erase checkpoints before they can be exfiltrated
5. **Proof of work / memory**: cryptographic proof that >95% of compute is doing inference (not training)

## Key Findings
- **Current prototypes likely ineffective**: RL training is mostly inference (rollout generation). Developers could use ~95% of verified compute for RL rollouts and route only gradient updates through 5% covert compute → training continues.
- **Slack in current proposals**: covert channel bandwidth (>0.01%) + checkpoint upload window + lack of PoW/PoM coverage leave too much wiggle room.
- **Three measures that *would* work** (any one sufficient):
  1. Proof of work/memory covering >95% compute
  2. Memory wipes every few minutes (erase covert checkpoints)
  3. Output recomputation reducing covert channel to <0.01%
- None of these has been prototyped robustly yet.
- Author estimates: with aggressive measures + 1yr lead time → buy **≥1 year** of slowdown (absent intelligence explosion).
- Intelligence explosion scenario: unclear if bandwidth limits would slow AI-accelerated algorithmic R&D.

## Connection to T5 (Audio Jailbreak / Safety)
- Governance angle: if compute verification becomes real, **safety probes must work at inference time** (not just training-time). Audio jailbreak detection running as inference-time verification aligns with this constraint model.
- "Inference is the allowed mode" → safety interventions baked into inference (our direction) would survive compute governance regimes.
- Indirect relevance: if AI training slows but inference expands, audio-LLM safety becomes more pressing (more inference-only deployments).
