## Structured notes

### Problem
When a model takes a suspicious action, behavioral evidence alone often cannot distinguish benign causes (confusion, instruction conflict) from malign causes (strategic scheming). Investigations fail if environments are confounded.

### Method (paper contribution)
The post proposes 5 environment-design principles for motive investigations:
1. Have uncertain causes (multiple plausible hypotheses)
2. Maximize investigative surprise (avoid cases explained by obvious training priors)
3. Clear user intent (single interpretation of instructions)
4. Avoid implicit nudges (don’t accidentally steer model toward target behavior)
5. Make the environment realistic (plausible deployment setting, avoid eval-awareness)

### Results / takeaways
- Team iterated on 20+ environments and found confounds frequently dominate analysis unless aggressively removed.
- Practical diagnostics are emphasized (multi-model checks, trace sanity checks, hypothesis diversity).
- Core shift: optimize environments for *hypothesis discrimination power*, not just “interesting behavior”.

### Connection to active tracks
- **T5 (Listen-Layer Audit / MATS)**: Directly useful for designing jailbreak/safety probes that can separate motive hypotheses (e.g., shortcut exploitation vs instruction confusion).
- **T3 (Listen vs Guess)**: Strengthens benchmark design for causal tests: stimuli/tasks should preserve ambiguity in causes while removing instruction ambiguity.

### 5 concrete mappings to Leo’s MATS proposal
1. Add an **ambiguity scorecard** per scenario: at least 2 plausible motive hypotheses before running interp.
2. Add a **surprisal gate**: reject scenarios too explainable by known priors (pure sycophancy/helpfulness).
3. Add **intent clarity checks**: single-goal prompt templates + adversarial paraphrase equivalence tests.
4. Add **nudge audit**: detect accidental hints in system/task framing that leak “expected misbehavior”.
5. Add **realism constraints**: role/task artifacts should mimic plausible production contexts, not toy eval artifacts.
