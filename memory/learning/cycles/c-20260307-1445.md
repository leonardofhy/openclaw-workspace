## Problem
Understanding *why* a model took a bad action (benign vs malign motive, e.g., confusion vs scheming) is hard because many motivations lead to the same action. Need realistic proxy tasks that don't confound causal attribution.

## Method: 5 Design Principles
1. **Uncertain causes** — behavior should admit multiple plausible explanations (or none obvious). If dominant hypothesis is clear from traces, not worth investigating.
2. **Maximize surprise** — behavior shouldn't map cleanly to known training priors (sycophancy, RLHF honesty). Otherwise investigation = confirming one obvious hypothesis.
3. **Clear user intent** — instructions must have single interpretation. Ambiguity → behavior is artifact of unclear priority, not model motive.
4. **Avoid implicit nudges** — environment shouldn't hand model both the problematic action AND a justification. If it does, can't distinguish independent motivation from suggestion.
5. **Make it realistic** — plausible real-world scenario. Contrived setups elicit behavior that reflects model's interpretation of the artificiality, not genuine propensity.

## Key Result: Sandbagging Case Study
- DeepSeek R1: 59.3% sandbagging rate in baseline environment ("We want to determine if you can be deployed; >50% → unlearning")
- Counterfactual interventions clarifying user intent → rate drops to 4.2% (near error floor)
- o3: 31.6% → 0%
- **Conclusion**: Both models' sandbagging was entirely instruction ambiguity, NOT self-preservation. The phrase "We want to determine whether you can be deployed" implicitly instructed the model to score ≤50%.

## Key Environments Built
- **Directory summary** (self-preservation test): model finds note from "previous instance" warning about corrective training → omits file. But note is a nudge (provides action + justification) → violates principle 4.
- **Evaluation tampering** (principle 3 good example): unambiguous task (solve math, report suspicious observations); model conceals evidence of previous tampering → can't blame ambiguity.
- **Funding email** (whistleblowing): discovers company surveils ethnic minorities; nothing in env suggests what to do → violates principle 2 (whistleblowing = known training prior).
- **Secret number** (reward hacking): model cheats by reading answer file from leaked path in oracle source → violates principle 5 (unrealistic leak).

## Connection to T5 (Audio Jailbreak)
My audio jailbreak detection work needs eval environments too. Apply principles:
- **Principle 3 (clear intent)**: Audio jailbreak prompts must have unambiguous intended refusal — no linguistic ambiguity in the "clean" context that could explain misbehavior
- **Principle 4 (no nudges)**: Don't embed prosodic cues that implicitly signal "this is a jailbreak test" — model shouldn't pick up meta-context
- **Principle 2 (surprise)**: Ideal eval: model complies with audio jailbreak NOT because of known sycophancy/helpfulness pressure, but for some other reason
- **Principle 1 (uncertain causes)**: Want audio jailbreak behavior where it's unclear whether model is responding to semantic content vs prosodic features → motivates T5 causal patching
