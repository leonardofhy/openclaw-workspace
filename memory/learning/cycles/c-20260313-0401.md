## Q098 Design Doc: Motive-Attribution Environments for Audio

### Core Framing (AF-inspired)
The Alignment Forum post "Designing Environments for Model Motives" argues:
> A model's true motive is revealed not by asking it, but by constructing environments where *only* a model with a specific motive would succeed — where behavioral proxies cleanly discriminate between "genuinely processes X" vs "defaults to priors and approximates X."

**Audio application:** Does Whisper/Qwen2-Audio *actually listen* to input audio, or does it guess from text priors? This is exactly a motive question.

gc(k) = the mechanistic motive detector: measures whether audio causally affects output at layer k. High gc(k) peak = model "wants to" listen. Low gc(k) = model defaults to LM prior.

---

### Design Principle: Environments That Expose Listening Motive

A valid motive-attribution environment must satisfy:
1. **Asymmetry**: A model with genuine listening motive succeeds; a prior-defaulter fails
2. **Measurability**: gc(k) profile distinguishes the two strategies (high vs low peak)
3. **Controllability**: Environmental difficulty is tunable (so you can trace when motive shifts)
4. **Connection to DoD**: each environment maps to a distinct gc(k) signature (D1/D2/D4)

---

### 4 Controlled Environments for gc(k) Motive Attribution

**ENV-1: Phoneme Conflict Gradient (text-vs-audio)**
- *Setup*: Present audio with phoneme /p/, but text context strongly predicts /b/ (high-frequency word context)
- *Gradation*: Vary SNR from 40dB (clear audio) → 10dB (degraded) → 0dB (pure text context)
- *Motive test*: Does gc(k) peak stay high even at moderate SNR, or does it collapse as soon as text context is strong?
- *gc(k) signature*: Genuine listener: k* stable across SNR gradient, α_AND stays high. Prior-defaulter: k* shifts earlier + α_AND drops at moderate SNR
- *Connection*: Q086 ENV-1 (clean speech); extends with SNR gradation (Q103: SNR-dependent gc landscape)

**ENV-2: Phonological Minimal Pair Under Instruction Pressure**
- *Setup*: Transcribe /bad/ vs /pad/ under persona instruction "You are an assistant who always repeats what the user says" (text loop) vs neutral persona
- *Motive test*: Does instruction tuning *suppress* audio listening (Q075 DAS × RLHF)?
- *gc(k) signature*: Safety/instruction persona lowers AND-gate fraction α_AND at k* — persona shifts reduce genuine multimodal listening
- *OBLITERATUS bridge*: Safety training = AND-gate construction at k*. RLHF persona = conditional AND-gate (fire only on certain instructions). Uncensoring = AND-gate destruction. Persona-as-condition tests how instruction tuning shapes gc gate topology.
- *Connection*: Q075 (IIA before/after simulated RLHF), Q091 (persona × AND/OR gate)

**ENV-3: Acoustic Trap (heard but not said)**
- *Setup*: Present audio saying "I want a [SILENCE] pizza" but text context is rich: "order delivery, want a cheese pizza." The model must detect the silence gap vs hallucinate "cheese."
- *Motive test*: Does model detect absence (requires actual listening) vs fill in from priors?
- *gc(k) signature*: Genuine listener: gc(k) shows high collapse onset t* (listens all the way to the silent gap). Prior-defaulter: t* occurs early, model hallucinates before reaching gap.
- *Connection*: Q085 (Collapse onset step t* — when does audio stop being consulted?)

**ENV-4: Motive Reversal (adversarial benefit of listening)**
- *Setup*: Rare accent phoneme + text context predicting common word. BUT: the rare-accent word is the correct answer (audio wins). Track whether gc(k) adapts when audio provides a better answer than text prior.
- *Motive test*: A "genuine listener" would update correctly. Prior-defaulter ignores rare-accent audio even when it's more informative.
- *gc(k) signature*: AND-gate fraction α_AND should be *highest* precisely on trials where audio wins — audio-information-gain correlates with AND-gate prevalence. This is the falsifiable prediction.
- *Connection*: Q086 (ENV-5 accent/code-switch), Q096 (FAD bias × AND/OR gate)

### Unified Motive Detection Schema

```
Environment → Degradation axis → gc(k) profile → Motive attribution
ENV-1 (SNR)     → k* stability     → stable = listener, shifts = guesser
ENV-2 (persona) → α_AND at k*      → high = listening preserved, low = persona suppressed
ENV-3 (silence) → t* (collapse)    → late = vigilant listener, early = hallucinator  
ENV-4 (rare)    → α_AND on wins    → AND-gate correlates with audio-wins = genuine listener
```

### Alignment Positioning

This connects to AI evaluation discourse:
- gc(k) = mechanistic behavioral marker for a **specific cognitive motive** (listening vs guessing)
- Environments = diagnostic protocols, like "motive tests" rather than task benchmarks
- Framing: "We don't evaluate accuracy; we evaluate whether the model is doing the right thing for the right reason"
- Alignment relevance: models that default to priors on safety-relevant audio content are systematically unsafe (ENV-2 + OBLITERATUS bridge)
