## Source: Heimersheim & Nanda 2024, "How to use and interpret activation patching"

### Core Insight: Patching is Ambiguous — Gate Direction Matters

Naive activation patching ("swap activations from clean→corrupted run") has a hidden problem: it conflates **necessity** and **sufficiency**. Heimersheim & Nanda systematize this via four gate modes:

| Gate | Operation | Interpretation |
|------|-----------|----------------|
| **AND** (noising) | Clean model, corrupt ONE component | "Is this component *necessary* for correct output?" High effect = yes |
| **OR** (denoising) | Corrupted model, restore ONE component | "Is this component *sufficient* to rescue output?" High effect = yes |
| **Canonical (corrupt→clean)** | Corrupted baseline, restore component | Mixed: necessity under corrupted context |
| **Inverse (clean→corrupt)** | Clean baseline, corrupt component | Mixed: sufficiency under clean context |

**Key finding:** AND and OR can disagree sharply. A component can be necessary (AND=high) but not sufficient (OR=low), meaning it's part of a chain, not a bottleneck.

### Attribution Patching (AtP) — Linearized Version

Full patching is O(N²). AtP uses gradient-weighted activation differences:
```
AtP(component) = (∂L/∂a) · (a_clean - a_corrupt)
```
Fast (one backward pass) but **underestimates indirect effects** (gradient doesn't see downstream). AtP* adds a correction for attention indirect paths.

### Denoising vs Noising for gc(k) — Key Design Choice

**gc(k) measures:** "Does layer k ground audio signal in final prediction?"

Existing gc(k) uses clean→corrupt patching (replace layer k activation). This measures: "Is layer k necessary for the model to use audio?"

**Heimersheim upgrade for gc(k):**

1. **AND-gate gc(k)** (recommended default):
   - Baseline: clean speech input
   - Corrupt one layer's activation (with noise or from wrong-text run)
   - Measure output logit change
   - Interpretation: "Layer k is necessary for audio grounding"
   - Expected result: mid-layers should show high AND-gate gc(k) (listen layer)

2. **OR-gate gc(k)**:
   - Baseline: corrupted input (noise/text-only)
   - Restore layer k from clean run
   - Measure recovery
   - Interpretation: "Layer k alone can rescue grounding"
   - Expected result: late-fusion layers might show high OR (can compensate)

3. **Dual-gate agreement zone:**
   - High AND AND high OR = true bottleneck layer ("the" listen layer)
   - High AND, low OR = necessary but not sufficient (upstream dependency)
   - Low AND, high OR = sufficient but not necessary (parallel pathway)
   - Low AND, low OR = irrelevant layer

### Top-k Attribution Filtering

Standard gc(k) aggregates across all positions. Better: compute AtP per (layer, head/neuron) and take top-k components. This prevents "noisy layer" false positives where many weak components collectively shift gc(k).

**Protocol:**
```python
# Step 1: AtP to rank components
scores = atp_scores(model, clean_input, corrupt_input, metric=audio_metric)
top_k_components = scores.topk(k=20)

# Step 2: Full patching only on top-k
gc_k_refined = patch_components(model, top_k_components)
```
