## Unified Theory: AND/OR Gate as Audio-Reliance Index

### Core Claim (from Q001/Q002 validated)
```
AND-gate feature: f activates ONLY when audio is present (not predictable from text alone)
OR-gate feature:  f activates from EITHER audio OR text context alone
AND-frac(layer k) = |AND-gate features at layer k| / |total active features|
gc(k) ≡ "genuine comprehension" = AND-frac(k)  [normalized]
```

### Three-Axis Classification of Any Utterance Segment

```
            HIGH AND-frac              LOW AND-frac
HIGH PPL    Genuine acoustic signal    Contradictory (rare)
            (new word, rare phoneme)
LOW PPL     FAD-biased text-guessing   Hallucination risk zone
            → Q123, Q139
```

**Key insight**: PPL (language model perplexity) + AND-frac together predict:
- High PPL + High AND = correct, audio-grounded transcription (Q132)
- Low PPL + Low AND = FAD bias zone — model guessing from context (Q123)  
- Low AND across ALL steps = hallucination onset (Q134)
- AND→OR transition at silence/noise = hallucination trigger (Q134)

### Temporal Structure (gc(k) across decoder steps)

```
Step 1  2  3  4  5  6  7  8  ...
AND: ██ ██ ██ ▓▓ ▓▓ ░░ ░░    Normal decay (model uses audio early, text late)
AND: ██ ██ ░░ ░░ ░░ ░░ ░░    Early collapse = hallucination risk (Q134)
AND: ██ ██ ██ ░░ ██ ██ ░░    U-shape = re-anchoring (Q146)
AND: ██ ██ ██ ██ ██ ██ ██    Plateau = hard acoustic content (Q146)
```

**t* = step where AND-frac drops below threshold** → "transition point"

t* large → model stays audio-grounded longer → lower WER expected
t* small → early text-mode → FAD bias / hallucination risk

### RAVEL Metrics Integration (Q123, Q133)
- **Cause(k)**: does intervening on feature k cause output change? → AND-gate features should have high Cause (audio IS the cause)
- **Isolate(k)**: is feature k specific to one concept? → AND-gate features should have high Isolate (audio-specific, not entangled with text priors)
- **Test**: FAD-biased phonemes = High Cause (text context IS causing prediction) but Low Isolate (feature entangled with text + audio)
- This makes RAVEL a *diagnostic* for FAD bias — not just a classification tool

### ENV Taxonomy Integration (Q126, Q136, Q145)
```
ENV-1 (hub): high betweenness centrality → semantically universal features
             → should be AND-gate: required for any phoneme recognition
             → hypothesis: these are cross-language universals (Q145)

ENV-3 (isolated): low connectivity → fine-grained, specific features
                  → maps to RVQ-N (high-frequency acoustic detail)

```
Connection: ENV type predicts AND-gate likelihood AND RVQ layer depth (Q126).

### Safety/Jailbreak Extension (T5, Q129, Q138, Q141)
```
Clean audio:      AND-frac stays high for safety-relevant features
Jailbreak audio:  adversarial noise → t* shifts LEFT → safety features go OR-gate
                  = model stops "listening" to safety-relevant acoustic cues
```
This unifies speech safety with the core T3 framework: **jailbreak = induced FAD bias on safety features**.
