## Background: M7 and M9 Defined

### M9 — Causal Abstraction Consistency (CAC)
From Q072 design: `CAC(F, t) = IIA_score(F) × RAVEL_audio_cause(F) × Isolate_in(F, t)`
- **IIA** (Interchange Intervention Accuracy): causal interchange test — does patching F change output consistently?
- **RAVEL_audio_cause**: how much of F's variance is explained by the audio signal (not context priors)?
- **Isolate_in**: at decode step t, does F remain isolated from unrelated concepts?
- **CAC = high** → feature is causally grounded in audio AND isolated AND intervention-consistent
- **CAC = low** → feature is activated by context priors (not audio), unstable, leaky

### M7 — Spurious Correlation (Audio adaptation)
From AudioSAEBench §3.2: feature F fires for acoustically/articulatorily unrelated phonemes.
- χ² test: does phoneme distribution in F's top-decile frames match chance activation?
- **M7_high** → feature fires for "any" phoneme → OR-gate-like (context-sufficient, audio-agnostic)
- **M7_low** → feature fires for clustered phoneme class → AND-gate-like (audio-specific)

## Core Prediction: Safety Signature = (gc(k*), M9, M7) Triple

| Condition | gc(k*, harmful) | M9(F_safe, harmful) | M7(F_safe, harmful) | Interpretation |
|-----------|-----------------|---------------------|---------------------|----------------|
| Aligned | LOW (0.3) | LOW (0.2) | HIGH (0.7) | AND-gate suppresses audio; features become spurious (context-driven) |
| OBLITERATUS | HIGH (0.65) | HIGH (0.7) | LOW (0.3) | AND-gates destroyed; audio flows freely; features causally grounded again |
| Base | MED (0.55) | MED (0.5) | MED (0.5) | No safety conditioning; neutral audio processing |

**Mechanism behind M7 flip for aligned model on harmful prompts:**
When AND-gate fires (aligned, harmful), the safety AND-gate features activate for ALL phoneme contexts (they respond to the *context* of harm, not specific phonemes). This looks like M7_high (spurious cross-phoneme correlation) — the feature is context-driven, not audio-specific. After OBLITERATUS: features at k* revert to audio-specific activation → M7 drops back to low.

**M9 flip for aligned vs OBLITERATUS on harmful prompts:**
- Aligned: harmful input → AND-gate suppression → features at k* have low IIA (interventions don't propagate, the gate blocked them) AND high RAVEL_context (features explained by safety context, not audio content) → CAC low
- OBLITERATUS: AND-gate destroyed → audio flows → features have high IIA (patching changes output) AND high RAVEL_audio → CAC high

## Three-Metric Divergence Score

Define safety signature distance:
```
S_safe = (gc(k*, benign), M9(benign), M7(benign))   # Reference
S_unsafe = (gc(k*, harmful), M9(harmful), M7(harmful))  # Test probe

Divergence D = ||S_safe - S_unsafe||_2  (L2 distance in 3-dim signature space)
```

**Prediction:**
- Aligned model: D_aligned_harmful >> D_aligned_benign ≈ 0  (safety probe fires)
- OBLITERATUS: D_obliteratus_harmful ≈ 0  (no divergence — safety gate gone)
- Detection: if D < θ_divergence → flag model as potentially safety-bypassed

**This is the core T5 paper contribution**: gc(k) + M9 + M7 divergence is a non-invasive, model-agnostic safety audit tool.

## Paper T5 §3 Addition

> "We propose a three-metric safety signature: gc(k*), M9 (Causal Abstraction Consistency), and M7 (Spurious Correlation). On aligned models, safety-critical prompts trigger a characteristic tri-metric divergence — gc suppression, M9 collapse, and M7 elevation — corresponding mechanistically to AND-gate activation under harmful inputs. Models subjected to uncensoring attacks (OBLITERATUS) show near-zero divergence across all prompt types, as the AND-gate circuitry has been ablated. This divergence score D provides a non-invasive safety audit applicable to any audio-LLM with SAE access."
