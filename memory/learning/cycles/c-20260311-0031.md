## Motivation

SPIRIT (Djanibekov et al. EMNLP 2025, arXiv:2505.13541) defends against audio jailbreaks by
suppressing adversarial activations at the **neuron level** (MLP neurons). This works but has
a key limitation: neurons are polysemantic — suppressing neuron i to block one attack may
degrade benign audio processing (false-positive patch damage).

**Hypothesis:** SAE features are more monosemantic than neurons. A feature-level defense should:
1. Be more *precise* (fewer benign features harmed)
2. Be more *interpretable* (blocked feature has a human-readable label)
3. Enable "interpret-then-defend": first diagnose what semantic content the attack exploits,
   then surgically neutralize only that feature.

This spec formalizes the Interpret-Then-Defend framework and positions it as a contribution
for the MATS research task proposal.

## Framework: Interpret-Then-Defend (ITD)

### Two-Stage Pipeline

```
Stage 1: INTERPRET
  Input: adversarial audio sample x_adv
  Step a: Run x_adv through Whisper + AudioSAE → get feature activations {a_f(x_adv)}
  Step b: Compare to benign baseline {a_f(x_benign)} using ΔAS(f):
          ΔAS(f) = a_f(x_adv) - mean_benign(f) / std_benign(f)  [z-score]
  Step c: Find adversarial feature set A = {f : ΔAS(f) > threshold τ}
  Step d: Query KG or label dict for each f ∈ A → human-readable description
  Output: Alert: "Attack exploits features [F_i, F_j ...] associated with [concept_i, concept_j ...]"

Stage 2: DEFEND
  Input: adversarial audio sample x_adv, adversarial feature set A
  Option A (ablation): clamp a_f = 0 for all f ∈ A before decoder → harmless transcript
  Option B (steering): steer a_f toward benign distribution mean for f ∈ A
  Option C (flag-only): pass sample to human review with feature explanation (interpretable safety)
  Output: defended transcript or alert
```

### Key Design Difference vs SPIRIT

| Dimension | SPIRIT (neuron-level) | ITD (feature-level) |
|-----------|----------------------|---------------------|
| Granularity | MLP neurons (polysemantic) | SAE features (monosemantic) |
| Interpretability | None (neuron → opaque) | Feature label via KG/concept dict |
| Precision | Blunt — suppresses all functions of neuron i | Surgical — targets only adversarial concept |
| Explainability | "Neuron 42 fired" | "Attack exploits voiced-fricative features" |
| Defense mode | Ablation only | Ablation / steering / flag |
| CPU-feasible? | Yes | Yes (forward pass + feature lookup) |

## Evaluation Plan: AUC Comparison (ITD vs SPIRIT)

### Dataset
- SPIRIT adversarial examples (arXiv:2505.13541 — publicly described stimuli)
- Synthetic audio jailbreaks: TTS-generated harmful content embedded in innocuous audio wrappers
- Benign LibriSpeech samples as true negatives

### Metrics
1. **Attack detection AUC** — ROC curve, comparing ITD feature-level detection vs SPIRIT neuron-level
   - Prediction: ITD AUC ≥ SPIRIT AUC (same or better detection power with fewer false alarms)
2. **Precision on benign audio**: what fraction of benign samples get ≥1 feature ablated?
   - SPIRIT: ablates MLP activations broadly → higher benign disruption
   - ITD: ablates only features ΔAS > τ → lower benign disruption
3. **Interpretability coverage**: what fraction of detected attacks have a labeled feature (not just an index)?
   - Expected: ≥40% if AudioSAE concept labels available
4. **Task preservation under defense**: WER on defended adversarial vs clean
   - SPIRIT benchmark: defense reduces jailbreak success by 99% but WER on clean?
   - ITD target: comparable defense with WER_defended / WER_clean ≥ 0.98

### CPU Feasibility
- Forward pass: Whisper-small, 10-30s audio → ~0.5s on MacBook Air M2
- SAE feature lookup: O(n_features) per sample, n_features ≈ 8192 → negligible
- AUC computation: 200 adversarial + 200 benign samples → ~10 min total
- **Tier 1: auto-allowed**

## Connection to MATS Research Task Proposal

**T5 objective:** "Build audio jailbreak detection prototype + MATS research task proposal"

This spec is the technical core of the MATS proposal. Key framing:

> "Current audio jailbreak defenses suppress activations at the neuron level (SPIRIT) —
> effective but uninterpretable. Interpret-Then-Defend proposes a feature-level alternative
> using sparse autoencoders (SAEs) to isolate and label the specific acoustic concepts
> exploited by each attack. This enables *post-hoc forensics* (what did the attack exploit?)
> and *surgical defense* (suppress only the adversarial feature, not the whole neuron)."

**MATS-relevant angle:** Interpretability-based safety — not just "does it work" but "why it
works and what it tells us about the attack's semantic structure."

## Open Questions (not blocked)

1. **Feature sparsity at attack sites**: do adversarial inputs cause feature saturation
   (all features high) or targeted feature activation (specific features spike)?
   — SPIRIT implies targeted; ITD hypothesis assumes this
   
2. **Threshold τ selection**: tau=2.0 (2σ) is standard; validate on SPIRIT held-out set
   
3. **Concept label coverage**: AudioSAE labels ~8K features for Whisper-large. Are adversarial
   features interpretable (labeled) or in the unlabeled "dead feature" zone?
   
4. **Cross-model generalization**: ITD trained on Whisper — does it transfer to Qwen2-Audio's
   audio encoder? (Paper B AudioSAEBench Cat 4 cross-model controllability test)

## Relationship to Paper B (AudioSAEBench)

**Category 4 (Causal Controllability):** ITD directly demonstrates steerable control of
SPIRIT-class attacks via feature ablation. This is an applied instance of Cat 4's
`steering_precision` metric.

**Category 5 (Grounding Sensitivity):** Adversarial audio that causes a model to follow
malicious text commands rather than audio content = a jailbreak exploiting the grounding
failure mechanism. ITD can be framed as: "defend by restoring audio-grounding features."

**Cross-paper framing (Paper A → Paper B → ITD):**
```
Paper A:  Finds Listen Layer L* (where audio grounding occurs)
Paper B:  Finds which SAE features at L* carry audio-grounded vs text-predicted info
ITD:      Uses L* features for surgical jailbreak defense
```

This chain is a natural MATS research proposal story: mechanistic understanding → feature
identification → safety application.
