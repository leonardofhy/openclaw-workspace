## Problem

Whisper's internal feature graph has two extreme node types (ENV taxonomy from SuperPos / MI work):
- **ENV-1 (hub features):** high in-degree, activated by many inputs, strongly connected — *broadly responsive*
- **ENV-3 (isolated features):** low connectivity, activated rarely, monosemantic — *narrowly tuned*

Neural audio codecs (EnCodec, DAC, SoundStream) use Residual Vector Quantization (RVQ):
- **RVQ-1 (first codebook):** captures coarse, semantically rich structure — pitch, speaker identity, broad phoneme class
- **RVQ-N (Nth residual, N>4):** captures fine-grained acoustic residual — prosodic nuance, timbre detail, noise texture

**Hypothesis:** The ENV connectivity class predicts which RVQ layer a feature is most "compatible" with:
- ENV-1 hub ↔ RVQ-1 (both = coarse/semantic level)
- ENV-3 isolated ↔ RVQ-N (both = fine acoustic level)

---

## Method (CPU, Tier 0 — design + mock)

### Matching Protocol

1. **Codec feature extraction:** For a batch of audio clips, extract RVQ-1 and RVQ-N token sequences from a pretrained codec (EnCodec 24kHz). RVQ-1 = first codebook residual. RVQ-N = Nth residual (N=8 for EnCodec-32kbps).

2. **Whisper feature extraction:** Run Whisper encoder; extract intermediate activations at layer L (e.g., L=6 for Whisper-base). Each activation = one feature vector per audio frame.

3. **ENV classification:** Build feature co-activation graph. Node = feature index (SAE latent). Edge weight = Pearson r of activation patterns across clips. ENV-1 = top 10% by degree; ENV-3 = bottom 10% by degree.

4. **RVQ alignment:** For each audio frame, compute:
   - Correlation of ENV-1 feature activations with RVQ-1 codebook token identity (one-hot probe)
   - Correlation of ENV-3 feature activations with RVQ-N token identity
   - **Prediction:** r(ENV-1, RVQ-1) > r(ENV-1, RVQ-N) and r(ENV-3, RVQ-N) > r(ENV-3, RVQ-1)

5. **Intervention protocol (from DoD):** Per ENV type, intervention should target different residual levels:
   - ENV-1 hub intervention → modify RVQ-1 (semantic-level effect)
   - ENV-3 isolated intervention → modify RVQ-N (acoustic-detail effect)
   - **Hypothesis:** ENV-3 interventions produce local, surgical acoustic changes; ENV-1 interventions produce global semantic shifts

## Connections

- **AND/OR gate alignment:** ENV-1 hub features are likely OR-gate (broadly activated from either text or audio path); ENV-3 isolated features are likely AND-gate (specific activation condition = AND of audio feature + context). This links to Q096 / Q107 / Q123 synthesis in `kg/and-or-gate-ravel-fad-unification.md`.
  - ENV-3 isolated × AND-gate × RVQ-N fine-acoustic = three-way alignment hypothesis (analogous to Q144's T-SAE × Schelling × AND-gate triple alignment)
  - ENV-1 hub × OR-gate × RVQ-1 semantic = another three-way (semantic features are text-predictable, polysemantic, coarse)

- **T5 safety (Q136, Q129):** Multi-speaker boundary detection and adversarial audio detection should target ENV-3 × RVQ-N features (fine acoustic = where manipulation signatures live), not ENV-1 × RVQ-1 (semantic level masked by OR-gate).

- **T3 hallucination (Q134):** Hallucination = AND-gate dropout = ENV-3 features lose activation = fine acoustic layer representation collapses. Model falls back to RVQ-1-level (coarse semantic) generation = hallucinated words that are phonetically plausible but acoustically ungrounded.

- **Intervention design:** Controllable audio editing via codec swap:
  - Swap RVQ-1 tokens between speakers → semantic identity transfer (speaker voice clone)
  - Swap RVQ-N tokens with noise → fine acoustic degradation without semantic change
  - ENV-3 intervention = surgical RVQ-N level = audiologist-level editing

## Open Questions

1. **Cross-layer analysis:** Does the ENV-1 ↔ RVQ-1 correspondence hold at every Whisper encoder layer, or only at specific layers (e.g., near t*)?
2. **Task dependency:** Does the ENV-RVQ alignment vary by task (ASR vs. audio QA vs. speaker ID)? ENV-1 features in speaker-ID heads might be more RVQ-1-aligned than in phoneme-decoding heads.
3. **Codec choice:** EnCodec vs. DAC vs. Mimi — do different codec architectures preserve the same ENV-RVQ correspondence? (Normalization scheme differences could disrupt.)
4. **Safety surface:** If ENV-3 (AND-gate) features carry RVQ-N acoustic residuals, adversarial perturbations targeting RVQ-N should primarily attack ENV-3 features. Verifiable prediction: FGSM attacks on RVQ-N tokens → ENV-3 feature activation collapses.
