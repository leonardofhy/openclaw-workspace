# Q092: Schelling Features × AND/OR Gates
**Cycle:** c-20260314-1045 | **Phase:** explore-fallback | **Track:** T3 | **Action:** learn

## Research Question
Are cross-seed-stable (Schelling) SAE features more likely to be AND-gate features?

## Framework Recap

**Schelling stability** (MicroGPT context): A SAE feature is "Schelling-stable" if it appears consistently across ≥N of 5 random seeds trained on identical data. These features represent the "focal solution" that models converge to — the natural way to represent that concept given the architecture + data constraints.

**AND-gate features**: Activate only when BOTH audio context AND text context align (e.g., phoneme /s/ predicted by text AND audio waveform confirms it). NOT predictable from text alone.

**OR-gate features**: Activate when EITHER audio OR text is sufficient (e.g., phoneme highly predictable from language model alone; audio just confirms).

## Core Argument

### Why AND-gate features should be MORE Schelling-stable:

1. **Constraint uniqueness**: AND-gate features must represent audio-specific information that text cannot predict. The acoustic signal is a strong, specific constraint — there's essentially ONE correct representation for, say, the high-frequency fricative burst of /s/ in a given phoneme context.

2. **Fewer equivalent solutions**: OR-gate features (text-predictable) can be implemented via multiple equivalent circuits: pure language model path, phoneme-from-bigrams, syntactic prediction... many solutions satisfy OR. AND-gate features require the audio path to be "load-bearing" — this is a stricter requirement with fewer implementations.

3. **RAVEL Isolate connection** (from Q099 cycle): High-Isolate features (disentangled, monosemantic) should be Schelling-stable (from RAVEL analysis). AND-gate features, being audio-specific, should have higher Isolate scores — because the attribute (audio information) has fewer confounding causes in the model's computation.

4. **Analogy**: In human languages, Schelling points are "natural" focal solutions under coordination pressure. AND-gate features are the "natural" representation for purely acoustic information — every model that processes audio must converge to something similar.

### Potential confound: OR-gate features for universal phonemes may also be stable

- Common phonemes (/a/, /s/, /t/) appear in virtually every utterance → extremely stable across seeds due to high frequency
- Counterargument: stability from frequency ≠ stability from constraint uniqueness. High-frequency OR-gate features are stable because of data pressure, but may be implemented differently across seeds (multiple circuit paths work). High-constraint AND-gate features are stable because there IS only one solution.
- Testable: Schelling stability at FEATURE level (which SAE latent) vs at CONCEPT level (which phoneme). AND-gates stable at feature level; OR-gates stable at concept level but potentially distributed across multiple features per seed.

## Testable Prediction

For MicroGPT × gc(k) setup:
