# Cycle c-20260317-1445 — learn — GSAE paper scan

**Task:** Q083 (M9-gated adversarial detector), Q095 (safety audit design)
**Paper:** GSAE: Graph-Regularized Sparse Autoencoders for Robust LLM Safety Steering
**arXiv:** 2512.06655 — Yeon, Cinus, Wu, Luceri (Dec 2025 / Feb 2026)

## Problem
Standard SAE safety steering assigns each concept to a *single* latent feature.
Evidence: "refusal" and "temporality" are distributed across multiple features, not isolated.
→ Single-feature steering fails (42% refusal rate).

## Method: GSAE
- Extend SAE loss with a **Laplacian smoothness penalty** on the neuron co-activation graph
- Co-activation graph: edge weight between features i,j ∝ frequency of joint activation
- Laplacian penalty encourages co-activating features to encode similar/consistent concepts
- At inference: **two-stage gating** — first detect harmful prompt/continuation, then steer
- Steering = weighted combination of safety-relevant feature directions (not single vector)

## Results
- 82% selective refusal rate vs 42% for standard SAE steering
- 70% TriviaQA / 65% TruthfulQA / 74% GSM8K (utility preserved)
- ≥90% refusal against GCG, AutoDAN jailbreaks
- Generalizes across LLaMA-3, Mistral, Qwen, Phi

## Connections to T5 tracks

**Q083 (M9-gated detector):**
- GSAE's two-stage gating = "detect first, intervene second" → exact same structure as our M9 dual-factor alarm
- But GSAE uses co-activation graph edges; we use causal consistency (M9)
- Synergy: can M9 consistency score = a *causal* refinement of GSAE's correlational graph?
  - GSAE edge: i,j co-activate (correlation)
  - M9 consistency: i→j is causal (patching-validated)
  - Hybrid: GSAE identifies candidate safety feature clusters; M9 confirms causal ones
  - This reduces FP rate compared to pure GSAE

**Q095 (safety audit design):**

**New gap (Gap #33): Audio-GSAE**
- GSAE tested only on text-LLMs (LLaMA-3, Mistral, Qwen, Phi)
- No audio-LLM version exists
- Qwen2-Audio has cross-modal features; audio jailbreak bypasses text-space refusal
- Hypothesis: audio jailbreaks succeed *because* they don't activate text-space refusal features
- Audio-GSAE would need to add cross-modal co-activation edges (audio features ↔ text features)
- This is tractable for AudioSAE (layer 6-7) + GSAE penalty in the connector region

**Connection to gc(k):**
- Laplacian smoothness ≈ measuring feature "coherence" across a computation step
- gc(k) at layer k measures how consistently audio information is gated into text predictions
- High gc(k) = high causal coherence = features in that layer form a tight cluster
- → gc(k) collapse might show up as GSAE co-activation graph fragmentation at layer t*
