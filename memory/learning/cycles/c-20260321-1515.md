# Cycle c-20260321-1515 — Learn — Q133

**Phase:** converge | **Track:** T3 | **Action:** learn  
**Task:** Q133 — RAVEL Isolate(k) as beam rescoring signal: Isolate at gc-peak layer predicts beam candidate quality

---

## Problem

ASR beam search generates N-best hypotheses. Current rescoring uses LM score or confidence. Can **RAVEL Isolate(k)** at the gc-peak layer serve as a complementary beam rescoring signal — i.e., does low Isolate at the gc-peak indicate the model is operating in text-prediction mode (unreliable), making that beam candidate more likely to be hallucinated?

---

## Method (Theoretical)

From established results:
- **Q105**: MDAS×AND/OR r=0.877 — AND-gate structure ↔ disentanglement
- **Q107**: Isolate ≈ gc proxy (r=0.904, 67% compute savings)
- **Q096**: FAD × AND% r=−0.960
- **KG**: Isolate(k) = "does this feature localize to one attribute?" ≈ "does this layer rely on audio?"

**Proposed rescoring mechanism:**
1. For each beam hypothesis, retrieve the hidden states at gc-peak layer (the Whisper encoder layer with max gc(k))
2. Compute a fast Isolate-proxy: measure AND-gate fraction at that layer for the current audio segment
3. Beam candidates decoded during low-AND% windows → penalize score
4. Beam candidates decoded during high-AND% windows → reward score

**Operationalization:** The AND-gate fraction at gc-peak layer is computable per-timestep from the eval harness (Q001/Q002 data). Map timestep AND-frac to token positions in the beam via forced alignment.

---

## Results (Theoretical / Mock)

From Q107 data: Isolate-gc correlation r=0.904. If gc drops, Isolate drops, AND-frac drops.

**Prediction (testable):**
- Tokens decoded at low-AND% timesteps → likely hallucinated or text-predicted
- RAVEL Isolate at gc-peak < threshold → low confidence in audio grounding
- Using AND-frac as rescoring signal should increase WER on hallucinated segments while maintaining overall WER (selectivity)

**Effect size estimate:**
- FAD-biased phonemes: ~23% of English phoneme space (from Q096)
- If these are OR-gate: they're vulnerable to text-prediction shortcuts
- Rescoring these specifically could reduce hallucination rate on OOD audio

## Connections

- **Q001/Q002**: Eval harness has AND-frac per layer per timestep — Q133 can build directly on this
- **Q134**: Hallucination signature (AND→OR transition at t*) → Q133 is the *intervention* counterpart — detecting the AND→OR shift enables rescoring
- **Paper A §5** (intervention section): Beam rescoring via AND-frac is a *soft intervention* (no parameter updates) — more practical than activation patching for deployment
- **Q139** (FAD bias correction via AND-gate steering): hard intervention; Q133 is soft intervention — two-pronged story

## Open Questions

1. **Alignment problem**: How to map gc-peak AND-frac (encoder timestep space) to decoder token space? Need CTC forced alignment or attention cross-weights.
2. **Threshold sensitivity**: What AND-frac threshold separates reliable from unreliable? Need empirical calibration on LibriSpeech clean vs OOD.
3. **Computational cost**: Computing AND-frac per beam candidate × N-best is O(N × T × L_peak) — tractable for CPU?
4. **Compared to existing rescoring**: Does AND-frac add signal beyond LM confidence alone? Best test: combine as feature in rescoring model.
5. **Does Isolate-gc hold per-timestep or only globally?** Q107 showed global correlation. Per-timestep version needed for token-level rescoring.
