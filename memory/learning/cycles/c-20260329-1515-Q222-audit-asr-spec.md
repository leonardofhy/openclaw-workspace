# AUDIT-ASR Benchmark Design Spec
**Cycle:** c-20260329-1515 | **Task:** Q222 | **Phase:** converge/T3

## Overview

**AUDIT-ASR** (AND-frac-Unified Diagnostic Interpretability Testset for ASR) is a diagnostic benchmark that annotates ASR failure types by their AND-frac signature at L* in Whisper.

**Core hypothesis:** Different ASR failure modes (hallucination, deletion, substitution) have distinguishable AND-frac profiles at the commit layer L*.

---

## Dataset Spec

### Error Taxonomy

| Error Type | Definition | Expected AND-frac Signature |
|-----------|-----------|----------------------------|
| **Hallucination** | Output tokens with no acoustic grounding | AND-frac HIGH at L* despite noisy/absent audio → premature overconfident commit |
| **Deletion** | Reference words missing from hypothesis | AND-frac LOW or delayed → no commit formed; token "falls through" |
| **Substitution** | Wrong word, acoustically similar | AND-frac MODERATE; commit layer present but peak misaligned to wrong token |
| **Insertion** | Extra tokens not in reference | AND-frac spike on spurious token → false commit trigger |
| **Repetition** | Same word repeated | AND-frac multiple peaks at same position → commit oscillation |

### Data Sources

1. **LibriSpeech test-clean / test-other** — controlled baseline (deletions, substitutions)
2. **L2-ARCTIC** — accented speech (substitution-heavy, commit layer stress)
3. **CHiME-4 / DEMAND augmented** — noisy speech (hallucination-inducing)
4. **Synthetic hallucination corpus** — near-silence inputs fed to Whisper (hallucination ground-truth)
5. **Manual annotation layer** — 200 samples hand-labeled with error type + timestamp alignment

### Sample Budget

| Split | Size | Purpose |
|-------|------|---------|
| dev | 200 | AND-frac threshold calibration |
| test | 500 | Benchmark evaluation |
| probe | 50 | Human-verified, full activation stored |

Total: 750 samples. Feasible on CPU with Whisper-base.

---

## AND-frac Feature Extraction

For each sample, extract:
1. `andfrac_L*` — AND-frac value at commit layer (scalar)
2. `andfrac_peak_layer` — layer where AND-frac peaks (commit depth)
3. `andfrac_Lstar_ratio` — L*/D (normalized commit depth)
4. `andfrac_pre_peak` — AND-frac at L*-2 (ramp-up rate)
5. `andfrac_post_peak` — AND-frac at L*+2 (decay rate)
6. `steering_delta_wer` — WER change after power-steering at L* (+α)
7. `andfrac_variance_across_tokens` — per-token commit stability

**Label space:** {hallucination, deletion, substitution, insertion, repetition, correct}

## Evaluation Protocol

### Primary Task: Error-type prediction from AND-frac features
- Input: AND-frac feature vector (7 dims)
- Output: error_type label
- Metrics: macro-F1, per-class precision/recall
- Baseline: WER-only (no AND-frac features)

### Secondary Task: Controllability validation
- Apply power-steering at L* for samples predicted "hallucination"
- Measure WER delta — does steering suppress hallucinations specifically?
- Hypothesis: AND-frac-guided intervention more targeted than uniform steering

### Diagnostic Task: Universality check
- Compare L*/D across Whisper-tiny / Whisper-base (same dataset)
- Hypothesis: L*/D ratio is architecture-invariant (≈0.5)
