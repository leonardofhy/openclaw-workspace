# c-20260315-1415 — Plan: Q108 Design Doc

**Task**: Q108 — Jailbreak = Isolate peak leftward shift  
**Track**: T5 | **Type**: design | **Action**: plan  
**Hypothesis**: Audio jailbreaks cause the model to enter text-only mode *before* the normal gc peak, 
detectable as a leftward shift in the min-Isolate(k) layer.

---

## Background

- **gc(k)**: ground-contribution metric at decoder layer k — measures how much the audio cross-attention 
  contributes to the final output. Peaks at some k*, then decays.
- **Isolate(k)**: counterfactual metric — prediction accuracy when only layers ≤k can attend to audio. 
  min-Isolate layer ≈ last layer where audio info still matters → call this `t*`.
- **Normal behavior**: `t*` aligns with or slightly precedes gc peak layer.

## Hypothesis (formal)

Let `t*_normal` = min-Isolate layer on clean audio prompts.  
Let `t*_jailbreak` = min-Isolate layer on adversarially perturbed audio.

**Prediction**: `t*_jailbreak < t*_normal` — jailbreak audio is processed as "text-like" earlier, 
audio influence is discarded before it would normally be integrated.

**Intuition**: Adversarial audio perturbations cause the model to distrust/skip audio channel, 
collapsing to text-only path prematurely. This is mechanistically detectable via `t*` shift.

---

## Detection Protocol (Mock Test)

### Setup
```
Model: Whisper-medium or AudioLM (CPU-friendly mock)
Inputs:
  - clean_prompts: N normal speech samples
  - jailbreak_prompts: same N samples + adversarial noise (FGSM or uniform Gaussian for mock)
Metric: t* = argmin_k Isolate(k) per sample
```

### Procedure
1. For each sample, compute `Isolate(k)` for k = {1, 4, 8, 12, 16, 20, 24} (subset of layers)
2. Find `t* = argmin_k Isolate(k)`
3. Compare distribution: `t*_clean` vs `t*_jailbreak`
4. Test: Mann-Whitney U, or just compare medians

### Expected result
- `median(t*_jailbreak) < median(t*_clean)` by ≥ 2 layers
- Distribution shift visible in histogram

### Null result interpretation
- If no shift: adversarial noise still goes through normal audio path → different jailbreak mechanism
- If rightward shift: audio over-reliance (unexpected, worth investigating)

## T5 Detection Framing

**Why this matters for T5 (Listen-Layer Audit)**:
- `t*` is a cheap per-sample diagnostic (no SAE needed for initial detection)
- A jailbreak detector based on `t*` threshold could be:
  - Lightweight (just a few forward passes with ablations)
  - Interpretable ("model stopped listening at layer 8 instead of 16")
  - Complementary to M9/M7-based detector (Q083)

**Two-stage detection**:
1. Stage 1: `t*` threshold check (fast, cheap) → flag suspicious
2. Stage 2: M9 profile audit on flagged samples (expensive but precise)

## Mock Implementation Plan (Tier 0/1)

```python
# Scaffold: t_star_jailbreak_detector.py
# Input: audio_feature_activations[n_layers], Isolate_scores[n_layers]
# Output: t_star (int), flag (bool if < threshold)

def compute_t_star(isolate_scores):
    return np.argmin(isolate_scores)

def jailbreak_flag(t_star, threshold_layer):
    return t_star < threshold_layer
```

**Threshold calibration**: use 10th percentile of `t*_clean` distribution as threshold.
