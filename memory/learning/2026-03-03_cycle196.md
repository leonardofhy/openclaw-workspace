## Content

### Paper
**"Finding Alignments Between Interpretable Causal Variables and Distributed Neural Representations"**  
Geiger, Wu, Potts, Icard, Goodman — Stanford Pr(Ai)² Group, arXiv:2303.02536v4 (Feb 2024)

### Core Problem DAS Solves (and why it matters for Paper A)

**Localist patching limitation**: Prior causal abstraction methods (including vanilla activation patching, ROME, causal tracing) assume high-level variables map to **disjoint groups of neurons**. This is almost never true:
- Smolensky (1986): distributed representations = linear combinations, not individual neurons
- AudioSAE (2026): Whisper needs ~2000 features per layer for speech concepts vs. tens for text → **extreme polysemanticity**
- Implication: vanilla layer patching on Whisper residual stream MISSES causal structure because it patches all dimensions including irrelevant ones → noise, low IIA

**DAS solution**: Distributed Interchange Interventions (DIIs):
1. Take a group of neurons at layer k with representation h
2. Rotate: `h_rot = R·h` (change of basis via learned rotation matrix R)
3. Fix targeted dimensions: `(h_rot)_d = (h_source_rot)_d` for targeted dimension indices d
4. Rotate back: `h_new = R⁻¹·h_rot`

The rotation R is **learned by gradient descent** to maximize IIA (not brute-force searched). R is constrained to be orthogonal (so R⁻¹ = Rᵀ).

### Why Linear Rotation is the Right Tool for Paper A

**Sutter et al. 2507.08802 (NeurIPS 2025 Spotlight)** showed: with *non-linear* maps, any NN maps to any algorithm at 100% IIA → vacuous. Linear rotation constraint is therefore not just convenient — it's the **theoretical requirement for causal abstraction to be non-trivial**.

**Geiger DAS confirmation**: gradient descent over orthogonal matrices finds the BEST linear alignment, not an arbitrary one. The rotation matrix R can be parameterized via:
- Cayley parameterization: R = (I - A)(I + A)⁻¹ where A is skew-symmetric
- OR: SVD projection of unconstrained gradient update

For Paper A, `pyvene.RotatedSpaceIntervention` implements exactly this, with automatic gradient-based learning of R.

### IIA as the Graded Metric

Interchange Intervention Accuracy (IIA) = proportion of cases where the DII-intervened model produces the same output as the high-level model would:

```
IIA(k, F) = E_base,source [ NN_k(base→source for F) == HLM(F(source)) ]
```

For Paper A's gc(k):
- **High-level model (HLM)**: binary phonological variable F (e.g., voicing: voiced=1, unvoiced=0)
- **Low-level model (NN)**: speech LLM (e.g., Qwen2-Audio, DeSTA2-8B)
- **Intervention**: patch the DAS-learned subspace at layer k with source's value of F
- **gc(k) = DAS-IIA at layer k** = probability that patching F-direction makes model "switch" its behavior to match F's source value

**Layer-wise sweep**: gc(k) curve over layers k=0,1,...,L-1 → the layer where gc peaks = "Listen Layer"

### Direct Analogy to NLI Experiment (Paper A Validation)

Geiger's NLI experiment:
- High-level variable: binary entailment(w_p, w_h) ∈ {entails, contradicts}
- Stimulus pairs: (dog, mammal) = entails vs. (dog, fish) = contradicts
- DAS finds: 100% IIA at entailment-relevant layer

Paper A analog:
- High-level variable: voicing(phoneme) ∈ {voiced, unvoiced} — from Choi et al. 2602.18899 phonological pairs
- Stimulus pairs: minimal pair audio (e.g., /d/ vs /t/ in identical carrier phrases)
- DAS finds: peak IIA layer = Listen Layer for phonological grounding

**Key difference**: Geiger's NLI is text-only (single modality). Paper A = FIRST application of DAS to multimodal speech-LLM setting with cross-modal variable (audio → LLM behavior).

### Two-Variable High-Level Model Design

For Paper A, there are two nested high-level variables:
1. **PHONOLOGICAL(audio)** → phoneme class (vowel/consonant, voiced/unvoiced, manner, place)
2. **GROUNDING(layer k)** → does LLM use F at this layer for prediction?

The high-level model: `{F(audio), context} → PREDICTION`  
The causal question: at which layer k does patching F(audio) change PREDICTION?

This is cleaner than the original "gc as ratio" formulation because DAS-IIA has a clean causal interpretation: we're directly testing if the model has *causally internalized* the phonological variable at that layer.

### Training Dynamics (Important for Implementation)

DAS training:
- Loss = cross-entropy on prediction after DII-intervention (base input, source values for F)
- Only R is optimized (NN weights frozen)
- Training data: balanced set of (base, source) pairs with known F values
- Convergence: typically fast (<1000 steps for small subspaces)
- Subspace dimensionality: hyperparameter, usually 1-32 dimensions for binary variables

For Paper A:
- Train R separately per layer k (layer sweep = 6-8 independent DAS runs for Whisper-small)
- Subspace dim = 4-8 for voicing direction (Choi et al. showed phonological vectors are low-rank)
- MacBook feasible: ~5 min per layer × 6 layers = 30 min total for Whisper-small gc(k) curve

### New Insight: Two-Stage Experiment Design

Reading Geiger's NLI case carefully reveals something for Paper A:

**Decomposability of representations**: In the NLI case, the entailment variable could be decomposed into representations of the individual words w_p and w_h. In the equality task, the relations w=x and y=z could NOT be decomposed — they were truly abstract relations.

For Paper A: Does the LLM at the "Listen Layer" encode:
- (a) A decomposable representation (word identity of phoneme → voicing derivable) — would show up as high IIA but with phoneme-identity also localized to same subspace
- (b) A truly abstract voicing feature (voicing = abstracted from phoneme identity) — IIA high but phoneme identity NOT in same subspace

This is testable! Run DAS for voicing AND for phoneme-identity at the same layer. If voicing subspace ⊥ phoneme-identity subspace = abstract encoding (interesting!). If overlapping = phoneme-identity includes voicing.

**This is a new Paper A ablation worth adding.** Differentiates "LLM encodes phonological structure directly" vs "LLM retrieves phoneme labels and derives phonology."

## Open Questions

1. **What's the right subspace dimensionality for audio phonological variables?** Choi et al. show voicing has high linear separability in S3M space (~1-3 principal components), but in LLM residual stream the geometry may spread. Recommendation: test d=1,2,4,8,16 and report IIA vs. d curve.

2. **How many minimal pairs needed for reliable IIA?** Geiger et al. used ~few hundred pairs. Choi et al. provide validated stimuli across 96 languages → could use 50 language × 5 pairs = 250 examples for robust bootstrap CI.

3. **Can DAS handle temporal audio representations (frame-level)?** pyvene patches single-position activations. For audio tokens from Whisper (one token per ~80ms), each token = one patch position. This is fine for Whisper encoder (token = frame slice). For SpeechTokenizer, each RVQ layer-1 token = semantic unit → patch correctly.
