## Paper: AudioSAE (Aparin, Kushnareva et al., EACL 2026 main track)
**arXiv:2602.05027** | Code: github.com/audiosae/audiosae_demo

### 1. Problem
SAEs have become powerful tools for interpreting neural representations in LLMs, but audio remains essentially untouched. How do we train and evaluate SAEs for speech/audio encoder models, and can they be practically useful?

### 2. Method
**Models**: Whisper (encoder) and HuBERT — both 12 layers, base/small variants
**SAE Architecture**: TopK variant (BatchTopK, following Bussmann et al. 2024), 6144 features per layer (expansion factor ~8x relative to 768-dim model). Trained on 12 datasets spanning speech, music, environmental sounds, and paralinguistics (FSD50k, Musan, WHAM, MTG-Jamendo, VocalSound, IEMOCAP, CREMA-D, LibriHeavy, VGGSound, etc.)

**Novel Stability Metric**: They introduce a cross-seed/cross-layer SAE consistency metric — two SAEs trained with different random seeds, then feature-match by activation correlation. >50% of features remain consistent across seeds.

**Evaluation Suite (5 methods)**:
1. **Stability metric** (novel) — cross-seed feature matching
2. **Distributional analysis** — frame-level vs audio-level feature ratios by domain
3. **Classification-based** — top-k features for logistic regression on 4 tasks (gender, noise, accent, emotion) + feature unlearning curves
4. **Label-based classification** — identify individual features encoding specific audio events (phonemes, laughter, sneezing, etc.) using MFA phoneme aligner
5. **EEG correlation** — correlate SAE features with human brain activity during audiobook listening (Pz electrode, 19 participants)

**Steering**: Feature suppression for hallucination reduction in Whisper — top-100 features selected from FSD50k non-speech activation, then suppressed at inference on non-speech audio inputs.

### 3. Key Results

**Layer specialization** (Fig. 2 — audio-level features by domain):
- Whisper: music peaks layer 5 (~20-28% audio-level features); speech audio-level peaks mid-layers, drops sharply after layer 6 → **layer 6-7 = acoustic-to-semantic transition zone** (consistent with Beyond Transcription saturation layer finding)
- HuBERT: different profile — speech-related features more spread across layers
- Frame-level speech features RISE at layer 6-7 even as audio-level FALL — local encoding replaces global encoding at the transition

**Classification (top-k probing)**:
- Small k=10–150 features captures most task-relevant info for binary tasks (gender, noise)
- Complex tasks (accent 5-class, emotion 5-class) require k=500–3000 features
- **Erasing** speech concepts: needs 1000s of features (19-27% of all features for phoneme erasure) vs text-based SAEs where 10s of features erase semantic concepts
  - This confirms Gap #19 intuition: speech representations are MORE distributed/redundant than text
  - BUT: top-k and unlearning curves converge FASTER than random → structure exists, just wider spread

**Phoneme encoding** (using MFA aligner):
- Layer 12 features: 92% frame-level phoneme accuracy (Whisper), 89% (HuBERT) — SAE features cleanly encode phoneme identity
- Feature 6106 in HuBERT layer 4 detects "whisper" vocal quality (F1=0.6)
- Laughter: captured by both models, layers 1-6 (HuBERT), layers 1,6,9 (Whisper) — distributed representation
- Animal sounds, breathing: NOT reliably identified → fully distributed encoding (no localized features)
- HuBERT layer 11 features 3249, 3081: speech ONSET and OFFSET detectors!

**Hallucination steering** (Table 2):
- Whisper baseline FPR on non-speech: 0.37 (37% of non-speech clips falsely trigger speech)
- Top-100 SAE features suppressed with α=1: FPR → 0.11 (70% reduction); WER: 5.1% → 5.5% (minimal)
- α=3 (aggressive): severe WER increase — **clear trade-off between hallucination reduction and speech quality**
- Outperforms simple S-vector steering at moderate α

**EEG correlation** (neuroscience angle):
- Some SAE features correlate with Pz EEG at 0-500ms lags (statistically significant, Holm-Bonferroni corrected)
- Correlated features often activate on specific vowels (IPA /ɔ/ or /ɑ/)
- Alignment direction can be positive OR negative — model features don't perfectly mirror brain (expected)
- **Implication**: SAE features are not random — at least some align with human perceptual units

**Auto-interpretation pipeline**:
- Activated frames → audio captions (Xu et al. 2024 captioning model) → GPT-4o mini aggregation → feature label
- Discovers non-annotated features: "ringing alarms", "high-pitched beeping", "birds chirping", "guitar playing"
- FAILS on phoneme-specific features → captioning model too coarse for fine-grained speech features

### 4. Method Limitations (Explicit in Paper)
1. Only base/small models — no scaling experiments
2. Limited task coverage (no speaker verification, enhancement, generation)
3. Auto-interpretation limited by captioning model (music/sound biased, misses phonemes)
4. EEG: only Pz, linear model — likely underestimates alignment

### 5. Connection to Goals

**Track 2 (AudioSAEBench)**:
- Their stability metric = natural Category 0 for AudioSAEBench (pre-existing baseline)
- Their top-k probing / unlearning curves = Category 3 candidate (feature utility for downstream tasks)
- Their layer-wise specialization analysis = Category 2 (feature disentanglement by domain)
- Gap #19: They use custom code; no SAELens compatibility → **this confirms the community infrastructure gap Leo identified**
- Their evaluation covers 4 audio tasks — AudioSAEBench should include TCS(F) + causal metrics + EEG alignment as additional dimensions

**Track 1 (Audio Causal Benchmark)**:
- Layer 6-7 transition zone in Whisper (confirmed by both this paper AND Beyond Transcription) → strong prior for where to place patching interventions
- The 1000s-of-features erasure requirement: **challenges simple patching assumptions** — audio features are distributed; single-feature patching will have small effect. Need SAE-mediated aggregate interventions.

**Track 3 (Listen vs Guess)**:
- Phoneme-level features confirmed at layer 12 with MFA alignment (92% accuracy) → if audio-LLMs pass these features through the connector (Gap #18), they ARE listening
- Speech onset/offset detectors (HuBERT layer 11 features 3249, 3081) = boundary features useful for isolating audio token contributions in DAS

**Idea #7 (Audio T-SAE)**:
- AudioSAE confirms distributed phoneme encoding → T-SAE temporal contrastive loss would NATURALLY group distributed phoneme features by temporal structure
- AudioSAE does NOT exploit temporal structure (as expected from Gap #17) — all features trained i.i.d. across frames
- TCS(F) metric from Idea #7 would directly address the unlearning curve limitation (they measure erasure of a CONCEPT not temporal coherence)

**New insight for Paper B**:
- Their EEG correlation = **unexpected validation path** for AudioSAEBench — biological plausibility as a Category 4 metric?
  - If high-quality SAE features correlate with EEG, this is a novel evaluation dimension
  - But: requires expensive data collection (19 participants EEG) — probably too heavy for benchmark inclusion; flag as "future work" metric

### 6. Open Questions / New Gaps

**Gap (AudioSAE-specific, not previously tracked)**:
- They use CUSTOM evaluation code with no public benchmark protocol → AudioSAEBench can standardize and extend their methodology
- Their stability metric is internal (cross-seed) — does it correlate with downstream utility? AudioSAEBench should validate this
- Their MFA aligner for phoneme labeling (2-step pipeline) = principled method Leo should adopt for TCS(F) metric in Paper B

**Gap #22 (NEW — identified this cycle)**:
**"SAE Feature Consistency vs. Causal Utility in Audio Models"**
- AudioSAE shows >50% feature consistency across seeds = stable representation
- BUT: no causal validation — do consistent features actually cause behavior or are they epiphenomenal?
- This is the exact gap that RAVEL (Huang et al. 2024) addresses for LLMs, and SAEBench's "intervention" category addresses
- For audio: AudioSAEBench Category 1 should be "causal consistency" — top-k features that are BOTH consistent across seeds AND causally verified via patching/steering
- **Paper B contribution**: first audio SAE benchmark that measures causal utility (not just representation quality)
