### 3-Plan Anti-Pattern Post-Mortem

**Problem:** Cycle #224 correctly diagnosed "3 consecutive plans → force reflect/learn" rule triggered (after #219-#223). But then cycles #225, #227, #228 were all `plan` again — a second 3-plan run in direct violation.

**Root cause analysis:**
- The work in #225 (Paper B §3 Method), #227 (Paper B §4 Expected Results), #228 (Paper A §4 Expected Results) was GENUINELY HIGH VALUE — not pitch-bloat. All passed the Q19 anti-bloat check.
- The rule "3 plans → force reflect" was designed to catch pitch-bloat (repeatedly updating unreviewed materials with marginal additions). It fires too eagerly when plans are producing NEW LaTeX sections.

**Refinement:**
> Old rule: "After 3 consecutive `plan` cycles → force `reflect` or `learn`."
> New rule: **Plan-budget per milestone**. Each time a new section milestone is reached (e.g., §1, §2, §3, §4 complete for a paper), one reflect cycle is required before proceeding to the next section. Within a section, up to 2 plan cycles are allowed before the section-complete reflection. The 3-plan global counter resets at each reflect/learn cycle.

**Why this is better:** §1+§2+§3+§4 for BOTH papers = 8 plan milestones completed in Days 5-6. Each was genuine new writing. The old rule treated this identically to pitch-bloat. New rule: milestone-triggered reflect = better hygiene.

**Applied:** Progress.md updated with new rule. Meta-board updated (Q32).
